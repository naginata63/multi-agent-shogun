## 2. 章別構成 (ストーリー + 解説アウトライン + 学習目標)

### 序章 Part A: 生成AIとはなにか — AIが変わった瞬間 (L1: Remember)

**所要**: 15 min / **無料プレビュー**

#### ストーリー (Emotional Hook)
> 「2022年11月、ChatGPTが登場して世界が変わりました。しかし『生成AI』は単なるチャットボットではありません。なぜAIが文章を書き、コードを生成し、画像を作れるのか——その仕組みを15分で理解しましょう。」

- 受講者の「これだ！」ポイント: 「AIって魔法？」と思っている人が、LLMの仕組みを「次の単語を予測するだけ」と理解した瞬間の「なるほど！」体験。

#### 学習目標
- 生成AIとは何か（従来のAIとの違い）を説明できる
- LLM（大規模言語モデル）の基本的な仕組みを理解する
- ChatGPT / Claude / Gemini 等の主要サービスの違いを説明できる
- なぜ2026年今、生成AIが重要なのか理由を言える

#### 解説アウトライン
1. 生成AIとは: テキスト・コード・画像を「生成」するAIの定義
2. 従来のAI（ルールベース・検索）との違い: 「探す」から「作る」への転換
3. LLMの仕組み: 次の単語を確率的に予測するモデル（図解: "吾輩は" → "猫"）
4. 主要サービス比較 (2026年): ChatGPT (OpenAI) / Claude (Anthropic) / Gemini (Google)
5. なぜ今重要か: 開発者1人の生産性が10倍になる「AI開発」の時代
6. 本講座の位置づけ: 生成AIの中でも「開発」に特化したClaude Codeを学ぶ

---

### 序章 Part B: Claude Code とは — AI開発の新しい形 (L1: Remember)

**所要**: 15 min / **無料プレビュー**

#### ストーリー (Emotional Hook)
> 「今までのAI開発は『ブラウザを開いてチャットする』だけでした。Claude Codeは、あなたのターミナルの中でAIが直接コードを読み、書き、実行する——まったく新しい開発体験です。」

- 受講者の「これだ！」ポイント: 3分のデモで「ブラウザAI vs CLI AI」の違いを目で見せる。Chat版は会話だけ、Code版は自動でファイルを編集・テストを実行。このギャップが「やってみたい」のトリガー。

#### 学習目標
- Claude Codeと他のAIツール（ChatGPT、Cursor等）の違いを説明できる
- なぜCLI（コマンドライン）でAIを使うのか理由を言える
- 本講座の終着点（3つの学習到達点）を把握する

#### 解説アウトライン
1. AI開発ツールの現在（2026年）: Chat / IDE拡張 / CLI の3形態
2. Claude Codeの立ち位置: CLI特化 × Anthropic製 × 開発者向け
3. 他のAIツールとの違い: Cursor, GitHub Copilot CLI, Aider等との比較
4. 本講座のゴール: 3つの学習到達点のプレビュー（実際の画面キャプチャ）

### 第1章 プロンプト基礎 — AIに確実に伝える技術 (L3: Apply)

**所要**: 30 min / Bloom: L3 Apply
**参考**: 中級編 第1章の圧縮版（完全な代替ではない）

#### ストーリー (Emotional Hook)
> 「『AIに伝わればいい』と思っていませんか？ 実は、伝わるプロンプトと**再利用できる**プロンプトは別物です。この章では、1回書いて終わる指示を、何度でも使える形に変えます。」

- 受講者の「これだ！」ポイント: 自分の適当な指示がXMLタグで構造化される瞬間。同じタスクを2回目に実行した時の違い（再現性）をスライド比較で解説。

#### 解説アウトライン
1. プロンプトの3層構造: 入力(`<input>`) / 処理(`<task>`) / 出力(`<output_format>`)
2. XMLテンプレートの解説: 各タグの役割と書き方
3. Before/After比較: 非構造化プロンプト vs 構造化プロンプトの出力差（画面キャプチャ）
4. 再現性の確認: 同じプロンプトを2回実行した結果の安定性を解説
5. よくある失敗パターン3選: 曖昧な指示 / 出力指定なし / コンテキスト不足

#### 学習目標
- XMLタグを使った構造化プロンプトの書き方を理解する
- 入力・処理・出力の3パートに指示を分解できる
- 同じプロンプトで再現性のある結果を得る仕組みを理解する

### 第2章 CLAUDE.md ミニマム — プロジェクトの「説明書」 (L3: Apply)

**所要**: 30 min / Bloom: L3 Apply
**参考**: 中級編 第3章の圧縮版（完全な代替ではない）

#### ストーリー (Emotional Hook)
> 「新しいプロジェクトに入った時、最初にREADME.mdを読みますよね？CLAUDE.mdは、AIに読ませるREADMEです。30行書くだけで、AIがプロジェクトのルールを理解します。」

- 受講者の「これだ！」ポイント: CLAUDE.mdなしで「テストを書いて」と頼んだ時と、ありで頼んだ時の結果の違いをスライドで対比。コーディング規約や使用ライブラリをAIが勝手に守る仕組みを解説。

#### 解説アウトライン
1. CLAUDE.mdとは: AIに読ませるプロジェクトREADME
2. 5セクション構成の解説: 使用言語/フレームワーク / コーディング規約 / よく使うコマンド / 禁止事項
3. 30行ミニマム版の例示: 実際のサンプルコードをスライドで解説
4. 効果の測定: あり/なしでAIの振る舞いが変わる具体的な例（画面キャプチャ3枚）
5. よくある書き方の失敗: 長すぎる / 曖昧 / 矛盾する指示

#### 学習目標
- CLAUDE.mdの役割（AIへのプロジェクト説明書）を理解する
- 最低限の構成（5セクション・30行）でCLAUDE.mdを書ける
- あり/なしでAIの振る舞いが変わる理由を説明できる

### 第3章 簡単な自動化体験 — AIに任せてみる (L3: Apply + L4: Analyze)

**所要**: 30 min / Bloom: L3-L4

#### ストーリー (Emotional Hook)
> 「これまではAIに『質問』していました。この章からはAIに『作業』を頼みます。1つのファイル編集タスクで、AIに自動化させる仕組みを解説します。」

- 受講者の「これだ！」ポイント: 「このファイルのバグを修正して」と頼むだけで、AIが自らファイルを読み→原因特定→修正→テスト実行まで完走する様子をスライドで時系列解説。

#### 解説アウトライン
1. AIの自律作業フロー図解: Read → 分析 → Edit → Test → 完了 の流れ
2. バグ修正の実例解説: 意図的なバグ3個入りプロジェクトでの修正プロセス（画面キャプチャ6枚）
3. 各ステップの解説: AIがどうやって原因を特定し、どう修正するか
4. 人間のレビューが必須な理由: AIの修正が常に正しいとは限らない事例3選
5. CLAUDE.mdとの連携: 設定ファイルが自動化の精度にどう影響するか

#### 学習目標
- AIにファイル編集タスクを丸投げする際のフローを理解する
- AIの自律的な作業フロー（Read→Edit→Test）の仕組みを理解する
- 人間のレビューが必須である理由を説明できる

### 第4章 hookの入門 — AIの安全装置を作る (L3: Apply)

**所要**: 30 min / Bloom: L3 Apply
**参考**: 1本だけ解説。18本の全網羅は中級編で。

#### ストーリー (Emotional Hook)
> 「AIに『このファイルを消して』と言われたら、どうしますか？ hookは、AIが危険な操作をする前に**立ち止まらせる**安全装置です。たった1本で、あなたのコードを守れます。」

- 受講者の「これだ！」ポイント: hookなしで `rm -rf` を含むコマンドが実行される恐怖 → hookありで「STOP: 危険な操作を検知しました」とブロックされる安心感。スライドでこの差を1発で解説。

#### 解説アウトライン
1. hookとは: AIのライフサイクルイベントに反応する仕組み
2. 7つのイベント種類: SessionStart / PreToolUse / PostToolUse / Stop 等
3. PreToolUse hookの仕組み解説: コマンド実行前に検査 → 危険パターンをブロック
4. 設定ファイルの構造: `.claude/settings.json` の hook エントリ解説
5. ブロック例の解説: `rm -rf` / `sudo` / `DROP TABLE` 等の検知パターン
6. hookの拡張性: より高度な18本のhookは中級編で解説

#### 学習目標
- hookとは何か（AIのライフサイクルイベントに反応する仕組み）を理解する
- PreToolUse hook の仕組みと設定方法を理解する
- AIの危険操作を自動検知・ブロックする仕組みを説明できる

---

### 第5章 MCP 接続入門 — AIの記憶を拡張する (L3: Apply)

**所要**: 30 min / Bloom: L3 Apply
**参考**: 「使う側」だけの解説。自前構築は中級編で。

#### ストーリー (Emotional Hook)
> 「AIとの会話を終えるたびに、記憶がリセットされていたらどうでしょう？毎回同じ説明を繰り返すのは非効率です。MCPでAIに『記憶』を与えましょう。」

- 受講者の「これだ！」ポイント: MCPなし → セッションを跨ぐと忘れる。MCPあり → 前回の会話内容を引き継ぐ。この違いを2つの画面キャプチャで対比解説。

#### 解説アウトライン
1. MCP（Model Context Protocol）とは: AIにプラグインを追加する仕組み
2. memory MCP の役割: セッションを跨いで記憶を保持する仕組み
3. 設定手順の解説: `claude mcp add memory` → 設定ファイルへの追加内容
4. Before/After比較: MCPなし/ありでのセッション間の振る舞い差（画面キャプチャ）
5. 利用可能なMCPサーバーの紹介: 公式・サードパーティの代表的なもの

#### 学習目標
- MCP（Model Context Protocol）の役割を理解する
- 既存のMCPサーバーをClaude Codeに接続する手順を理解する
- AIの記憶がセッションを跨いで引き継がれる仕組みを説明できる
