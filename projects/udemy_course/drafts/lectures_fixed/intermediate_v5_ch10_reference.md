## スラッシュコマンドの 4 階層 — 区別の前提

> 本章で頻出する `/command` 系の用語は **4 階層に分かれる**。「公式」と「本リポジトリ独自」を混同しないこと

| 階層 | 例 | 出所 |
|------|----|------|
| **機構** (built-in mechanism) | スラッシュコマンドという仕組み自体 | Claude Code 本体機能 |
| **組込コマンド** | `/clear` `/compact` `/init` `/memory` `/agents` `/model` 等 | 公式コマンド表に掲載 |
| **バンドルスキル** | `/review` `/security-review` `/simplify` 等 | Anthropic 公式同梱 |
| **独自カスタム** | `/handoff` `/advisor` `/senkyou` 等 | **本リポジトリ独自実装** (各自で作成) |

> **作成方式**: Skills (`.claude/skills/<name>/SKILL.md`) が推奨・legacy `.claude/commands/<name>.md` 単一ファイルも併存。公式は「カスタムコマンドはスキルにマージ」と明記

<!--
本章では何度も /handoff や /advisor が出てきます。これらは「本リポジトリ独自のカスタムスラッシュコマンド」で、Claude Code 標準にはありません。自分の現場で類似機能を使うには、自前で Skills (`.claude/skills/<name>/SKILL.md`) を作る必要があります。一方、advisor「ツール」(advisor_20260301) や PostToolUse hook は Anthropic 公式機能です。コマンド名とツール名を区別する目線が大事です。
-->

## 3段早見表 (2/3) — コンテキスト層の困りごと

| 症状 | 現象 | 対策 | 章 |
|------|------|------|----|
| 長文の中央にある重要情報が無視された | 長文の中央が読み飛ばされる現象 (Lost in the Middle: 長い文章の真ん中にある情報が抜け落ちる現象) | TL;DR + 重要情報を冒頭/末尾に配置 | ch03 |
| 先週の修正内容をAIが知らない | 過去の記憶が引き継がれていない | **自動メモリ** (公式 `MEMORY.md` + トピックファイル・`~/.claude/projects/<project>/memory/` に格納) + 引き継ぎファイル | ch04 |

|------|------|------|----|

# 3層インデックス — 「どの層が足りないか」から引く

> 症状から引くだけでも便利ですが、**「どの層が欠けているか」** を特定できれば、もっと早く解決できます。

11の困りごとはすべて、**3つの層のいずれかが足りない** から起きています。

<!--
もう一つの逆引き方法です。「症状」から探すのではなく、「どの層が足りないか」から探します。ハーネスは Anthropic 公式概念で、業界標準では Agent = Model + Harness と定義されます。
-->

---

## 3層別分類 — 全11件を層で整理

| 不足している層 | 該当する困りごと | 対策章 |
|--------------|-----------------|-------|
| **プロンプト層不足** | ①既存を無視 ②手動繰返 ③推測で突進 | ch01 / ch02 |
| **コンテキスト層不足** | ④中央が読まれない ⑤完了を忘れる | ch03 / ch04 |
| **ハーネス層不足** (Anthropic 公式概念) | ⑥言ったのに違う ⑦盲信 ⑧仕様変更で壊れる ⑩失敗を隠す ⑪止まらない | ch05 / ch06 / ch07 / ch08 |
| **複合 (要組み合わせ)** | ⑨資料未確認 (コンテキスト + ハーネスの両方) | ch04 + ch07 |

<!--
3つの層で11件を整理しました。ハーネス層が5件と最多です。「何かが変だ」と感じたら、まず「ハーネス層が足りないのでは?」と疑うのがコツです。ハーネスは Anthropic Engineering Blog「Effective harnesses for long-running agents」(2025-11) で定義された公式概念で、hook・advisor・watcher・禁止操作リスト等の実装パーツの総称です。
-->

## 見分け方①: 「推測で突進」と「盲信して失敗」

| | ③ 推測で突進 (AIが確認せず勝手に進める) | ⑦ 盲信して失敗 (AI出力を信じ込む) |
|--|----------------------|----------------------|
| **いつ起きる?** | AIが**出力する前** | AIの**出力を見た後** |

> **判断のコツ**: AIが**出力する前**に起きるなら③、**出力を見た後**に気づくなら⑦

## 見分け方③: 「言ったのに違う」と「止まらない」

| | ⑥ 言ったのに違う (指示を部分的に無視) | ⑪ 止まらない (止めても実行が続く) |
