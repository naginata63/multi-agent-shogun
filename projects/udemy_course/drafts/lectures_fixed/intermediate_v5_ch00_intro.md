## 初級編からの橋渡し — 3 階層モデル復習

> 初級編で身につけた **3 つの力** を、ここで一度だけ俯瞰します

| 階層 | 役割 | 初級編で学んだこと |
|------|------|------------------|
| **L1 プロンプト** | AI に確実に伝える | 構造化プロンプト (入力・処理・出力の 3 要素) |
| **L2 コンテキスト** | プロジェクト知識の共有 | CLAUDE.md / commands / skills (.md の 3 大カテゴリ) |
| **L3 ハーネス** | AI モデルを実用基盤に乗せる | PreToolUse Hook 1 本 (公式 29 種類のうち代表) |

> Anthropic 公式の業界標準定義: **Agent = Model + Harness**
> (Engineering Blog 2025-11「Effective harnesses for long-running agents」)
> hooks / skills / sub-agents 等で AI を実用レベルで動かす周辺基盤全体を「ハーネス」と呼びます

中級編では、この 3 層を **実戦レベルまで深掘り** していきます。

<!-- Speaker note: 中級編に入る前に、初級編で学んだ 3 階層モデルを軽く復習します。初級編を未受講の方も、この 1 スライドで全体感が掴めるよう作っています。L1 プロンプトは「指示のブレをなくす構造化プロンプト」、L2 コンテキストは「CLAUDE.md / commands / skills の 3 大カテゴリでプロジェクト知識を共有」、L3 ハーネスは「PreToolUse Hook 1 本で AI の動作を制御」 — これが初級編の到達点でした。ハーネスについては Anthropic 公式の業界標準定義「Agent = Model + Harness」が 2025 年 11 月の Engineering Blog 記事「Effective harnesses for long-running agents」で明示されており、本講座でも公式概念として扱います。中級編では、この 3 層をそれぞれ実戦レベルまで深掘りしていきます。 -->

## あなたに聞きます

こんな経験、ありませんか？

- AI に**同じ説明を 5 回**繰り返した
- 「完了しました」と言われたが、**後で見たら嘘**だった
- 先週直したバグの調査を、**また最初から**やり直している
- 長い資料を渡したのに、**真ん中の重要な数字**を見落としていた

<!-- Speaker note: 受講者に「ありますよね？」と問いかけます。ここで共感を作るのがこの章の最大の目的です。4つの例のうち、どれか一つでも当てはまる人は手を挙げてもらうと参加度が上がります。 -->

## 「あるある」を 11 件集めました

| # | 困りごと | よくある光景 |
|---|---------|------------|
| ① | **既にあるものを確認せず新規作成** | 先週書いたプロンプトを探すのに 15 分 |
| ② | **同じ作業を毎回手動で繰り返す** | 議事録要約のプロンプト、毎回ゼロから書く |
| ③ | AI が**推測で勝手に進める** | 「理解しました」→ 結果が全然違う |
| ④ | **長文の中央**が読み飛ばされる | 30 ページの議事録、中盤の決定事項が抜ける |
| ⑤ | **完了したことを**次に伝え忘れる | 先週直したこと、今週は忘れてる |
| ⑥ | **「言ったのにやってくれない」** | 指示したはずが反映されていない |

| # | 困りごと | よくある光景 |

「AI が悪い」「自分がダメ」ではありません。

本講座で、この 3 層を順に強化していきます。

## 3 層の世界観

```
┌─────────────────────────────────────────┐
│  プロンプト層 (Prompt)                   │
│  「AI への 1 回の指示の質」               │
│  ── 定型プロンプトの保存・診断             │
├─────────────────────────────────────────┤
│  コンテキスト層 (Context)                │
│  「AI に何を覚えさせ、何を見せるか」       │
│  ── 必要な情報を必要な時だけ取り出す       │
├─────────────────────────────────────────┤
│  ハーネス層 (Harness)                    │
│  「AI を実用レベルで動かす周辺基盤」       │
│  ── hooks / skills / sub-agents / 自動化   │
└─────────────────────────────────────────┘
```

> ハーネスは Anthropic 公式の業界標準概念 (Agent = Model + Harness)

<!-- Speaker note: 3 層の図を板書するイメージで説明します。プロンプト層は「1 回の指示の質」＝目に見えやすい。コンテキスト層は「AI に何を見せるか」＝設計が必要。ハーネス層は「AI を実用レベルで動かす周辺基盤」＝Anthropic 公式の「Agent = Model + Harness」に対応する層で、hooks・skills・sub-agents・自動化全般を含みます。最も設計難度が高いですが効果も最大。上から順に強化していきます。 -->

---

## 3 層のイメージ

- **プロンプト層** = 「1 回の指示の質」
  - 例：「議事録を要約して」を資産化する仕組み
- **コンテキスト層** = 「AI に何を見せるか」
  - 例：長文を全部渡すのではなく、必要な部分だけ取り出す
- **ハーネス層** = 「AI を実用レベルで動かす周辺基盤」 (Anthropic 公式: Agent = Model + Harness)
  - 例：hooks でルールを書いておけば、AI が自動で守る

<!-- Speaker note: 各層を日常的な例えで説明します。プロンプト層＝「お願いの仕方メモ」。コンテキスト層＝「必要な資料だけ机に置く」。ハーネス層＝「手順書を壁に貼っておく + 担当を分けて + レビュー役を置く」。ハーネスは安全装置だけでなく、hook・skill・サブエージェント等、AI モデルを実用レベルで動かす周辺基盤全体を指します。具体例でイメージを掴ませてください。 -->

## 集計すると

- **プロンプト層** が欠けている → 3 件 (①②③)
- **コンテキスト層** が欠けている → 2 件 (④⑤)
- **ハーネス層** が欠けている → 5 件 (⑥⑦⑧⑩⑪)
- **複合** → 1 件 (⑨)

> ハーネス層の欠如が一番多い。
> つまり「AI を実用レベルで動かす周辺基盤」が一番のボトルネック。

<!-- Speaker note: 集計結果のポイントは「ハーネス層が 5 件で最多」です。多くの人はプロンプトの書き方ばかり気にしますが、実は「AI を実用レベルで動かす周辺基盤」(ハーネス) の欠如が最も多い困りごとの原因。この発見に少し驚く受講者もいるはずです。 -->

## 講座が終わると、あなたのリポジトリにこれが残る

| 道具 | 役割 | いつ使うか |
|------|------|-----------|
| カスタムスラッシュコマンド (Skills / `.claude/commands/<名前>.md`) | 毎回書いているプロンプトを 1 コマンド化 | 毎回同じプロンプトを書く時 |
| `memory` (過去の知識の引き出し) | AI が過去の合意事項を思い出す | 「前に合意したこと」を思い出す時 |
| `hook` (ライフサイクルイベントへの自動処理) | 「このファイルは必ず確認」などを自動化 | 忘れがちな確認作業を自動化する時 |
| `Skill` (再利用可能なワークフロー定義・必要時にオンデマンド読込) | 特定の業務フロー全体を仕組み化 | 一連の作業を AI に任せる時 |

> これらが **あなたのプロジェクトフォルダ** に残ります。
> 講座が終わっても、そのまま業務で使い続けられます。

<!-- Speaker note: 講座の実用性を強調します。「座学で終わらない」ことが受講者のモチベーションに直結します。各道具は実際にファイルとして残るので、明日から業務で使えることを伝えてください。なお、Claude Code 公式には「/command」という単一コマンドは存在しません。本講座でいう「自作のスラッシュコマンド」は、推奨される Skills (`.claude/skills/<名前>/SKILL.md` のディレクトリ形式) か、legacy の `.claude/commands/<名前>.md` (単一 .md) として保存します。詳細は ch01 で扱います。Skill は「常駐」ではなく、description が常駐して本体は必要時にオンデマンド読込される設計です — この点も ch11 で実装演習します。 -->

## この講座の学び方

1. **動画を止める** — コードが出たら実際に手を動かす
2. **自分で試す** — サンプルをそのままではなく、自分の業務に置き換えて実行
3. **章末で確認** — 各章の「持ち帰り」で理解度をセルフチェック

> 「見るだけ」ではなく「やる」講座です。

<!-- Speaker note: 学び方の3つのポイント。「動画を止める」「自分で試す」「章末で確認」は本講座の基本スタンスです。特に「自分の業務に置き換えて試す」ことを強調。サンプルをコピペするだけでなく、自分の現場に当てはめることで理解度が跳ね上がります。 -->
