## 0.1 ペルソナ定義 (5タイプ)

本講座の主ターゲット層を5つのペルソナで定義する。

### P1: 中堅バックエンドエンジニア「田中」
- **年代**: 32歳
- **職種**: Web系バックエンドエンジニア (Ruby/Python)
- **業界**: SaaSスタートアップ (従業員30名)
- **年収**: 650万円
- **具体困りごと**: 自社プロダクトにClaude Codeを導入したが、プロンプトが長すぎてコンテキスト爆発を起こす。何度も同じ失敗をAIが繰り返す。社内の他エンジニアとAI利用ナレッジを共有する仕組みがない。
- **購買動機**: 「AI開発の生産性を上げたい」ではなく「AIの暴走を制御する仕組みを作りたい」。プロンプトの書き方ではなく、システムとしてAIを組み込みたい。

### P2: フリーランスAI活用コンサルタント「佐藤」
- **年代**: 28歳
- **職種**: フリーランス / AI導入支援
- **業界**: 中小企業DX支援
- **年収**: 480万円 (不安定)
- **具体困りごと**: クライアント先でClaude Codeを導入する案件が増えているが、導入後の「運用設計」ができず、1ヶ月で使われなくなる。差別化できるスキルセットが欲しい。
- **購買動機**: 「ハーネスエンジニアリング」という固有スキルで競合コンサルと差別化。クライアントに「導入後も自分で改善できる仕組み」を納品したい。

### P3: 情報発信者・テックブロガー「林」
- **年代**: 35歳
- **職種**: テック系YouTuber / Zenn Qiita同人
- **業界**: 個人メディア運営
- **年収**: 420万円 (広告+note+YouTube)
- **具体困りごと**: Claude Codeで動画パイプラインを自動化しているが、セッションを跨ぐたびに設定をやり直している。毎回同じようなプロンプトを書いている時間が無駄。
- **購買動機**: コンテキストの永続化で「設定ゼロの作業再開」を実現したい。自身の情報発信ネタとしても「3階層統合」は独自性が高い。

### P4: SIerのプロジェクトマネージャー「木村」
- **年代**: 40歳
- **職種**: プロジェクトマネージャー (元SE)
- **業界**: 中堅SIer (従業員500名)
- **年収**: 800万円
- **具体困りごと**: 顧客からのAI開発案件が急増しているが、社内の若手エンジニアが「プロンプトを頑張る」止まりで、本番運用で事故る。AI開発の品質管理フレームワークが欲しい。
- **購買動機**: AI開発品質を「個人のプロンプトスキル」から「組織のシステム」に引き上げたい。done_gate や silent_fail_watcher のような品質ハーネスを社内標準にしたい。

### P5': SI企業のリードエンジニア「鈴木」
- **年代**: 34歳
- **職種**: リードエンジニア (フルスタック)
- **業界**: 中堅SI企業 (従業員200名)
- **年収**: 700万円
- **具体困りごと**: 自社プロジェクトでClaude Codeを導入したが、エンジニアごとに使い方がバラバラで成果物の品質にばらつきが出ている。標準化された開発フレームワークが欲しい。
- **購買動機**: 3階層エンジニアリングのフレームワークを社内標準として展開し、AI開発の品質と再現性を組織レベルで担保したい。hookによる自動品質管理は社内レビューの負荷を下げる。

## 0.3 LP（ランディングページ）コピーライティング案

### ヒーローセクション
```
AI開発で、こんな経験ありませんか？

☑ 同じプロンプトを何度も書き直している
☑ AIが急に変な返事をして、セッションを作り直す
☑ 「AIで効率化した」と言ったのに、結局手直しの時間が増えた
☑ AIに任せたタスクが本当に完了したか、確認する手段がない

その原因は「プロンプト」ではありません。
3つの階層（プロンプト / コンテキスト / ハーネス）を
統合設計することで、AI開発は予測可能で再現性のあるものになります。

本講座は、18本の実コードと共に教える
「AI開発のシステム化」実践ガイドです。
```

### 差別性ブロック
```
本講座だけにある5つの要素:

1. 「3階層統合」は市場唯一（2026年4月・Udemy日本語圏60コース調査で確認）
2. 18本のhook/watcherスクリプトを実コードで解説（概論のみの講座とは一線を画す）
3. 実運用での失敗事例15件以上を教材化（成功例だけの講座では学べない）
4. 受講完了時、自分のプロジェクトに3層システムが動く状態で導入完了
5. flagship case study: 3階層統合hookを60分で一行ずつ解剖
```

### 購買心理フック (AIDA フレーム)
- **Attention**: 「AI開発の3階層」という聞き慣れない概念で好奇心を刺激
- **Interest**: 無料プレビュー序章で「before/after」の開発体験ギャップを提示
- **Desire**: 3層システム稼働の画面キャプチャ + 成果物ツアーで「これが作れる」と実感
- **Action**: 価格1,980円（セール時980円）の低ハードル + 30日返金保証
