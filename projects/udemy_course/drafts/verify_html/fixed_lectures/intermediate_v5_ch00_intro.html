初級編からの橋渡し — 3 階層モデル復習

初級編で身につけた 3 つの力 を、ここで一度だけ俯瞰します

階層 役割 初級編で学んだこと
L1 プロンプト AI に確実に伝える 構造化プロンプト (入力・処理・出力の 3 要素)
L2 コンテキスト プロジェクト知識の共有 CLAUDE.md / commands / skills (.md の 3 大カテゴリ)
L3 ハーネス AI モデルを実用基盤に乗せる PreToolUse Hook 1 本 (公式 29 種類のうち代表)

Anthropic 公式の業界標準定義: Agent = Model + Harness
(Engineering Blog 2025-11「Effective harnesses for long-running agents」)
hooks / skills / sub-agents 等で AI を実用レベルで動かす周辺基盤全体を「ハーネス」と呼びます

中級編では、この 3 層を 実戦レベルまで深掘り していきます。

あなたに聞きます

こんな経験、ありませんか？

  • AI に同じ説明を 5 回繰り返した
  • 「完了しました」と言われたが、後で見たら嘘だった
  • 先週直したバグの調査を、また最初からやり直している
  • 長い資料を渡したのに、真ん中の重要な数字を見落としていた

「あるある」を 11 件集めました

# 困りごと よくある光景
① 既にあるものを確認せず新規作成 先週書いたプロンプトを探すのに 15 分
② 同じ作業を毎回手動で繰り返す 議事録要約のプロンプト、毎回ゼロから書く
③ AI が推測で勝手に進める 「理解しました」→ 結果が全然違う
④ 長文の中央が読み飛ばされる 30 ページの議事録、中盤の決定事項が抜ける
⑤ 完了したことを次に伝え忘れる 先週直したこと、今週は忘れてる
⑥ 「言ったのにやってくれない」 指示したはずが反映されていない

「あるある」を 11 件集めました (続き)

# 困りごと よくある光景
⑦ AI 出力を盲信して失敗 「完璧です」の言葉を信じて提出 → 数字が間違ってた
⑧ API 変更後にテストせず壊れる 昨日動いたコードが今日は動かない
⑨ 大事なドキュメントを確認しない 手元の仕様書があるのに参照せず進める
⑩ 失敗を隠して動かす エラーが出たのに「たぶん大丈夫」で進める
⑪ 「止めろ」が伝わらない 「そこはやらないで」と言ったのに実行された

全 11 件、心当たりありませんか？

ここで宣言します

これらの困りごとは、すべて「3 層のどこかが欠けている」から起こります

「AI が悪い」「自分がダメ」ではありません。
仕組みの欠落 が原因です。

本講座で、この 3 層を順に強化していきます。

3 層の世界観

┌─────────────────────────────────────────┐
│  プロンプト層 (Prompt)                   │
│  「AI への 1 回の指示の質」               │
│  ── 定型プロンプトの保存・診断             │
├─────────────────────────────────────────┤
│  コンテキスト層 (Context)                │
│  「AI に何を覚えさせ、何を見せるか」       │
│  ── 必要な情報を必要な時だけ取り出す       │
├─────────────────────────────────────────┤
│  ハーネス層 (Harness)                    │
│  「AI を実用レベルで動かす周辺基盤」       │
│  ── hooks / skills / sub-agents / 自動化   │
└─────────────────────────────────────────┘

ハーネスは Anthropic 公式の業界標準概念 (Agent = Model + Harness)

3 層のイメージ

  • プロンプト層 = 「1 回の指示の質」
  • 例：「議事録を要約して」を資産化する仕組み
  • コンテキスト層 = 「AI に何を見せるか」
  • 例：長文を全部渡すのではなく、必要な部分だけ取り出す
  • ハーネス層 = 「AI を実用レベルで動かす周辺基盤」 (Anthropic 公式: Agent = Model + Harness)
  • 例：hooks でルールを書いておけば、AI が自動で守る

11 困りごと → どの層が欠けているか

# 困りごと 欠けている層 解決する章
① 確認せず新規作成 プロンプト層 ch01
② 同じ作業を毎回手動 プロンプト層 ch01
③ 推測で勝手に進める プロンプト層 ch02
④ 長文の中央が読み飛ばされる コンテキスト層 ch03+04
⑤ 完了したことを忘れる コンテキスト層 ch04
⑥ 「言ったのにやってくれない」 ハーネス層 ch05
⑦ AI 出力を盲信 ハーネス層 ch06
⑧ API 変更後に壊れる ハーネス層 ch07
⑨ 大事な情報を確認しない コンテキスト+ハーネス ch07
⑩ 失敗を隠す ハーネス層 ch08
⑪ 「止めろ」が伝わらない ハーネス層 ch08

集計すると

  • プロンプト層 が欠けている → 3 件 (①②③)
  • コンテキスト層 が欠けている → 2 件 (④⑤)
  • ハーネス層 が欠けている → 5 件 (⑥⑦⑧⑩⑪)
  • 複合 → 1 件 (⑨)

ハーネス層の欠如が一番多い。
つまり「AI を実用レベルで動かす周辺基盤」が一番のボトルネック。
