「AIに開発を任せたい」

— でも、どうやって？

初級編 — 序章 (約 15 min)
FREE PREVIEW

講師: なぎなた
**この章で出てくる用語** - **プロンプト** — AIに出す指示文のこと - **コンテキスト** — AIに共有する前提知識（メモ）のこと - **ハーネス** — Claude本体に周辺の仕組み（hook等）を組み合わせて、信頼できる動きにする層（Anthropic公式の概念） - **hook** — AIの動作の節目（プロンプト送信時/ツール実行前後/セッション開始時など）に任意の処理を差し込む仕組み。安全装置はその代表用途の一つ - **Claude Code** — この講座で使うAI開発ツール（第1章以降で詳しく説明します） - **ターミナル** — パソコンに入っている黒い画面のアプリ - **レイヤー** — 技術を積み上げる一段一段のこと（レンガを積むようなイメージ）

こんな経験、ありませんか？

AIを使っているはずなのに、思った通りに動いてくれない

  • ChatGPTにコードを書かせたけど、意図と違うものが返ってきた
  • AIの使い方は分かったけど、開発の流れに組み込めていない
  • 「もっと効率よくAIを使えるはず」— そう感じている

壁① — AIに意図が伝わらない

「もっとシンプルにして」→ AIは別の解釈をする

  • 同じ指示を出しているのに、毎回違うコードが返ってくる
  • 出力が長すぎたり、的外れだったり
  • 「日本語で言えば分かるはず」— でも伝わらない

壁② — AIがプロジェクトを知らない

会話のたびに前提条件を説明し直す…非効率すぎる

  • プロジェクトのルールや構造をAIに説明しきれていない
  • 会話のたびに「このプロジェクトは…」と毎回ゼロから説明
  • AIが文脈を理解しないままコードを書くので不整合が起きる

壁③ — AIの動きをコントロールできない

AIが勝手にファイルを変更したら…怖くないですか？

  • AIに自動化を任せたいけど、怖い
  • テストなしでコードを書くのは不安
  • 「人間の承認」を挟む仕組みが分からない

実は、壁は3つだけ

この3つの壁を乗り越えれば、AIが開発パートナーになります

  • AI開発の悩みはどれだけ複雑に見えても、分解すると「伝える」「共有する」「制御する」のどれかに分類できる — 他の悩みも試しに当てはめてみてください
  • たとえば「AIが古いライブラリを使う」→ 前提を共有していない (壁②)。「余計なファイルまで変える」→ 制御できていない (壁③)
  • 3つの壁にそれぞれ解決策を1つずつ対応させれば、すべての悩みに対応できます

壁① → 伝える技術
壁② → 共有する仕組み
壁③ → 動作を制御する層

この講座は、この3つを順番に学ぶカリキュラムです

3つの鍵

L = Layer（層）。レンガを下から積み上げるように、3つの技術を順に身につけます

鍵 できるようになること 対応する章
L1: プロンプト (= 指示文) AIに意図が伝わる 第1章
L2: コンテキスト (= 前提メモ) プロジェクト知識をAIに共有 第2章〜第4章
L3: ハーネス (= AIの動作を制御・拡張する層 / Anthropic公式概念) hookなどでAIの動きを安全に方向付ける 第5章

L1 → L2 → L3 の順に段階的に身につけます
L2 が3章分あるのは、プロジェクトの前提を共有する方法が一番ボリュームが多いからです

Claude Code — この講座で使うツール

L1/L2/L3をそのまま実現する、ターミナルで動くAI開発ツール

  • ターミナル（黒い画面のアプリ）上でAIがファイルを読み、コードを書き、テストを実行する
  • ブラウザのチャットとは違う — プロジェクト全体を操作できる（あなたのパソコンにあるコードファイルを直接書き換えてくれます）
  • この講座のすべての内容は Claude Codeで実践します
  • 本講座は Claude Code インストール済みの環境を前提とします (インストール手順は公式ドキュメント参照)

この講座を終えると

AIと一緒に開発するのが 当たり前 になります

  • L1 — AIに指示が伝わる → やり直しが減る
  • L2 — 前提を共有できる → 毎回説明不要
  • L3 — ハーネスで動きを方向付けられる → AIを信頼できる

3.5時間で、AI開発の基礎をすべて身につけられます
