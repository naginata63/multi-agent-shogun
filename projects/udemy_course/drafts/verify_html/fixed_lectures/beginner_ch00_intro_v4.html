「AIに開発を任せたい」

— でも、どうやって？

序章 (約 15 min)
FREE PREVIEW

**この章で出てくる用語（※ ここは全部覚えなくてOKです。この後のスライドで一つずつ説明します）** - **プロンプト** — AIに出す指示文のこと - **コンテキスト** — AIに共有する前提知識（メモ）のこと - **Claude Code** — この講座で使うAI開発ツール（ターミナル上で動く・第1章以降で詳しく説明します） - **ターミナル** — パソコンに入っている黒い画面のアプリ - **ツール実行** — AIがファイルを読み書きしたり、コマンドを動かしたりすること - **hook** — AIの動作の節目（セッション開始時/プロンプト送信時/ツール実行前後 など）に任意の処理を差し込む仕組み。安全装置はその代表用途の一つ - **ハーネス** — Claude本体の周りに hook などの仕組みを組み合わせて、AIの動きを信頼できる形に整える「層」（Anthropic公式の概念） - **レイヤー** — 技術を積み上げる一段一段のこと（レンガを積むようなイメージ）

こんな経験、ありませんか？

AIを使っているはずなのに、思った通りに動いてくれない

  • ChatGPTやClaudeにコードを書かせたら、意図と違うものが返ってきて、結局自分で書き直した
  • 同じ前提を何度もAIに説明し直すうちに、「自分で書いた方が早いのでは」と思った
  • 「もっと効率よくAIを使えるはず」— そう感じているのに、何から手をつければいいか見えてこない

壁① — AIに意図が伝わらない

「もっとシンプルにして」→ AIは別の解釈をする

  • 同じ指示を出しているのに、毎回違うコードが返ってくる
  • 出力が長すぎたり、的外れだったり
  • 「日本語で言えば分かるはず」— でも伝わらない

壁② — AIがプロジェクトを知らない

会話のたびに前提条件を説明し直す…非効率すぎる

  • プロジェクトのルールや構造をAIに説明しきれていない
  • 会話のたびに「このプロジェクトは…」と毎回ゼロから説明
  • AIが文脈を理解しないままコードを書くので不整合が起きる

壁③ — AIの動きをコントロールできない

AIが勝手にファイルを変更したら…怖くないですか？

  • AIに自動化を任せたいけど、怖い
  • テストなしでコードを書くのは不安
  • 「人間の承認」を挟む仕組みが分からない

実は、壁は3つに集約できる

3つの壁に対策を対応させると、多くの悩みが整理しやすくなります

  • AI開発の悩みは複雑に見えても、多くは「伝える」「共有する」「制御する」のどれかに分類しやすい — 他の悩みも試しに当てはめてみてください
  • たとえば「AIが古いライブラリを使う」→ 前提を共有していない (壁②)。「余計なファイルまで変える」→ 制御できていない (壁③)
  • 3つの壁それぞれに対策を1つずつ対応させていくのが、本講座の基本方針です

壁① → 伝える技術
壁② → 共有する仕組み
壁③ → 動作を制御する層

この講座は、この3つを順番に学ぶカリキュラムです

3つの鍵

L = Layer（層）。レンガを下から積み上げるように、3つの技術を順に身につけます

鍵 できるようになること 対応する章
L1: プロンプト (= 指示文) AIに意図が伝わる 第1章
L2: コンテキスト (= 前提メモ) プロジェクト知識をAIに共有 第2章〜第4章
L3: ハーネス (= AIの動作を制御・拡張する層 / Anthropic公式概念) hookなどでAIの動きを安全に方向付ける 第5章

L1 → L2 → L3 の順に段階的に身につけます
L2が3章分ある理由：①AIに渡す設定ファイル本体 (CLAUDE.md) ②そこから派生するAI用ドキュメント群 ③なぜそうした仕組みが必要かの歴史 — 共有の仕組みは扱う要素が一番多いからです

Claude Code — この講座で使うツール

L1/L2/L3をそのまま実現する、ターミナルで動くAI開発ツール

  • ターミナル（黒い画面のアプリ）上でAIがファイルを読み、コードを書き、テストを実行する
  • ブラウザのチャットとは違う — プロジェクト全体を操作できる（あなたのパソコンにあるコードファイルを直接書き換えてくれます）
  • この講座のすべての内容は Claude Codeで実践します
  • 本講座は Claude Code インストール済みの環境を前提とします（インストール手順は公式ドキュメント https://docs.claude.com/ja/docs/claude-code/overview を参照）

この講座を終えると

AIと一緒に開発する流れが、自分の中で自然な選択肢になります

  • L1 — AIに指示が伝わりやすくなる → やり直しが減る
  • L2 — 前提を共有する仕組みを持てる → 毎回ゼロから説明しなくて済む
  • L3 — ハーネスで動きを方向付けられる → AIに任せる範囲を自分で決められる

約 3.5 時間で、AI開発の基礎をひと通り学べる構成です
