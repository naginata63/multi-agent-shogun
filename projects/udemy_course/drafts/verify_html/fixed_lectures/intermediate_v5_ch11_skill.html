カスタムスラッシュコマンドから Skill へ — なぜもう一段上が必要か

具体例: 「毎週月曜に議事録を要約して日報を作る」業務

やり方 手順 問題点
毎回ゼロから書く プロンプトを毎週新規作成 非効率・品質バラつき
カスタムスラッシュコマンド 化 (Skills/legacy commands) 日報プロンプトを /weekly-report 等で資産化 まだ自分で呼び出す必要がある
Skill (自律発動) 化 ★ 業務知識を Skill 本体に書き込み・description で発動条件を定義 AI が文脈を読み取って自律的に呼び出す

手動呼出: 人が /<コマンド> で意図的に呼び出す
自律発動: description のキーワードマッチで AI が自律的に Skill 本体を読み込む

Skill の構造 — ディレクトリ + エントリポイント

.claude/skills/
└── weekly-report/        ← ディレクトリ名がコマンド名 /weekly-report になる
    └── SKILL.md          ← エントリポイント (公式標準)

公式標準: .claude/skills/<skill-name>/SKILL.md (ディレクトリ + SKILL.md エントリポイント)
personal: ~/.claude/skills/<skill-name>/SKILL.md
legacy 互換: .claude/commands/<name>.md (単一 .md 単独動作・サポートファイル機能なし)

Skill ファイルの中身 — frontmatter + 本文

.claude/skills/weekly-report/SKILL.md:

---
name: weekly-report
description: 週次業務日報を半自動生成する。議事録要約 + 過去記憶検索 + 指定フォーマットのドラフト出力まで一貫して担当。Use when ユーザーが週報・日報・週次サマリの作成を依頼したとき、または議事録ファイル更新後に週次集約が必要なとき。
---

## この Skill がやること
1. 今週の議事録ファイル (.md) を読み込む
2. 「決定事項」「宿題」「次週の予定」を抽出
3. 過去の週報 (MEMORY.md / トピックファイル) を必要時に参照し、継続中タスクを補完
4. 指定フォーマットで週報ドラフトを出力

## 出力フォーマット
### 今週のハイライト (3 行以内)
### 決定事項
### 宿題 (担当・期限付き)
### 来週の予定

## 注意事項
- 推測で補完せず、議事録にない情報は「未記載」と明記
- 前週からの継続タスクは必ず参照

Skill を作る手順 (4 ステップ)

Step 1: 自分の「繰り返し作業」を 1 つ選ぶ (5 分)

「毎週 / 毎日 / 毎月、同じ手順でやっていること」を 1 つ見つける

具体例:
- 営業: 商談メモから提案書ドラフトを作る
- PM サポート: ミーティング議事録からタスク管理表を更新する
- データ分析: 分析レポートの定型フォーマットを生成する

Step 2: 「何を読み込み → 何を出力するか」に分解 (10 分)

入力 (何を読むか) → 処理 (どう変換するか) → 出力 (何を作るか)

Step 3: .claude/skills/<name>/SKILL.md に frontmatter + 本文を書く (10 分)

ディレクトリを作り、SKILL.md をエントリポイントとして配置 (legacy 単一 .md 形式ではなくディレクトリ形式が公式標準)

Step 4: 実際に呼び出して動作確認 (5 分)

AI に「週報作って」と伝えるだけで description マッチで自律発動するかを確認

3 層統合 — Skill が 3 つの層を統合する仕組み

Skill 1 つの中に、講座で学んだ 3 層がすべて入っている:

層 Skill 内での役割 対応箇所
プロンプト層 日報フォーマット・抽出ルールの指示 Skill 本文の「やること」「フォーマット」「注意事項」
コンテキスト層 過去の週報を MEMORY.md / トピックファイルから必要時に参照 「過去の週報を参照」ステップ
ハーネス層 (Anthropic 公式概念) hook (公式 29 種類イベントの 1 つ) 連携で議事録更新時に自律発動 ch07 で学んだ hook と連携 (構想レベル)

Skill = 3 層を 1 つにまとめた「自分専用の業務知識パッケージ」

演習: 自分の業務 Skill を作ってみよう

演習の流れ

  1. 自分の業務で「毎週繰り返している手作業」を 1 つ選ぶ (5 分)
  2. その手作業を「入力 → 処理 → 出力」に分解する (10 分)
  3. .claude/skills/<name>/SKILL.md ディレクトリ + エントリポイント形式で frontmatter + 本文を書く (10 分)

Skill と hook の連携 — さらに先へ

ch07 で学んだ hook (公式 29 種類イベント) と Skill を組み合わせると、完全な自動化が実現する:

┌───────────────────┐
│  議事録が更新された │ ← タイミング (FileChanged 等の公式 hook が検知)
└────────┬──────────┘
         │
         ▼
┌───────────────────┐
│  Skill が発動する   │ ← 業務知識 (Skill 本体をオンデマンド読込)
│  「週報作成」       │
│  1. 議事録を読む    │
│  2. MEMORY.md を参照│
│  3. ドラフトを出力  │
└───────────────────┘

hook = 「いつ動くか」の引き金 (公式 29 種類のライフサイクルイベント)
Skill = 「何を呼ぶか」の業務知識セット (description で自律発動条件を定義)
hook + Skill = 完全な自動化

発展: hook 連携で自動化

ch07 の hook (公式 29 種類イベントの 1 つ) と連携して「ファイル更新で自律発動」を実現する

まとめ

  • Skill = .claude/skills/<name>/SKILL.md というディレクトリ + エントリポイント形式に業務知識を書き込む公式機能
  • description のみが常駐し、本体は使用時にオンデマンド読込される
  • description の自然文 + Use when 形式が 自律発動の引き金
  • Skill は 3 層 (プロンプト / コンテキスト / ハーネス) を統合する
  • カスタムスラッシュコマンド (Skills/legacy commands) は「人が呼ぶ」→ Skill 自律発動は「AI が文脈で呼ぶ」
  • hook (公式 29 種類イベント) と Skill を連携させれば、完全な自動化が実現する
  • 講座の到達点は 「自分専用の業務知識パッケージ」を 1 つ完成させる こと

強化した層: 全層統合 (L1 プロンプト層 + L2 コンテキスト層 + L3 ハーネス層) — Skill で 3 層を 1 つのディレクトリに統合し、自分専用の業務知識パッケージを作り上げる。3 層統合の到達点。
