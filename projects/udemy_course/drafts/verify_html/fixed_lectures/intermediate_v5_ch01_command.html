カスタムスラッシュコマンド — プロンプトを資産にする仕組み

.claude/commands/
  └── summarize-meeting.md   ← このファイルがプロンプト本体
  • .claude/commands/ フォルダに Markdown ファイルを置くだけ
  • ファイル名がそのままコマンド名になる
  • 例: summarize-meeting.md → /summarize-meeting で呼び出し

※ Claude Code 公式に /command という単一コマンドは存在しません。
「カスタムスラッシュコマンド」とは、自分で /<任意の名前> を作る仕組みのことです。

実演: カスタムスラッシュコマンドを作ってみよう

Step 1 — フォルダを作る

プロジェクトのルートに .claude/commands フォルダを作ります:

mkdir -p .claude/commands

※ VS Code のエクスプローラーから「新しいフォルダー」で作っても OK です

Step 2 — プロンプトファイルを書く

<!-- .claude/commands/summarize-meeting.md -->
以下の議事録を読み、要点を 3 つにまとめてください。

## 出力形式
- 決定事項
- 次回のアクション
- 未解決の課題

## 対象ファイル
$ARGUMENTS

$ARGUMENTS には、コマンド呼び出し時に渡した内容が入ります。
後ほど実演します。

実演: カスタムスラッシュコマンドを呼び出す

Step 3 — 使う

プロンプト欄にこう入力するだけ:

/summarize-meeting @meeting-2026-05-06.md
  • /summarize-meeting = コマンド名
  • @meeting-2026-05-06.md = 対象ファイル (引数)
  • @ファイル名 で AI にそのファイルを読み込ませる Claude Code の参照記法

毎回プロンプトを書く必要がなくなりました

引数付き呼出の仕組み

<!-- .claude/commands/weekly-report.md -->
以下の進捗データから、週次レポートを作成してください。

## フォーマット
1. 今週の成果 (3 行以内)
2. 課題と対応状況
3. 来週の予定

## 対象
$ARGUMENTS

/weekly-report @progress.md
  • $ARGUMENTS の位置に @progress.md の中身が差し込まれるイメージ
  • プロンプトの中身は変えずに、対象だけ変えられる

Script / Slash Command / Skill — 3 つの道具の使い分け

観点 Script (Bash 等) カスタムスラッシュコマンド (/name) Skill
起動 コマンドライン手動 ユーザーが /<名前> と入力 description のキーワードマッチで AI が自律呼出
配置 任意のフォルダ .claude/commands/<名前>.md (legacy) .claude/skills/<名前>/SKILL.md (公式標準・ディレクトリ形式)
引数 CLI 引数 $ARGUMENTS / @file 参照 プロンプト本文から AI が抽出
学習負荷 低 中 やや高 (但し再利用最大)
用途 手動繰り返し作業 定型プロンプトの資産化 業務領域知識のオンデマンド読込
本講座での位置 補助 ← 今ここ (ch01) 到達点 (ch11)

公式は「カスタムコマンドはスキルにマージされました」と明記。本章では分かりやすさのため legacy の .claude/commands/ から入り、ch11 で Skill (ディレクトリ形式) に進みます。

3 つの道具 — ざっくり言うと

  • Script: シェルスクリプトと同じ。手動で叩く
  • カスタムスラッシュコマンド: /<任意名> で呼び出す定型プロンプト。本章の主役
  • Skill: description のキーワードに合致すると AI が文脈を判断して自律的に呼び出す。ch11 で作る

まずはカスタムスラッシュコマンドをマスターして、最後に Skill に到達する。

業務 3 例 — カスタムスラッシュコマンド化候補あぶり出し

営業パーソンの例

  • 毎回やっている: 顧客への提案メールをゼロからプロンプトで書く
  • カスタムスラッシュコマンド化: .claude/commands/proposal-email.md
  • 「顧客名・商談内容・条件を入力して提案メールを作成」
  • 次から /proposal-email で 1 発

PM サポートの例

  • 毎回やっている: 進捗レポートのフォーマットから毎回作り直す
  • カスタムスラッシュコマンド化: .claude/commands/weekly-report.md
  • /weekly-report @進捗.md で即座に生成

データ分析者の例

  • 毎回やっている: 「このデータの傾向を教えて」と毎回手動で書く
  • カスタムスラッシュコマンド化: .claude/commands/analyze-trend.md
  • /analyze-trend @sales.csv で 1 発

あなたの業務で 3 つ挙げてみよう

動画を止めて、自分の業務で「毎回ゼロから書いているプロンプト」を 3 つ挙げてください。

  1. ___ → コマンド名: /_________
  2. ___ → コマンド名: /_________
  3. ___ → コマンド名: /_________

この 3 つが、あなたの最初の資産になります。

Skill 布石 — 次の段階への予告

カスタムスラッシュコマンドは ユーザーが手動で呼ぶ 道具 (/<名前> と入力)
Skill は AI が description のキーワードマッチで自律的に呼ぶ 道具

例:
- カスタムスラッシュコマンド版: /proposal-email と手動で呼ぶ
- Skill 版: 商談メモを渡すと、description に「提案メール作成」相当のキーワードがマッチして AI が勝手に過去の提案パターンを参照してメールを作る

Skill の詳細は ch11 で。まずはカスタムスラッシュコマンドで土台を固めましょう。
