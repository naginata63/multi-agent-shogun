L1: プロンプトエンジニアリング — AIに確実に伝える技術

第1章 (約 30 min)

「Claude Code はじめての一歩 — AI開発の基礎を1日で習得」

**この章で出てくる用語** - **プロンプト** — AIに出す指示文のこと - **@file参照** — `@ファイル名` と書くだけでAIがそのファイルを自動読み込みする機能 - **再現性** — 同じ指示を出した時に、毎回同じ品質の結果が得られること - **バグ** — プログラムの間違い・不具合のこと - **ユニットテスト** — 関数が正しく動くか確認する小さなテストのこと - **スクリプト** — AIへの指示やコマンドをファイルに書いて、何度でも繰り返し実行できるようにしたもの

🗺️ 全体像とこの章のゴール

序章でも紹介した 3階層モデル の最初の鍵 L1 を本章で習得します

階層 名前 何をするか どこで学ぶ
L1 プロンプト AIに指示を出す ← 今ここ (第1章)
L2 コンテキスト AIに前提知識を与える 第2〜4章
L3 ハーネス AIの動作を自動化・拡張する層 ← 第5章以降で扱います。今はスルーOK 第5章以降

🎯 この 30 分で得られること
1. 良いプロンプトの3要素 (文脈・指示・形式) を理解する
2. @file参照・段階分解・箇条書き などの実践テクニックを知る
3. 同じプロンプトで 再現性のある結果 を得るコツを理解する

こんな指示を出していませんか? — 3要素で解決

「いい感じにテストを書いて」「これ、よろしく」 — 同じ指示なのに毎回違う結果 が返ってくる

人間なら分かる、AIには伝わらない:
- 「これ」が何を指すか不明・「よろしく」の範囲が無限大
- AIは 意図を推測 して動く → 曖昧な指示 = 曖昧な結果

解決策: プロンプトを 3 要素に分けて書く

要素 役割 例
文脈 AIに与える背景・素材 「この記事について…」「@src/app.ts を見て…」
指示 AIにやってほしいこと 「要約して」「バグを特定して」
形式 結果をどう返すか 「箇条書き3つで」「表形式で」

💡 3要素を整理すると、AIの誤解が減って 出力品質が安定 します

テクニック1: @file でファイルを参照する

@src/app.ts のバグを特定して。
各バグについて深刻度（高/中/低）と
修正方法を箇条書きで教えて。

3要素に分解してみよう:
| 要素 | 対応部分 |
|------|---------|
| 文脈 | @src/app.ts（Webアプリのコードファイルを読み込む） |
| 指示 | 「バグを特定して」 |
| 形式 | 「深刻度と修正方法を箇条書きで」 |

ポイント:
- @ファイル名 でプロジェクト内のファイルを直接参照（コードをコピペ不要）
- AIが自動的にファイル内容を読み込む

💡 Claude Code なら、ファイルの中身を 貼り付け不要

テクニック2: 段階分解 — 「まず○○、次に△△」

このプロジェクトで以下の手順で進めて：
1. まず @package.json（パッケージ管理ファイル）の依存関係（どのライブラリを使っているか）を確認して
2. 次に 使っていないパッケージを特定して
3. 最後に 削除しても安全か判断して

ポイント:
- 複雑な作業は 段階に分ける と精度が上がる
- 「まず」「次に」「最後に」で順序を明示
- AIが途中で迷わない

テクニック3: 具体的に書く — 曖昧→具体

悪い例:

いい感じにテストを書いて

良い例:

@src/utils.ts の各関数に対して
ユニットテストを書いて。
テストフレームワークはvitest（JavaScriptのテストツール）を使い、
正常系（普通の入力）・異常系（おかしい入力）・境界値（ギリギリの値）
の3パターンを
それぞれカバーすること。

「いい感じに」→ 毎回違う結果
「正常系・異常系・境界値」→ ブレが大幅に減る

テクニック4: 箇条書きで整理

箇条書きで複数条件を渡す:

以下の観点で @README.md を改善して：
- セットアップ手順が分かりやすいか
- 必要な前提条件が書かれているか
- 使用例が含まれているか

ポイント:
- 人間にもAIにも分かりやすい形式
- 複数の条件や観点を伝える時に効果的
- チェックリストのように使える

Before / After で見る再現性

Before（曖昧な指示）: 毎回違う形式で出力

1回目: 段落形式で要約
2回目: 箇条書きで要約
3回目: 表形式で要約

After（3要素 + 具体性）: 形式が揃い、ブレが大幅に減る

1回目: 箇条書き3項目 ✓
2回目: 箇条書き3項目 ✓（細部は少し変わることも）
3回目: 箇条書き3項目 ✓

再現性が重要な3つの場面:
- 使い回し: 同じ処理を別のファイルに適用
- 共有: チームメンバーにプロンプトを渡す
- 自動化: スクリプト（AIへの指示をファイルに書いて繰り返し実行する方法）の中で何度も実行

再現性のないプロンプト = 使い捨て / 再現性のあるプロンプト = 資産

よくある失敗パターン 3選

# 失敗パターン 例 修正方法
1 曖昧な指示 「いい感じに要約して」 具体的な条件を書く
2 形式の指定なし 指示だけで「どう返すか」未指定 「箇条書きで」「表で」を必ず書く
3 文脈不足 背景情報を何も伝えない 前提条件・@file参照で文脈を与える

この3つを避けるだけで、プロンプトの品質が 劇的に向上 する

📚 まとめ — プロンプト3要素の復習

文脈  → 素材・背景（例: @src/app.ts でファイルを読み込む）
指示  → やってほしいこと（例: 「バグを特定して」）
形式  → 結果の形式（例: 「箇条書き3つで」）

この章で学んだこと:
1. プロンプトを文脈・指示・形式の3要素で整理する
2. @file参照・段階分解・箇条書き等の実践テクニック
3. 具体的に書くことで再現性を担保する
