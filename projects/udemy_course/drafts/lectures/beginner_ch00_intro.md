# あなたの今の AI 開発、こうなっていませんか？

> 🪞 「ブラウザでAIに質問して、コピペして、動かなくて困っている」

- AIにコードを書かせたが **コピペが面倒**
- 会話が長くなると **AIが前の話を忘れる**
- 毎回同じ前提条件を **説明し直している**
- 「このファイルを直して」が **ブラウザからできない**

<!--
多くの方がChatGPTやClaudeのブラウザ版を使って開発の補助をしています。しかし、ブラウザとエディタを行き来する作業は、実は大きなロスです。AIがあなたのプロジェクトのファイルを直接読み、書き、実行できたら——と思う方も多いはず。それを実現するのがClaude Codeです。
-->

# AI 開発ツールの 3 つの形態 (2026年)

| 形態 | 代表例 | 特徴 |
|------|--------|------|
| **Chat** | ChatGPT, Claude.ai | ブラウザで会話。コピペ必須 |
| **IDE 拡張** | Cursor, GitHub Copilot | エディタ内補完。能動的な作業は限定的 |
| **CLI** | Claude Code, Aider | ターミナルで自律的に読む・書く・実行する |

> 💡 Claude Code は **CLI 形態** に分類されます

<!--
2026年現在、AI開発ツールは大きく3つの形態に分かれています。Chatは手軽ですがプロジェクトとの連携が弱い。IDE拡張は補完に強いですが、AIが自律的に動く範囲は限定的です。CLI形態は、AIがあなたのプロジェクト全体にアクセスし、自らファイルを読み、編集し、テストを実行できます。この「自律性」の高さが最大の違いです。
-->

# 🎬 ブラウザ AI vs CLI AI — 何が違う？

[ここで実画面録画を 3 分流す]

**ブラウザ版 (60 秒)**:
- コードをコピペして質問 → 回答をコピペして戻す
- ファイルを直接操作できない

**CLI 版 Claude Code (90 秒)**:
- AIが自らファイルを読み → 編集 → テストを実行
- 人間は指示を出すだけ、AIが作業を完走

**差分 (30 秒)**:
- ブラウザ版: 5分の手作業 → CLI版: 30秒で完了

<!--
ここで実際の画面をお見せします。左半分にブラウザ版Claude、右半分にCLI版Claude Codeを並べて、同じタスクを依頼します。ブラウザ版ではコードをコピーして貼り付け、回答をまたコピーしてエディタに戻す作業が必要です。一方Claude Codeでは、「このファイルのバグを直して」と伝えるだけで、AIが自らファイルを読み、原因を特定し、修正し、テストまで実行します。この差が「AI開発の新しい形」の正体です。
-->

# 他の AI ツールとの違い

| ツール | 形態 | 自律的にファイル編集 | テスト実行 | 提供元 |
|--------|------|---------------------|-----------|--------|
| **Claude Code** | CLI | ✅ | ✅ | Anthropic |
| Cursor | IDE | ✅ (限定的) | ❌ | Anysphere |
| GitHub Copilot | IDE/CLI | ❌ (補完中心) | ❌ | GitHub |
| Aider | CLI | ✅ | △ (外部連携) | オープンソース |

> **Claude Code = 読む・書く・実行する の3拍子**

<!--
他のツールとの違いを整理します。CursorはIDE内でファイル編集ができますが、テストの実行は人間が行う必要があります。GitHub Copilotはコード補完に特化しており、自律的な編集は行いません。AiderはCLIツールで自律的な編集が可能ですが、テスト実行は外部ツールとの連携が必要です。Claude Codeは、読む・書く・実行するの3つを1つのツールで完結できる点が最大の強みです。
-->

# 本講座で学ぶ 3 つのこと

1. **CLAUDE.md の役割** — プロジェクト固有の設定ファイルの書き方と効果
2. **PreToolUse hook の仕組み** — AIの安全装置の仕組みと設定方法
3. **MCP 接続の基礎** — AIの記憶を拡張する仕組み

> ✅ この 3 点が分かれば、Claude Code を「自分のプロジェクトで使える」状態になります

<!--
この講座で最終的に到達するのは、この3つの理解です。CLAUDE.mdはAIにプロジェクトのルールを教えるファイル。hookはAIが危険な操作をしないよう監視する安全装置。MCPはAIの記憶をセッションを超えて引き継ぐ仕組みです。この3点が揃えば、あなたはClaude Codeを自分のプロジェクトで本格的に活用できるようになります。
-->
