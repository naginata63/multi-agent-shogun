# 生成AIとは — 「探す」から「作る」へ

> 🪞 「AI」= 便利な検索機能だと思っていませんか？

**従来のAI**:
- ルールベース — 人間が決めた条件で「分類」する
- 検索 — 既にある情報を「見つける」

**生成AI**:
- テキスト・コード・画像を **新しく「生成」する**
- 人間が書いていない文章を、AIが **自分で組み立てる**

<!--
従来のAIは「正解を探す」のが得意でした。例えばスパムメールを分類するとか、検索結果を上位表示するなど。生成AIはまったく違います。今まで存在しなかった文章を、確率計算で組み立てて「新しく作る」。これが最大の違いです。
-->

---

# LLMの仕組み — 「次の単語」を予測するだけ

```
入力:   "吾輩は"
予測:   "猫"  ← 確率が最も高い
        "犬"  ← 次に高い
        "鳥"  ← さらに低い
```

**LLM = Large Language Model（大規模言語モデル）**:
- インターネット上の膨大なテキストを学習
- 入力に対して **次に来る確率が最も高い単語** を出力
- これを繰り返して「文章」を作る

> ⚡ 魔法ではなく、**超高速の確率計算**

<!--
「AIって魔法？」と思うかもしれませんが、仕組みはシンプルです。「吾輩は」と入力されたら、次に来る確率が一番高いのは「猫」。これを1文字ずつ、1秒間に何十回も繰り返す。それが生成AIの文章生成の正体です。シンプルな仕組みですが、学習データが膨大だからこそ、まるで人間が書いたような自然な文章になります。
-->

# AI開発ツールの3形態 — Chat / IDE / CLI

```
┌──────────────────────────────────────────────┐
│              AI開発ツールの全体観              │
├────────────┬────────────┬────────────────────┤
│  Chat形式  │  IDE形式   │    CLI形式          │
│  ブラウザ   │  エディタ   │    ターミナル       │
│            │            │                    │
│ ChatGPT    │ Cursor     │  Claude Code       │
│ Claude.ai  │ Copilot    │  Aider             │
│ Gemini     │ Cline      │  Codex CLI         │
├────────────┼────────────┼────────────────────┤
│ 会話のみ   │ 行単位補完  │ ファイルを直接操作   │
│ 手動コピペ │ 自動提案    │ 自動読み・書き・実行 │
└────────────┴────────────┴────────────────────┘
```

<!--
AI開発ツールは大きく3つの形態に分かれます。Chat形式はブラウザで会話するだけ。IDE形式はVSCodeなどのエディタ内で動いて、行単位でコードを提案。そしてCLI形式は、ターミナル上でAIがファイルを直接読み、書き、テストまで実行する。本講座で学ぶClaude Codeは、この3つ目のCLI形式にあたります。
-->

# Claude Codeの位置づけ

| 特徴 | ChatGPT | Cursor | **Claude Code** |
|------|---------|--------|----------------|
| 操作 | ブラウザで会話 | エディタ内補完 | **ターミナルで自律作業** |
| ファイル操作 | できない | エディタ内のみ | **プロジェクト全体を操作** |
| テスト実行 | できない | できない | **自動で実行** |
| 設定ファイル | なし | あり | **CLAUDE.md で永続化** |

**Claude Code = AIがあなたのターミナルで「開発者」として働く**
