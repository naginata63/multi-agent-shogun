# 到達点1: プロンプト [L1] — AIに確実に伝える

**何を学んだか**:
- 構造化プロンプトの書き方（入力・処理・出力の3要素）
- Before/Afterで確認した「伝わる指示」と「伝わらない指示」の差

**何が解決できたか**:
- AIへの指示が **毎回ブレる問題** を解決
- 同じプロンプトで **再現性のある結果** を得られる

> プロンプト = AIとのコミュニケーションの「基礎言語」

<!--
第1章で学んだプロンプト基礎。これは3階層の一番下、L1にあたります。AIに「伝わる指示」と「伝わらない指示」の差は、構造化するだけで劇的に変わります。入力・処理・出力の3要素に分解して書くことで、AIへの指示がブレなくなり、再現性のある結果が得られる。これがAI開発の第一歩であり、すべての基盤です。
-->

---

# 到達点2: コンテキスト [L2] — プロジェクト知識の共有

**何を学んだか**:
- CLAUDE.md（第2章）— AIに読ませるプロジェクトの「説明書」
- .md仲間たち（第3章）— commands / skills / imports 等の設定ファイル群
- 歴史と意義（第4章）— なぜ3階層が必要なのか

**何が解決できたか**:
- AIに毎回前提条件を説明する手間が **ゼロに**
- 設定ファイル1つでAIの振る舞いが **自動で変わる**

> コンテキスト = AIに「プロジェクトの文脈」を永続的に与える仕組み

<!--
第2章から第4章で学んだコンテキスト層。CLAUDE.mdは「AI向けのREADME」として、プロジェクトのルールをAIに教える仕組み。第3章では、CLAUDE.md以外にもcommands、skills、importsなどの.mdファイル群が存在することを学びました。そして第4章で、DevOpsからSRE、Platform Engineeringを経てAI Engineeringに至る歴史の流れの中で、このコンテキスト層がなぜ必要なのかを理解しました。人間が毎回説明しなくても、AIがプロジェクトの文脈を理解して動く。これがL2の価値です。
-->

---

# 到達点3: ハーネス [L3] — AIの安全装置

**何を学んだか**:
- hookの仕組み — AIのライフサイクルイベントに反応する安全装置
- PreToolUse hook でコマンド実行前に検査・ブロック

**何が解決できたか**:
- `rm -rf` や `sudo` 等の **危険操作を自動検知**
- AIが勝手にファイルを消すリスクを **排除**

> ハーネス = 「AIにブレーキをかける」仕組み

<!--
第5章で学んだhook。3階層の最上層、L3にあたります。PreToolUse hookは、AIがコマンドを実行する前に立ち止まらせる安全装置です。AIが危険なコマンドを実行しようとした瞬間に自動ブロックします。この1本のhookで、あなたのコードとプロジェクトが守られます。安心してAIに作業を任せるための必須の仕組みです。プロンプトで指示し、コンテキストで文脈を与え、ハーネスで制御する。この3層が揃って初めて、AIは安全な開発パートナーになります。
-->

# 初級編で解決した3つの課題

| 課題 | 解決策 | 階層 | 学んだ章 |
|------|--------|------|---------|
| AIへの指示が毎回ブレる | **構造化プロンプト** | L1 | 第1章 |
| AIに毎回ルールを説明する手間 | **CLAUDE.md + .md群** | L2 | 第2〜3章 |
| AIが危険な操作をする不安 | **hook** | L3 | 第5章 |

> この3つが揃えば、Claude Code は「安心して使える開発パートナー」になる

<!--
