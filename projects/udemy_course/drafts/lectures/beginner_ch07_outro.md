# 到達点1: CLAUDE.md — プロジェクトの「説明書」

**何を学んだか**:
- AIに読ませるプロジェクト固有の設定ファイルの書き方
- 5セクション構成で30行書くだけで、AIがルールを理解

**何が解決できたか**:
- AIに毎回同じ前提条件を説明する手間が **ゼロに**
- コーディング規約や使用ライブラリをAIが **自動で守る**

> CLAUDE.md = 「AI向けREADME」という発想の転換

<!--
第2章で学んだCLAUDE.md。これは「AIに読ませるプロジェクトの説明書」です。5セクション、たった30行書くだけで、AIはあなたのプロジェクトの使用言語、フレームワーク、コーディング規約、禁止事項を理解します。結果として、毎回同じ前提条件をAIに説明する手間がなくなり、AIが自動的にルールを守ってくれます。この「設定ファイル1つでAIの振る舞いが変わる」という体験は、AI開発の基礎中の基礎です。
-->

---

# 到達点2: hook — AIの「安全装置」

**何を学んだか**:
- AIのライフサイクルイベントに反応する仕組み
- PreToolUse hook でコマンド実行前に検査・ブロック

**何が解決できたか**:
- `rm -rf` や `sudo` 等の **危険操作を自動検知**
- AIが勝手にファイルを消すリスクを **排除**

> hook = 「AIにブレーキをかける」仕組み

<!--
第4章で学んだhook。特にPreToolUse hookは、AIがコマンドを実行する前に立ち止まらせる安全装置です。AIが`rm -rf`や`sudo`などの危険なコマンドを実行しようとした瞬間に、「STOP: 危険な操作を検知しました」と自動ブロックします。この1本のhookで、あなたのコードとプロジェクトが守られます。安心してAIに作業を任せるための必須の仕組みです。
-->

---

# 到達点3: MCP接続 — AIの「記憶」

**何を学んだか**:
- MCP（Model Context Protocol）でAIにプラグインを追加する仕組み
- memory MCP でセッションを跨いで記憶を保持

**何が解決できたか**:
- セッションを終えるたびに **記憶がリセットされる問題** を解決
- 前回の会話内容を **引き継いで** 作業を継続できる

> MCP = 「AIに永続的な記憶を与える」仕組み

<!--
第5章で学んだMCP接続。MCPはModel Context Protocolの略で、AIにプラグインを追加する仕組みです。memory MCPを使うと、セッションを跨いで記憶が引き継がれます。以前はセッションを終えるとAIが全てを忘れてしまい、毎回同じ説明を繰り返す必要がありました。MCP接続でこの問題が解決します。AIが前回の文脈を覚えていることで、作業の継続性が格段に向上します。
-->

# 3階層モデル — 中級編の中核概念

中級編の学習は **3つの階層** で構成されています:

```
階層3: ハーネス (harness)
  ↑ AIの行動を永続的に制御する仕組み
階層2: コンテキスト (context)
  ↑ AIに情報を永続的に提供する仕組み
階層1: プロンプト (prompt)
  ↑ AIへの指示を再利用可能にする仕組み
```

> 初級編で学んだのは **階層1と2の入り口**。中級編で全層を深掘りします

<!--
中級編を貫く中核概念が「3階層モデル」です。階層1のプロンプトは、AIへの指示を再利用可能な形にする仕組み。初級編第1章で学んだプロンプト設計がここに当たります。階層2のコンテキストは、AIに情報を永続的に提供する仕組み。CLAUDE.mdとMCPがここに当たります。階層3のハーネスは、AIの行動自体を永続的に制御する仕組み。hookがここに当たります。初級編では階層1と2の入り口を学びました。中級編では全3層を深く理解します。
-->
