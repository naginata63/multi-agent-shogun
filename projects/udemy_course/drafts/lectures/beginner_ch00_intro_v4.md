# 生成AIとは — 「探す」から「作る」へ

> 「AI」= 便利な検索機能だと思っていませんか？

**従来のAI**:
- ルールベース — 人間が決めた条件で「分類」する
- 検索 — 既にある情報を「見つける」

**生成AI**:
- テキスト・コード・画像を **新しく「生成」する**
- 人間が書いていない文章を、AIが **自分で組み立てる**

<!--
従来のAIは「正解を探す」のが得意でした。スパムメールを分類するとか、検索結果を上位表示するなど。生成AIはまったく違います。今まで存在しなかった文章やコードを、確率計算で組み立てて「新しく作る」。これが最大の違いです。2022年にChatGPTが登場して「生成AI」という言葉が一気に広まりましたが、本質はこの「作る」という一点にあります。
-->

---

# AIエンジニアリングとは

> 生成AIは、**チャットで質問するだけ**ではありません。

**AIエンジニアリング = AIが自律的にコードを読み・書き・実行する開発の形**

- 📖 **読む** — プロジェクトのファイルを自ら把握する
- ✏️ **書く** — コードを新規作成・修正する
- 🧪 **実行する** — テストを実行し、エラーがあれば修正する

> 「AIに質問する」から「AIに開発を任せる」への転換

<!--
ここがこの講座のキーポイントです。生成AIを「チャットで質問するツール」としてしか使っていない方は多いです。しかし本来の力はそこにとどまりません。AIがあなたのプロジェクトのファイルを自ら読み、コードを書き、テストを実行し、エラーが出たら自ら修正する。この「自律的な開発サイクル」こそがAIエンジニアリングの核心です。詳細な仕組みは中級編で扱いますが、まずは「AI開発＝質問するだけ」ではない、という世界観を掴んでください。
-->

# AI開発ツールの3形態

| 形態 | 場所 | 代表ツール | 操作の特徴 |
|------|------|-----------|-----------|
| **Chat** | ブラウザ | ChatGPT / Claude.ai | 会話のみ・手動コピペ |
| **IDE** | エディタ | Cursor / Copilot | 行単位補完・自動提案 |
| **CLI** | ターミナル | **Claude Code** | ファイルを直接操作・自律実行 |

> 本講座で学ぶ **Claude Code** は CLI 形式

<!--
AI開発ツールは大きく3つの形態に分かれます。Chat形式はブラウザで会話するだけ。IDE形式はエディタ内でコードを提案してくれる。そしてCLI形式は、ターミナル上でAIがファイルを直接読み、書き、テストまで実行する。本講座で学ぶClaude Codeは、この3つ目のCLI形式です。次のスライドで、なぜCLI形式が「AIエンジニアリング」に直結するのか見ていきましょう。
-->

# Claude Code — ターミナルで動く開発者

| 特徴 | ChatGPT | Cursor | **Claude Code** |
|------|---------|--------|----------------|
| 操作 | ブラウザで会話 | エディタ内補完 | **ターミナルで自律作業** |
| ファイル操作 | できない | エディタ内のみ | **プロジェクト全体を操作** |
| テスト実行 | できない | できない | **自動で実行** |

**Claude Code = AIがあなたのターミナルで「開発者」として働く**

- CLI特化 × Anthropic製 × 開発者向け
- 前のスライドの「読む・書く・実行する」を **そのまま実現**

<!--
この表の右列を見てください。ChatGPTは会話だけ、Cursorはエディタ内の補完だけ。Claude Codeは、あなたのターミナルの中でAIが自らファイルを読み、コードを書き、テストを実行し、エラーがあれば修正する。つまり先ほど紹介した「AIエンジニアリング」の読む・書く・実行するサイクルをそのまま実現するツール、それがClaude Codeです。
-->
