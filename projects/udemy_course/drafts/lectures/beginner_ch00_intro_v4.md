# こんな経験、ありませんか？

> AIを使っているはずなのに、思った通りに動いてくれない

- ChatGPTにコードを書かせたけど、**意図と違う**ものが返ってきた
- AIの使い方は分かったけど、**開発の流れ**に組み込めていない
- 「もっと効率よくAIを使えるはず」— そう**感じている**

<!--
スピーカーノート:
まず、あなたに質問です。ChatGPTやClaudeにコードを書かせたことはありませんか？書かせてみたものの、思い通りのコードが返ってこなかったり、同じ指示を出しているのに毎回違う結果が返ってきたり。AIの使い方自体は何となく分かった、でも「開発のフローの中でどう使えばいいのか」が見えていない。もっと効率よく、もっと確実にAIを使えるはずなのに——。そんな経験、ありませんか？実は、多くのエンジニアがここで立ち止まっています。次のスライドから、その理由を3つに分けて見ていきましょう。
-->

# 壁① — AIに意図が伝わらない

> 「もっとシンプルにして」→ AIは別の解釈をする

- 同じ指示を出しているのに、**毎回違うコード**が返ってくる
- 出力が長すぎたり、**的外れ**だったり
- 「日本語で言えば分かるはず」— でも伝わらない

<!--
スピーカーノート:
1つ目の壁です。「AIに指示を出しているのに、意図が伝わらない」という悩み。「もっとシンプルにして」と言っても、AIは別の解釈をして全く違うコードを返してくる。同じプロンプトを何度も入力しているのに、毎回微妙に違う出力が返ってくる。「日本語で言えば分かるだろう」と思っているのに、なぜか伝わらない。この経験、心当たりがありますよね。実はこれ、AI側の問題ではなく、指示の出し方にパターンがあるんです。そのパターンを第1章で学びます。まずは「自分だけの問題ではない」ということを知ってください。多くの人がこの壁にぶつかっています。
-->

---

# 壁② — AIがプロジェクトを知らない

> 会話のたびに前提条件を説明し直す…非効率すぎる

- プロジェクトの**ルールや構造**をAIに説明しきれていない
- 会話のたびに「このプロジェクトは…」と**毎回ゼロから説明**
- AIが文脈を理解しないまま**コードを書く**ので不整合が起きる

<!--
スピーカーノート:
2つ目の壁です。「AIがプロジェクトの前提を知らない」という問題。あなたのプロジェクトには固有のルールがありますよね。使っているフレームワーク、ディレクトリ構成、コーディング規約。でもAIは、あなたが教えない限りそれを知りません。だから会話のたびに「このプロジェクトはReactで、TypeScriptを使っていて…」と毎回ゼロから説明する羽目になる。それって、すごく非効率ですよね。しかも、説明し忘れたことがあると、AIはその前提を無視したコードを書いてしまいます。この壁も、ちゃんとした仕組みを作れば解決できます。第2章と第3章でその仕組みを学びます。
-->

---

# 壁③ — AIの動きをコントロールできない

> AIが勝手にファイルを変更したら…怖くないですか？

- AIに**自動化**を任せたいけど、**怖い**
- テストなしでコードを書くのは**不安**
- 「人間の承認」を挟む仕組みが**分からない**

<!--
スピーカーノート:
3つ目の壁です。少し踏み込んだ話になりますが、一番大切な壁かもしれません。「AIに開発を任せたい」けど「AIが勝手にファイルを変更したり、想定外の動きをしたら怖い」という不安。これは当然の感覚です。テストなしでコードを書くのは人間でも不安ですよね。それをAIにやらせるとなると、なおさらです。「人間が必ず確認する仕組み」があれば安心できるのに、それをどう作ればいいか分からない。この壁を乗り越えると、AIを「怖いけど便利なツール」から「信頼できる開発パートナー」に変えることができます。第5章でこの安全装置の作り方を学びます。
-->

# 実は、壁は3つだけ

> この3つの壁を乗り越えれば、AIが**開発パートナー**になります

- AI開発の悩みはどれだけ複雑に見えても、分解すると**「伝える」「共有する」「制御する」**のどれかに分類できる — 他の悩みも試しに当てはめてみてください
- たとえば「AIが古いライブラリを使う」→ 前提を共有していない (壁②)。「余計なファイルまで変える」→ 制御できていない (壁③)
- **3つの壁にそれぞれ解決策を1つずつ対応させれば、すべての悩みに対応できます**

<span class="big">壁① → 伝える技術</span>
<span class="big">壁② → 共有する仕組み</span>
<span class="big">壁③ → 安全装置</span>

**この講座は、この3つを順番に学ぶカリキュラムです**

# 3つの鍵

> L = Layer（層）。レンガを下から積み上げるように、3つの技術を順に身につけます

| 鍵 | できるようになること | 対応する章 |
|----|---------------------|-----------|
| **L1: プロンプト** (= 指示文) | AIに意図が伝わる | 第1章 |
| **L2: コンテキスト** (= 前提メモ) | プロジェクト知識をAIに共有 | 第2章〜第4章 |
| **L3: ハーネス** (= 安全ベルト) | AIの動きを安全に制御 | 第5章 |

> L1 → L2 → L3 の順に段階的に身につけます
> L2 が3章分あるのは、プロジェクトの前提を共有する方法が一番ボリュームが多いからです

<!--

# Claude Code — この講座で使うツール

> **L1/L2/L3をそのまま実現する**、ターミナルで動くAI開発ツール

- ターミナル（黒い画面のアプリ）上でAIが**ファイルを読み、コードを書き、テストを実行**する
- ブラウザのチャットとは違う — **プロジェクト全体を操作**できる（あなたのパソコンにあるコードファイルを直接書き換えてくれます）
- この講座のすべての内容は **Claude Codeで実践**します
- **インストールは手取り足取り説明します**ので安心してください
