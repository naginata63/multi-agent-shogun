### 0.1 ペルソナ定義 (4タイプ)

本講座の主ターゲット層を4つのペルソナで定義する（中級編 P1-P5 とは別層）。

#### Q1: 新人バックエンドエンジニア「小林」
- **年代**: 24歳
- **職種**: Web系バックエンドエンジニア (Ruby/Python)
- **業界**: SaaSスタートアップ (入社1-2年)
- **年収**: 400万円
- **具体困りごと**: 業務でClaudeは触ったことがあるが、Chat画面で質問する程度。AIを使った「開発自動化」が何から始めればいいか分からない。先輩エンジニアがClaude Codeを使っているのを見て「自分も使いこなしたい」と思っている。
- **購買動機**: 「AI開発の第一歩」を確実に踏み出したい。難しいことは後回しで、まずは1日で基礎を身につけたい。

#### Q2: 文系出身プログラマー「高橋」
- **年代**: 28歳
- **職種**: Webフロントエンドエンジニア (JavaScript/TypeScript)
- **業界**: 中規模Web制作会社（転職組・未経験から3年）
- **年収**: 500万円
- **具体困りごと**: bash/git は使えるが、ターミナルでのAI操作は未経験。VSCodeの拡張機能としてClaudeを使っているが「CLI版のClaude Code」が何なのか分からない。もっと効率的にAIと開発したい。
- **購買動機**: 文系出身でも理解できる解説で、AI開発ツールの使い方を体系的に学びたい。

#### Q3: 副業希望IT会社員「中村」
- **年代**: 32歳
- **職種**: 社内SE / 情報システム部
- **業界**: 中堅メーカー (従業員300名)
- **年収**: 550万円
- **具体困りごと**: 本業は社内システム保守。AI開発スキルを身につけて、週末の副業案件（AI導入支援・スクレイピング自動化等）を獲得したい。何から始めればいいか全体像が見えない。
- **購買動機**: 「AI開発ができる」スキルを実務レベルで身につけ、副業市場で価値を提供したい。

#### Q4: 学生エンジニア「松本」
- **年代**: 22歳
- **職種**: 大学院生 (情報工学専攻)
- **業界**: 大学（研究/卒業課題でAI活用）
- **年収**: なし（奨学金+TA）
- **具体困りごと**: 研究でAIを使いたいが、APIを叩くだけでは限界を感じている。もっと本格的にAIと協働する開発手法を学びたいが、実務経験がないので中級〜上級向けの講座は難しすぎる。
- **購買動機**: 学生のうちにAI開発の基礎を固め、就職活動で「AI開発経験あり」をアピールしたい。

## 2. 章別構成 (ストーリー + ハンズオン + 学習目標)

### 序章: Claude Code とは — AI開発の新しい形 (L1: Remember)

**所要**: 15 min / **無料プレビュー**

#### ストーリー (Emotional Hook)
> 「今までのAI開発は『ブラウザを開いてチャットする』だけでした。Claude Codeは、あなたのターミナルの中でAIが直接コードを読み、書き、実行する——まったく新しい開発体験です。」

- 受講者の「これだ！」ポイント: 3分のデモで「ブラウザAI vs CLI AI」の違いを目で見せる。Chat版は会話だけ、Code版は自動でファイルを編集・テストを実行。このギャップが「やってみたい」のトリガー。

#### 学習目標
- Claude Codeと他のAIツール（ChatGPT、Cursor等）の違いを説明できる
- なぜCLI（コマンドライン）でAIを使うのか理由を言える
- 本講座の終着点（3つの成果物）を把握する

#### アウトライン
1. AI開発ツールの現在（2026年）: Chat / IDE拡張 / CLI の3形態
2. Claude Codeの立ち位置: CLI特化 × Anthropic製 × 開発者向け
3. 他のAIツールとの違い: Cursor, GitHub Copilot CLI, Aider等との比較
4. 本講座のゴール: 3つの成果物プレビュー（実際の画面キャプチャ）

#### ハンズオン具体化

- **どう作る**: (1) Node.js確認 → (2) npm install -g @anthropic-ai/claude-code → (3) claude login → (4) 最初のプロンプト「現在のディレクトリのファイル一覧を教えて」→ (5) AIが自発的にツールを使う様子を観察
- **完成品見た目**: ターミナル上でAIと会話しながらファイル操作ができる状態
- **所要時間**: 演習10min + 解説20min

#### 学習目標
- Claude Codeをインストールし、初回ログインを完了できる
- ターミナル上でClaude Codeとの会話を開始できる
- AIが自発的にツール（Read, Bash, Write等）を使う様子を観察する

#### 成果物
- 動作するClaude Code環境

### 第2章 プロンプト基礎 — AIに確実に伝える技術 (L3: Apply)

**所要**: 45 min / Bloom: L3 Apply
**参考**: 中級編 第1章の圧縮版（完全な代替ではない）

#### ストーリー (Emotional Hook)
> 「『AIに伝わればいい』と思っていませんか？ 実は、伝わるプロンプトと**再利用できる**プロンプトは別物です。この章では、1回書いて終わる指示を、何度でも使える形に変えます。」

- 受講者の「これだ！」ポイント: 自分の適当な指示がXMLタグで構造化される瞬間。同じタスクを2回目に実行した時の違い（再現性）を体感。

#### ハンズオン具体化
- **何を作る**: 構造化プロンプト `prompts/task_01_v1.md`（業務タスク1件の関数プロンプト）
- **どう作る**: (1) 日常の「口頭指示」を1つ選ぶ → (2) 入力(`<input>`)/処理(`<task>`)/出力(`<output_format>`)の3パートに分解 → (3) XMLテンプレートに当てはめる → (4) Claude Codeで実行して結果を確認 → (5) 同じプロンプトを再実行して再現性を確認
- **完成品見た目**: 30-50行の構造化マークダウン。XMLタグ付き。
- **所要時間**: 演習15min + 解説30min

#### 学習目標
- XMLタグを使った構造化プロンプトを書ける
- 入力・処理・出力の3パートに指示を分解できる
- 同じプロンプトで再現性のある結果を得られる

#### 成果物
- `prompts/task_01_v1.md`

### 第3章 CLAUDE.md ミニマム — プロジェクトの「説明書」 (L3: Apply)

**所要**: 45 min / Bloom: L3 Apply
**参考**: 中級編 第3章の圧縮版（完全な代替ではない）

#### ストーリー (Emotional Hook)
> 「新しいプロジェクトに入った時、最初にREADME.mdを読みますよね？CLAUDE.mdは、AIに読ませるREADMEです。30行書くだけで、AIがプロジェクトのルールを理解します。」

- 受講者の「これだ！」ポイント: CLAUDE.mdなしで「テストを書いて」と頼んだ時と、ありで頼んだ時の結果の違い。コーディング規約や使用ライブラリをAIが勝手に守る様子に驚く。

#### ハンズオン具体化
- **何を作る**: `CLAUDE.md`（30行ミニマム版）
- **どう作る**: (1) 自分のプロジェクト（またはサンプルプロジェクト）の構成を整理 → (2) 使用言語/フレームワーク → (3) コーディング規約（3-5ルール）→ (4) よく使うコマンド → (5) やってはいけないこと（禁止事項2-3個）を30行にまとめる
- **完成品見た目**: 30行前後のマークダウン。5セクション構成。
- **所要時間**: 演習15min + 解説30min

#### 学習目標
- CLAUDE.mdの役割（AIへのプロジェクト説明書）を理解する
- 最低限の構成（5セクション・30行）でCLAUDE.mdを作成できる
- あり/なしでAIの振る舞いが変わることを確認する

#### 成果物
- `CLAUDE.md`（30行ミニマム版）

### 第4章 簡単な自動化体験 — AIに任せてみる (L3: Apply + L4: Analyze)

**所要**: 45 min / Bloom: L3-L4

#### ストーリー (Emotional Hook)
> 「これまではAIに『質問』していました。この章からはAIに『作業』を頼みます。1つのファイル編集タスクで、AIに自動化させる体験をしましょう。」

- 受講者の「これだ！」ポイント: 「このファイルのバグを修正して」と頼むだけで、AIが自らファイルを読み→原因特定→修正→テスト実行まで完走する様子。手作業なら30分かかる作業が3分で終わる。

#### ハンズオン具体化
- **何を作る**: バグ修正の自動実行体験（サンプルプロジェクト使用）
- **どう作る**: (1) サンプルプロジェクト（意図的なバグ3個入り）を用意 → (2) CLAUDE.mdに「テストは必ず実行すること」を記載済み → (3) 「バグを修正して」とプロンプト → (4) AIが自律的にRead→Edit→Test→完了まで走る様子を観察 → (5) 修正内容を確認
- **完成品見た目**: バグ3個が修正されたソースコード
- **所要時間**: 演習15min + 解説30min

#### 学習目標
- AIに1つのファイル編集タスクを丸投げできる
- AIの自律的な作業フロー（Read→Edit→Test）を理解する
- 人間のレビューが必須であることを認識する

#### 成果物
- バグ修正済みのソースコード

### 第5章 hookの入門 — AIの安全装置を作る (L3: Apply)

**所要**: 45 min / Bloom: L3 Apply
**参考**: 1本だけ触れる。18本の全網羅は中級編で。

#### ストーリー (Emotional Hook)
> 「AIに『このファイルを消して』と言われたら、どうしますか？ hookは、AIが危険な操作をする前に**立ち止まらせる**安全装置です。たった1本で、あなたのコードを守れます。」

- 受講者の「これだ！」ポイント: hookなしで `rm -rf` を含むコマンドが実行される恐怖 → hookありで「STOP: 危険な操作を検知しました」とブロックされる安心感。この差が1発で分かる。

#### ハンズオン具体化
- **何を作る**: PreToolUse hook 1本（危険コマンド検知）
- **どう作る**: (1) `.claude/settings.json` に hook設定を追加 → (2) PreToolUse イベントで Bash コマンドを検査 → (3) `rm -rf` / `sudo` / `DROP TABLE` 等を含む場合はブロック → (4) 実際に危険なコマンドを送ってブロックされることを確認
- **完成品見た目**: `.claude/settings.json` + `hooks/pre_tool_use_safety.sh`（約20行）
- **所要時間**: 演習15min + 解説30min

#### 学習目標
- hookとは何か（AIのライフサイクルイベントに反応する仕組み）を理解する
- PreToolUse hook 1本を自分の環境に設定できる
- AIの危険操作を自動検知・ブロックできる

#### 成果物
- PreToolUse hook 1本（基礎安全装置）

### 第6章 MCP 接続入門 — AIの記憶を拡張する (L3: Apply)

**所要**: 45 min / Bloom: L3 Apply
**参考**: 「使う側」だけ。自前構築は中級編で。

#### ストーリー (Emotional Hook)
> 「AIとの会話を終えるたびに、記憶がリセットされていたらどうでしょう？毎回同じ説明を繰り返すのは非効率です。MCPでAIに『記憶』を与えましょう。」

- 受講者の「これだ！」ポイント: MCPなし → セッションを跨ぐと忘れる。MCPあり → 前回の会話内容を引き継ぐ。この違いを2つのデモで対比。

#### ハンズオン具体化
- **何を作る**: memory MCP接続済みのClaude Code環境
- **どう作る**: (1) MCPとは何かの概念説明（プラグインのようなもの）→ (2) `claude mcp add memory` で既存のmemory MCPを追加 → (3) 「私の名前は○○です」と覚えさせる → (4) 新しいセッションで「私の名前は？」と確認 → (5) 記憶が引き継がれていることを確認
- **完成品見た目**: `.claude/settings.json` にMCP設定が追加された状態
- **所要時間**: 演習15min + 解説30min

#### 学習目標
- MCP（Model Context Protocol）の役割を理解する
- 既存のMCPサーバーをClaude Codeに接続できる
- AIの記憶がセッションを跨いで引き継がれることを確認する

#### 成果物
- memory MCP接続済みのClaude Code環境（次のステップで使える状態）

## 5. 想定完走率と効果測定

### 5.1 完走率目標
