# /format-prompt — プロンプト統一フォーマット変換スキル

任意のプロンプトテキストを、coconalaプロンプト集の統一テンプレートに変換します。

## 使い方

```
/format-prompt
```

実行すると、変換したいプロンプトのテキストを貼り付けるよう求められます。

または直接テキストを渡して実行:

```
/format-prompt <プロンプトテキスト>
```

## 出力フォーマット

変換後は以下の統一テンプレート形式で出力されます:

```markdown
---
id: "カテゴリ_番号"
category: "カテゴリ名"
title: "プロンプトタイトル"
target_ai: "ChatGPT / Claude / Gemini / 全対応"
difficulty: beginner
tags: ["タグ1", "タグ2"]
---

## プロンプトタイトル

### 用途
このプロンプトで何ができるか（1〜2文で簡潔に）

### 対象AI
ChatGPT / Claude / Gemini / 全対応

### プロンプト
```
プロンプト本文（{{プレースホルダー}}形式）
```

### 使い方
プレースホルダーの置き換え方、効果的な使い方のコツ

### 出力例
```
AIが生成する出力のサンプル（概要でも可）
```
```

## 変換ルール

1. **タイトル**: 動詞＋目的語の形式（例: 「営業メール作成」「コードレビュー依頼」）
2. **用途**: 「〜する」で終わる1〜2文。具体的な成果を記述
3. **対象AI**: 動作確認済みのAIを列挙。汎用的なら「全対応」
4. **プロンプト本文**: 変数部分は `{{変数名}}` で統一。コードブロック内に記述
5. **使い方**: プレースホルダーの説明＋応用のヒントを記述
6. **出力例**: 実際の出力またはその概要を記述（長い場合は冒頭のみ）
7. **difficulty**: beginner / intermediate / advanced のいずれか
   - beginner: 特別な知識不要、コピペで即使える
   - intermediate: 業務知識や軽度のカスタマイズが必要
   - advanced: プロンプトエンジニアリングの理解が必要

## 実行手順

$ARGUMENTS が空の場合: 変換したいプロンプトテキストを教えてください。

$ARGUMENTS にテキストが含まれる場合: そのテキストを上記フォーマットに変換してください。

変換時は以下を実行:
1. プロンプトの意図・目的を分析する
2. 適切なカテゴリ・難易度を判断する
3. 統一テンプレートに変換して出力する
4. 変換できない情報は `{{要記入}}` プレースホルダーで示す
