## ★推奨差し替え案: **PEP 701 f-string 形式化** (クォート再利用)

**理由:**
1. **受講者ペルソナ適合**: 「IT 用語なんとか分かる社会人 2-3 年目」でも f-string は基本中の基本。**即座に「便利!」と実感**できる
2. **walrus との対比効果**: walrus (3.8・7 年前) に対し、f-string 改善は **Python 3.12 (2023-10) の正真正銘の最新機能**。「最新」訴求が成立する
3. **コードの視覚インパクト**: 修正前後の差が一目瞭然
   ```python
   # 〜Python 3.11: クォートが衝突しエラー
   f'playlist: {", ".join(songs)}'  # 外を ' で逃がす必要

   # Python 3.12+: 同じ " を中でも使える
   f"playlist: {", ".join(songs)}"
   ```
4. **講座章との親和性**: 中級編で取り扱う Hook stdin JSON や Skill metadata 整形で f-string は頻出 → 章をまたいで自然に応用可能

**講座差し替え文案** (60字程度):
> 「Python 3.12 (2023年) では f-string の制約が大幅緩和され、`f"{", ".join(items)}"` のようにクォートを内側でも自由に使えるようになりました」

walrus 演算子の記述は **削除推奨** (Python 3.8 由来のため「最新」と紹介すれば事実誤認になる)。
