## 致命傷 C5 判定

- **judgment**: ❌齟齬
- **official_fact**: 公式 Python 3.8 リリースノート (https://docs.python.org/ja/3/whatsnew/3.8.html) より:
  > 「大きな構文の一部として、変数に値を割り当てる新しい構文 `:=` が追加されました。この構文は [セイウチの目と牙] に似ているため、『セイウチ演算子』の愛称で知られています。」
  > 「完全な説明は **PEP 572** を参照してください。」
  > 正式名称: **代入式 (Assignment Expressions)**
  > 公式採用例: `if (n := len(a)) > 10:` / `while (block := f.read(256)) != '':` / `if (mo := re.search(r'(\d+)% discount', advertisement)):`
  > リリース日: 2019年10月14日 (Python 3.8.0)
  > 貢献者: Emily Morehouse (bpo-35224)

- **講座記述の問題**: 講座は walrus 演算子 `:=` を「Python **3.12** の例」として提示している (レビュアーが「実際は 3.8 由来」と指摘)。これは事実誤認:
  - walrus 演算子は **Python 3.8 (PEP 572・2019-10-14)** で導入された約 4 年半前の機能
  - Python 3.12 (2023-10-02) では walrus に関する新機能は追加されていない (3.12 の目玉は f-string PEP 701・型システム PEP 695 等)
  - もし「Python 3.12 で動く例」として walrus を提示しているだけなら無害だが、「3.12 の新機能 / 3.12 から使える」というニュアンスを与える書き方は完全な誤りであり、受講者 (生成 AI 勉強し始めた社会人 2-3 年目) を確実に誤誘導する

- **推奨修正**: 講座本文の walrus 言及箇所を以下のいずれかに書き換える:
  1. バージョン明示版: 「セイウチ演算子 `:=` は Python 3.8 (PEP 572) で導入された代入式構文です。Python 3.12 以降でも引き続き利用できます」
  2. バージョン非明示版: 「セイウチ演算子 `:=` (代入式・PEP 572) は ...」(3.12 への帰属表現を削除)
  3. 公式例の流用: `if (n := len(a)) > 10:` を使用例として採用 (公式ドキュメント由来で安全)
  - **必須**: 「Python 3.12 で walrus が ... 」「3.12 から使える ...」等の帰属表現は全削除
  - **推奨**: 脚注で「PEP 572・2019 年 10 月の Python 3.8 で導入」を一行追記
