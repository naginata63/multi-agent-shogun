## 統合の課題

### 1. 言語の不一致
- inbox_write.shは**書き込み専用**（一度だけ実行して終了）
- inbox_watcher.shは**長時間実行デーモン**（常駐プロセス）
- 両方bash+python3ハイブリッド。bash部分は統合に向かない

### 2. 責務の分離が適切
- Write: 同期的・確実な書き込み（1回実行）
- Watcher: 非同期的・継続的監視（デーモン）
- 統合すると「書き込み時に監視も起動する」のか「監視プロセスが書き込みも担当する」のか不明瞭

### 3. テスト容易性
- inbox_write.shは単体テスト可能（入力→出力）
- inbox_watcher.shはモックが必要（tmux, inotifywait）
- 統合するとテストが更に困難に

## 統合案（3案）

### 案A: Python統合（inbox_manager.py）

両方を単一Pythonスクリプトに統合。asyncioでイベントループ管理。

```
inbox_manager.py
├── write_message()    # 元inbox_write
├── watch_inbox()      # 元inbox_watcher（asyncio + inotify）
├── escalate()         # エスカレーションロジック
├── send_cli_command() # tmux send-keys wrapper
└── metrics()          # メトリクス記録
```

**メリット**: 型安全、テスト容易、保守性向上、yamlライブラリ統一
**デメリット**: 大規模リライト、Python依存追加、デバッグ難易度上昇

### 案B: 段階的Python化（推奨）

inbox_write.shのpython3インラインコードを抽出→共有モジュール化。watcherは段階的に移行。

```
Phase 1: 共通モジュール抽出
  lib/inbox_common.py  ← YAML読書、ロック、メッセージ生成
  inbox_write.sh → lib/inbox_common.py をimport
  inbox_watcher.sh → lib/inbox_common.py をimport

Phase 2: inbox_write.py化
  scripts/inbox_write.py ← bashラッパーをPythonに置換
  inbox_write.sh → `python3 scripts/inbox_write.py "$@"` の1行ラッパー

Phase 3: inbox_watcher段階移行
  scripts/inbox_watcher.py ← asyncioベースで再実装
  inotify → watchdog（Python純正）でクロスプラットフォーム対応
```

**メリット**: 段階的移行（リスク低減）、既存動作を壊さない、テスト段階導入
**デメリット**: 移行期間中の二重メンテ

### 案C: 最小限統合（共有ライブラリのみ）

統合せず、共通ロジックだけを共有ライブラリに抽出。

```
lib/inbox_common.py
  - yaml_read_write(path) → dict
  - acquire_lock(path) → context manager
  - generate_msg_id() → str
  - overflow_protection(messages, max=50) → list

inbox_write.sh → source or python3 -c "from lib.inbox_common import ..."
inbox_watcher.sh → 同上
```

**メリット**: 最小変更、DRY達成、リスク最小
**デメリット**: 二重メンテ継続、bash-python混合の複雑さ残存

## 推奨案: 案B（段階的Python化）

### 理由
1. **現状のbash-in-pythonハイブリッド**は保守性が低い（inbox_write.sh L173-227のpython3 -c "..."は可読性ゼロ）
2. **テストなし**で動いている → Phase 1でユニットテスト導入可能
3. **watcherのエスカレーション**が1263行の単一ファイル → モジュール分割で見通し改善
4. **段階的**なので1フェーズずつ検証可能

### Phase 1の詳細設計（共有モジュール）

```python
# lib/inbox_common.py
"""inbox_write/watcher 共通ユーティリティ"""
import fcntl
import os
import tempfile
import uuid
from datetime import datetime, timezone
from pathlib import Path
import yaml

def yaml_read(path: str | Path) -> dict:
    """YAMLファイルを読み込む。存在しない場合は空dictを返す"""
    path = Path(path)
    if not path.exists():
        return {"messages": []}
    with open(path) as f:
        return yaml.safe_load(f) or {}

def yaml_atomic_write(path: str | Path, data: dict):
    """アトミックにYAMLを書き込む（tmp + rename）"""
    path = Path(path)
    path.parent.mkdir(parents=True, exist_ok=True)
    tmp_fd, tmp_path = tempfile.mkstemp(dir=path.parent, suffix=".tmp")
    try:
        with os.fdopen(tmp_fd, "w") as f:
            yaml.dump(data, f, default_flow_style=False,
                      allow_unicode=True, sort_keys=False)
        os.replace(tmp_path, str(path))
    except BaseException:
        os.unlink(tmp_path)
        raise

def generate_msg_id() -> str:
    now = datetime.now(timezone.utc)
    rand = uuid.uuid4().hex[:8]
    return f"msg_{now.strftime('%Y%m%d_%H%M%S')}_{rand}"

def overflow_protection(messages: list, max_read: int = 30) -> list:
    """未読を全て保持 + 既読は最新max_read件のみ"""
    unread = [m for m in messages if not m.get("read", False)]
    read = [m for m in messages if m.get("read", False)]
    return unread + read[-max_read:]

class FileLock:
    """flockベースのファイルロック（context manager）"""
    def __init__(self, path: str, timeout: float = 5.0):
        self.path = path
        self.timeout = timeout
        self._fd = None

    def __enter__(self):
        self._fd = open(self.path, "w")
        fcntl.flock(self._fd, fcntl.LOCK_EX)  # blocking
        return self

    def __exit__(self, *args):
        if self._fd:
            fcntl.flock(self._fd, fcntl.LOCK_UN)
            self._fd.close()
```

### inbox_write.sh → inbox_write.py

```python
#!/usr/bin/env python3
"""inbox_write — メッセージ永続化 + cmd/task status自動更新"""
import sys
from lib.inbox_common import (
    yaml_read, yaml_atomic_write, generate_msg_id,
    overflow_protection, FileLock
)

def main():
    target = sys.argv[1]
    content = sys.argv[2]
    msg_type = sys.argv[3] if len(sys.argv) > 3 else "wake_up"
    from_agent = sys.argv[4] if len(sys.argv) > 4 else "unknown"

    inbox_path = f"queue/inbox/{target}.yaml"
    lock_path = f"{inbox_path}.lock"

    with FileLock(lock_path):
        data = yaml_read(inbox_path)
        data["messages"].append({
            "id": generate_msg_id(),
            "from": from_agent,
            "timestamp": datetime.now().astimezone().isoformat(),
            "type": msg_type,
            "content": content,
            "read": False,
        })
        data["messages"] = overflow_protection(data["messages"])
        yaml_atomic_write(inbox_path, data)

    # side effects（cmd status / task done）は別モジュールに
    update_cmd_status(content, msg_type)
    update_task_done(from_agent, content, msg_type)

if __name__ == "__main__":
    main()
```
