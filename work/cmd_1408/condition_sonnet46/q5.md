# AIエージェント7体に動画編集を丸投げしたら、思ったより「組織」だった

## はじめに

「AIに動画編集させる」といっても、実際には「AIが1人で全部やってくれる」わけじゃない。

私が作ったのは、AIエージェントが**役割分担して**動画編集を進める仕組みだ。将軍が命令を出し、家老が段取りを組み、7人の足軽が実際の作業をこなす。まるで戦国時代の軍隊みたいな構造になった。

この記事では、ドズル社の切り抜きチャンネルをAIで運営してきた約2ヶ月の試行錯誤を振り返る。

---

## なぜ「1体ではなく7体」なのか

最初は「Claude 1体に全部やらせれば楽じゃないか」と思っていた。

しかし実際にやってみると、動画編集の1サイクルは次のように分解できる：

1. **YouTube DL** → 動画のダウンロード
2. **Demucs分離** → ボーカルと BGM を分離
3. **WhisperX文字起こし** → 話者付き字幕生成
4. **シーン選定** → ハイライト箇所の選定
5. **編集指示生成** → ffmpegコマンド作成
6. **エンコード** → NVENC で高速エンコード
7. **サムネイル生成** → Gemini画像生成API

これを1体のAIに逐次処理させると、工程5で詰まった時に工程1〜4のコンテキストが消えてしまう。**コンテキスト消失問題**だ。

7体に分けることで：
- 各エージェントは1工程に集中できる
- 並列実行で処理時間が短縮できる
- 1体が失敗しても他が止まらない

## 実際にできたこと

### 1. 動画の自動切り抜き（ハイライト）

私が「この動画をハイライトにしろ」と指示すると：

1. 足軽がYouTube DL → Demucs → WhisperX を順次実行
2. 軍師が字幕を分析してタイムスタンプを提案
3. 家老が5体の足軽に並列でシーン選定させる（集合知）
4. 私が最終確認してエンコード承認

**実績**: 10〜30分の切り抜き動画を**2〜3時間**で完成（手作業だと半日かかっていた）

### 2. 漫画ショートの自動生成

ゲーム実況の名シーンを漫画風ショート動画にする試み：

1. WhisperX で字幕を取得
2. 足軽がシーンのパネル構成を提案
3. Gemini画像生成APIで漫画パネル生成（戦国武将キャラ）
4. ffmpegで音声と合成 → YouTube非公開にアップ

**実績**: 1本あたり**3〜5時間**（画像生成のやり直しを除く）

### 3. note記事の自動生成

このnote記事自体もAIが生成している（若干のアイロニー）。

## うまくいかなかったこと

### エージェントが「役割外」の仕事をしようとする

「家老が足軽の作業を直接やってしまう」問題が何度も起きた。コード上でルールを書いても、AIは「自分でやったほうが早い」と判断してしまう。

**対策**: CLAUDE.md に「実装禁止（必ず足軽に委任）」と明記。違反したら記録に残す。

### コンテキスト消失で中断

長い作業中に `/clear` が走るとそれまでの文脈が消える。足軽が途中で「何をしていたか」を忘れてしまう。

**対策**: 中間成果物を必ずファイルに保存するルールを設けた（変数だけに入れるのは禁止）。

### 画像生成のコスト暴走

Gemini画像生成APIを「いい感じになるまでやり直す」指示をしたら、3,409枚生成して**22,000円**の請求が来た。

**対策**: ガチャ上限3回ルール、1パネル試し打ちルールを設けた。

## 結論

AIエージェントで動画編集を自動化すると「楽になる」ではなく「**管理する仕事に変わる**」。

個々の作業はAIがやってくれる。しかし「どんな動画を作るか」「品質をどう保つか」「コストをどう管理するか」は人間がルールを設計し続けなければならない。

エンジニアが組織のマネジメントを理解する最短ルートは「AIエージェントを管理すること」かもしれない。
