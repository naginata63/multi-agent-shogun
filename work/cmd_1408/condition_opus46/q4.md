# Q4: ffmpegサンプルレート不一致問題（44100Hz vs 48000Hz concat）の根本対策設計

## 問題の概要

ffmpegで複数の音声ファイルをconcat（結合）する際、入力ファイル間でサンプルレートが異なると（44100Hz vs 48000Hz）、以下の問題が発生する：

1. **音声の速度変化**: 48000Hzの音声を44100Hzとして解釈すると、再生速度が約8.7%遅くなる（逆は速くなる）
2. **ピッチの変化**: サンプルレート誤認により音程が変わる
3. **concat demuxer のサイレント異常**: ffmpegのconcat demuxerは入力間のサンプルレート不一致をエラーにしない場合があり、結合後のファイルが破損していても気づかない
4. **再エンコード時の品質劣化**: リサンプリングなしで結合すると、デコーダが片方のサンプルレートで全体を解釈し、音声データが壊れる

## 原因分析

### 典型的な発生パターン

```
音源A: ゲーム内BGM・SE → 48000Hz（ゲームエンジン標準）
音源B: ボーカル分離出力（Demucs等）→ 44100Hz（音楽処理の慣例）
音源C: YouTube DL音声 → 48000Hz（YouTube標準）
音源D: SE効果音ファイル → 44100Hz（フリー素材配布の標準）
```

concat時に、これらのサンプルレートが混在する。

### ffmpegのconcat挙動

- **concat demuxer**（`-f concat -i list.txt`）: 入力ファイルのフォーマットが一致していることを前提とする。サンプルレートが異なる場合、最初のファイルのサンプルレートで全体を解釈し、後続ファイルの音声がピッチ・速度異常になる。
- **concat filter**（`-filter_complex "[0:a][1:a]concat=n=2:v=0:a=1"`）: 各入力を独立にデコードしてからPCMレベルで結合するため、サンプルレート不一致を内部で処理可能だが、**暗黙のリサンプリング品質が未定義**。

## 根本対策設計

### 対策レベル1: 入力正規化（推奨・最も安全）

結合前に全入力ファイルを統一サンプルレートにリサンプリングする。

```bash
# 正規化関数: 全入力を48000Hzに統一
normalize_audio() {
    local input="$1"
    local output="$2"
    local target_rate="${3:-48000}"
    
    # 現在のサンプルレートを取得
    local current_rate
    current_rate=$(ffprobe -v quiet -select_streams a:0 \
        -show_entries stream=sample_rate \
        -of csv=p=0 "$input")
    
    if [ "$current_rate" = "$target_rate" ]; then
        # サンプルレート一致 → コピーのみ（劣化なし）
        cp "$input" "$output"
    else
        # リサンプリング（SoXリサンプラー使用で高品質）
        ffmpeg -i "$input" \
            -af "aresample=${target_rate}:resampler=soxr:precision=28" \
            -c:a pcm_s16le \
            "$output"
    fi
}
```

**ポイント**:
- 出力サンプルレートは**48000Hz**に統一（YouTube/動画配信の事実上の標準）
- SoXリサンプラー（`resampler=soxr`）を使用（ffmpegデフォルトのswrよりも高品質）
- `precision=28`で28bit精度のリサンプリング（音楽制作品質）
- 既にターゲットレートの場合はコピーのみ（不要なリサンプリング回避）

### 対策レベル2: concat前の検証ゲート

```bash
# concat前にサンプルレート一致を検証
validate_concat_inputs() {
    local file_list="$1"
    local expected_rate=""
    local mismatches=()
    
    while IFS= read -r line; do
        [[ "$line" =~ ^file[[:space:]]+ ]] || continue
        local filepath="${line#file }"
        filepath="${filepath//\'/}"
        
        local rate
        rate=$(ffprobe -v quiet -select_streams a:0 \
            -show_entries stream=sample_rate \
            -of csv=p=0 "$filepath")
        
        if [ -z "$expected_rate" ]; then
            expected_rate="$rate"
        elif [ "$rate" != "$expected_rate" ]; then
            mismatches+=("$filepath: ${rate}Hz (expected ${expected_rate}Hz)")
        fi
    done < "$file_list"
    
    if [ ${#mismatches[@]} -gt 0 ]; then
        echo "ERROR: Sample rate mismatch detected:" >&2
        printf '  %s\n' "${mismatches[@]}" >&2
        return 1
    fi
    return 0
}
```

### 対策レベル3: パイプライン統合（恒久対策）

音声処理パイプライン全体で統一サンプルレートを強制する。

```python
# ffmpeg_ops.py への統合案
STANDARD_SAMPLE_RATE = 48000

def concat_audio(inputs: list[str], output: str):
    """安全なaudio concat — サンプルレート統一保証"""
    
    # Step 1: 全入力のサンプルレートを検査
    rates = {}
    for f in inputs:
        rate = get_sample_rate(f)  # ffprobe wrapper
        rates[f] = rate
    
    unique_rates = set(rates.values())
    
    if len(unique_rates) == 1 and STANDARD_SAMPLE_RATE in unique_rates:
        # 全て標準レート → そのままconcat（高速パス）
        _concat_demuxer(inputs, output)
    else:
        # サンプルレート不一致 → 正規化してからconcat
        normalized = []
        for f in inputs:
            if rates[f] != STANDARD_SAMPLE_RATE:
                norm_path = f + ".norm.wav"
                _resample(f, norm_path, STANDARD_SAMPLE_RATE)
                normalized.append(norm_path)
            else:
                normalized.append(f)
        
        _concat_demuxer(normalized, output)
        
        # 一時ファイル削除
        for f in normalized:
            if f.endswith(".norm.wav"):
                os.unlink(f)

def get_sample_rate(filepath: str) -> int:
    """ffprobeでサンプルレートを取得"""
    result = subprocess.run(
        ["ffprobe", "-v", "quiet", "-select_streams", "a:0",
         "-show_entries", "stream=sample_rate",
         "-of", "csv=p=0", filepath],
        capture_output=True, text=True
    )
    return int(result.stdout.strip())
```

## 48000Hz vs 44100Hz の選択理由

**48000Hzを標準とすべき理由**:
- YouTube/動画配信の事実上の標準（YouTube推奨: 48kHz）
- ゲーム内SE・BGMが48kHzで収録されるケースが多い
- 44100Hz→48000Hzのアップサンプリングは情報損失が軽微
- 逆（48000→44100）はエイリアシング対策が必要で品質面で不利

**44100Hzを避ける理由**:
- CD規格由来の歴史的仕様（動画制作では非標準）
- 48000→44100変換は整数比でない（割り切れない）ため、高品質リサンプラーが必須

## 結論

ffmpegサンプルレート不一致問題の根本対策は「concat前の入力正規化（48000Hz統一）」+「検証ゲート」の2段構え。最も重要なのはconcat前に全入力のサンプルレートを検査し、不一致を検出した時点でリサンプリングを挟むこと。これにより、音声速度・ピッチ異常を完全に防止できる。
