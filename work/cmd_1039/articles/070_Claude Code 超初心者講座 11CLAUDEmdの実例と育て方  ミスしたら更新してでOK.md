# プロジェクトルール
- 言語: 日本語で応答する
- ファイル保存先: デスクトップ/作業フォルダ/
- 命名: YYYY-MM-DD形式で日付をつける
# プロジェクト概要
- Webサイト制作案件（クライアント: A社）
- フレームワーク: Next.js
# 作業ルール
- コミットメッセージは日本語で書く
- CSSはTailwindを使う。独自CSSは書かない
- 画像は public/images/ に保存する
- テストは変更のたびに実行する
- エラーが出たら修正してからコミットする

ポイントは「判断に迷うこと」を書いておくこと。 Claude Codeが自分で判断せず、こちらのルール
に従ってくれます。
ただし、やりすぎ注意です。 50行を超えてくると、Claude Codeが全部を把握しきれなくなる。 指示

階層構造で整理する
フォルダごとのCLAUDE.md階層構造と継承
実はCLAUDE.mdはフォルダごとに置けます。
ルートのCLAUDE.mdは全体に適用されます。 サブフォルダのCLAUDE.mdは、そのフォルダ内だけに
適用。
面白いのは、親のルールを子が上書きできること。
たとえばルートに「日本語で応答して」と書いておいて、
プロジェクト/
├── CLAUDE.md          ← 全体ルール（日本語で応答）
├── src/
│   └── CLAUDE.md      ← コード固有のルール
└── docs/
    └── CLAUDE.md      ← ドキュメント固有のルール

育て方のコツ — 「ミスしたら更新して」
ミス発生→CLAUDE.mdに追加→同じミスは起きない のサイクル
ここが一番伝えたいところです。
CLAUDE.mdは、最初から完璧に書こうとしなくていい。 むしろ、使いながら育てていくものです。
やり方はシンプル。
Claude Codeがミスをしたら、こう言うだけ。 「今のルール、CLAUDE.mdに追加しておいて」
するとClaude Code自身がCLAUDE.mdを編集してくれます。
日付フォーマットを間違えた → 「日付のルール追加して」
保存先を間違えた → 「保存先のルール追加して」
敬語で書いてほしいのにタメ口だった → 「トーンのルール追加して」
一度ルールを追加すれば、同じミスは繰り返さない。 使えば使うほど、自分好みのClaude Codeに育
っていきます。
