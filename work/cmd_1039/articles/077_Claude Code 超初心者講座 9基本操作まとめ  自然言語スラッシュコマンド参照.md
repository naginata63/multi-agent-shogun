自然言語でClaude Codeに話しかけるフロー図
一番シンプルで、一番使うやつ。 日本語で「〜して」と頼むだけです。
「このファイル探して」
「中身を教えて」
「CSVの売上を月別に集計して」
#5〜#8 でやってきたことは、全部これでしたよね。
しかも複雑な指示もいけます。 「このフォルダのPDFを全部読んで、要点だけまとめて、CSVにして
保存して」 こんな一文でも、ちゃんと順番に処理してくれる。
曖昧な表現でも大丈夫です。 「なんかいい感じに整えて」みたいな雑な頼み方でも、文脈から意図を
汲んでくれます。
正直、9割の作業はこれで完結するかなと。
スラッシュコマンド — まず覚える5つ
まず覚える5つのスラッシュコマンド一覧
自然言語で頼めないことが一部あります。 Claude Code自体の設定やリセットです。

<!-- page 3 -->

そういうときに使うのがスラッシュコマンド。 入力欄で
/
を打つと候補が出てきます。
全部覚える必要はないです。まずはこの5つだけ。
特に
/clear
はよく使います。 前の作業の文脈が残っていると、意図しない動きをすることがあるので。 新しい作
業に入るときは
/clear
でリフレッシュする癖をつけておくと安心ですね。
@参照 — ファイルを直接指定する
@参照でファイルをピンポイント指定する図
/help   → 困ったらこれ。使い方の一覧が出る
/clear  → 会話をリセット。新しい作業を始めるときに
/model  → モデルやがんばり度を変更（#4で紹介済み）
/doctor → 動作がおかしいときの診断（#3で使った）
/init   → CLAUDE.mdの生成（#10で詳しく解説予定）

<!-- page 4 -->

自然言語で「あのファイル」と言っても伝わることが多い。 でも「このファイル」がどれなのか、確
実に伝えたいときもありますよね。
そんなときは
@
でファイルパスを指定します。
「@デスクトップ/報告書.pdf を要約して」
こう書くと、対象ファイルをピンポイントで指定できます。 自然言語より正確で、間違いが起きにく
い。
フォルダ全体の指定もできます。
「@書類/2025年度/ のファイルを一覧にして」
長いパスを手打ちするのが面倒に感じるかもしれません。 でもTabキーを押すと補完してくれるの
で、数文字打てば候補が出ます。
まぁ、自然言語でも十分伝わるので、「確実にこのファイル」と指定したいときだけ使う感じですね。
まとめ
今回整理した操作方法は、この3つ。
普段は自然言語。設定まわりはスラッシュ。ファイル指定は@。 この使い分けさえわかっていれば、
操作で迷うことはないはずです。
1. 自然言語    → 普段の操作はぜんぶこれ
2. スラッシュ   → 設定やリセットに使う（5つだけ）
3. @参照      → ファイルを確実に指定したいとき
