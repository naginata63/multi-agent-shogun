Claude Codeが最初から持っている能力の一覧図
MCPを追加する前に、Claude Codeが最初から持っている能力を確認しましょう。
「ファイル操作MCP」「Web検索MCP」を入れましょう、という情報を見かけることがありますが、
これらは最初からできます。
わざわざMCPを追加する必要はありません。
おすすめMCPサーバー
おすすめMCPサーバー — PlaywrightとMemory
最初からある機能ではできないことを補う。 それがMCPの役割です。
おすすめはこの2つ。
ファイル操作  → 読み書き・検索・編集、全部できる
Web検索      → ネットで最新情報を調べられる
URL読み込み  → Webページの中身を取得できる

<!-- page 3 -->

1. Playwright（ブラウザ操作）
前回紹介済み。Webブラウザを自動操作できるようになります。
URLの中身を読むだけならClaude Code単体でもできます。 でも「ボタンを押す」「フォームに入力す
る」「スクリーンショットを撮る」みたいな操作が必要な場面ではPlaywrightが必要。
これだけで使えるようになります。
2. Memory（記憶の持ち越し）
Claude Codeは、セッション（＝Claude Codeを開いてから閉じるまで）を終了すると会話の内容を忘
れます。
CLAUDE.mdに書けば方針は伝えられるけど、 会話の中で得た細かい知識は消えてしまう。
Memory MCPを入れると、PCを再起動しても情報を覚えていてくれるようになります。
たとえば、今日「このプロジェクトの方針を覚えておいて」と伝える。 翌日PCを起動し直して
Claude Codeを開いて「前回の方針は？」と聞くと、ちゃんと思い出してくれる。
普通なら閉じた時点で全部消えるところを、ずっと覚えていてくれる。 それがMemory MCPです。
まずは公式のMemory MCPを入れるのがおすすめ。
ちなみに僕は「claude-mem」という非公式のプラグインを使っています。 公式より高機能で、会話の
内容を自動で記録してくれる。 ただし非公式なので、使う場合は自己責任で。 最初は公式のMemory
MCPから始めてみてください。

MCPの選び方 — 「毎週使うサービス」で決める
MCPの選び方 — 必要なものだけ厳選する漏斗フィルター図
「他にもMCPを入れたい」と思ったら、こう考えてみてください。
Claude Code単体でできないことか？
ファイル操作やWeb検索はすでにできる。 MCPが必要なのは、外部サービスとの連携だけです。
毎週1回以上使うサービスか？
月1回しか使わないサービスのMCPは入れない方がいい。 MCPを入れすぎると、Claude Codeが「ど
のツールを使うか」で迷い始めます。
使いたいMCPが見つかったら、追加方法はいつもの通り。
Claude Codeに頼めば、設定を全部やってくれます。
必要なものだけ、シンプルに。 MCPは少ないほうがうまくいきます。
