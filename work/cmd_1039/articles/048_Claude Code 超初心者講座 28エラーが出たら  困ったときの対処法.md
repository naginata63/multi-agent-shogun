エラーが出たとき、まずやることはひとつ。
エラーメッセージをそのままコピペして、Claude Codeに貼る。
これだけです。
Claude Codeがエラーの原因を分析して、直してくれます。
「エラーメッセージの意味を自分で理解しなきゃ」と思うかもしれませんが、その必要はありませ
ん。赤い文字をそのままコピーして貼るだけ。
まぁ、翻訳サービスみたいなものかなと。英語のエラー文を読み解く作業は、全部Claude Codeに任
せてしまいましょう。
よくあるトラブル3選
よくあるトラブル3つ — 画面が真っ白、ボタンが反応しない、データが表示されない
アプリ開発でよく出くわすトラブルと、その伝え方をまとめておきます。
1. 画面が真っ白になる
コードに打ち間違いがあると、画面が何も表示されなくなることがあります。
Claude Codeにはこう伝えましょう。
このエラーが出ました。直してください。
TypeError: Cannot read properties of undefined (reading 'map')

<!-- page 3 -->

真っ白のときは、たいてい1文字の打ち間違いだったりします。Claude Codeにとっては見つけやすい
問題です。
2. ボタンを押しても反応しない
ボタンはあるのにクリックしても何も起きない。ボタンと動作の結びつけがうまくいっていないこと
が多いです。
「何をしたら、どうなってほしいか」を書くのがポイントです。
3. データが表示されない
保存はできたのに、一覧に出てこない。内部の設定がずれていることがよくあります。
3つとも共通しているのは、「何が起きたか」と「どうなってほしいか」を伝えるだけということ。原
因を推測する必要はありません。
自分で解決する力をつけるには
DevToolsの使い方 — F12でConsoleを開き、エラーメッセージを見つけてClaude Codeに伝える
画面が真っ白です。何も表示されません。
原因を調べて直してください。
「保存」ボタンを押しても何も起きません。
クリックしたらデータが保存されるようにしてください。
データを入力して保存したのに、一覧に表示されません。
保存する部分と表示する部分を確認して直してください。

<!-- page 4 -->

全部Claude Codeに任せられるとはいえ、ちょっとだけ自分でできることを増やしておくと対応が早
くなります。
それがDevTools（デベロッパーツール） です。ブラウザに最初から付いている「裏側を覗く機能」で
す。
使い方はシンプル。
ブラウザで
F12
キーを押す（Macなら
Cmd + Option + I
）
上部のタブから「Console（コンソール）」を選ぶ — ここにエラーメッセージが表示
されます
赤い文字があったらコピーする
Claude Codeに貼り付ける
「画面が真っ白です」だけより、この赤い文字（エラーメッセージ）も一緒に渡すほうが、Claude
Codeの対応がグンと正確になります。
完全に自力で直す必要はないんです。
「エラーの場所を見つけて、Claude Codeに伝える」。このスキルだけで十分。DevToolsは、エラーの
居場所を教えてくれる地図みたいなものですね。
まとめ
エラーは怖くありません。Claude Codeにコピペすれば直してくれます。
よくあるトラブルのパターンを知っておくと、伝え方が的確になって対応がぐっと早くなります。
DevToolsも使えるようになれば、さらにスムーズです。
