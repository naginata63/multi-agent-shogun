何が作れる？ — Webアプリと日常を楽にする仕組み
「アプリ開発」って聞くと構えちゃいますよね。
でも実は、Claude Codeで作れるものの幅は意外と広いです。 大きく分けると2つ。
1. Webアプリ（ブラウザで動くもの）
Webアプリの例 — ToDoリストや家計簿がブラウザで動く
ブラウザで動くツールやサイトです。
たとえば、こんなものが作れます。
やることを管理するToDoリスト
お金の出入りを記録する家計簿
日報を入力する業務ツール
プログラミングの部分は全部Claude Codeがやってくれる。 あなたが書く必要はありません。
2. 日常を楽にする仕組み
毎日やってる繰り返しの作業、ありませんか？

<!-- page 3 -->

たとえば、こんなの。
フォルダの中の写真100枚を、日付順にまとめて名前変更
毎朝の売上データを自動で集計してまとめる
ダウンロードフォルダのファイルを種類別に自動で振り分ける
こういう「手作業だと面倒なこと」を、Claude Codeに頼めば自動でやってくれる仕組みが作れます。
ここで大事なポイント。
コードを書くのはClaude Code。 あなたがやるのは「何を作りたいか」を伝えることだけです。
...まぁ、「伝えるだけ」って言っても、その伝え方にコツはあるんですけどね。 そこは後で触れます。
開発の流れ — 4ステップ
開発の流れ — Step1 言葉にする → Step2 設計する → Step3 コードを書く → Step4 動かす・直す
「で、実際どうやるの？」ってなりますよね。
アイデアから動くものができるまで、4ステップです。
Step 1: やりたいことを言葉にする
「こういうの欲しいんだけど」をClaude Codeに伝えます。 完璧な文章じゃなくていい。 箇条書きで
も、口語でも大丈夫です。
Step 2: Claudeが設計する

<!-- page 4 -->

Planモード（#16で紹介しましたね）を使えば、Claudeがまず設計案を出してくれます。 「こういう
構成でいきますか？」って提案してくれるので、OKかどうか答えるだけ。
Step 3: Claudeがコードを書く
設計が決まったら、Claudeがコードを書き始めます。 ファイルの作成、必要なソフトの準備、全部や
ってくれる。 あなたはターミナルの承認ボタンを押すだけです。
Step 4: 動かす → 直してもらう → 完成
書いたコードを実行して、動くか確認します。 「ここ、ちょっと違うな」と思ったら、そのまま伝え
る。 Claude Codeが修正してくれます。
この「動かす→直す」のループを何回か繰り返せば完成。
ここまで読んで気づいた方もいるかもしれません。 CLAUDE.md、Hooks、MCP...これまで学んできた
ことが全部活きてきます。
設定ファイルで好みを伝えて、Hooksで自動チェックして、MCPで外部ツールと連携する。 #1から
#21まで積み上げてきたものが、ここで繋がるわけです。
始める前に必要なもの
必要なもの3つ — 必要なソフト、ぼんやりしたアイデア、Claude Code
「じゃあ何を準備すればいいの？」
必要なものはシンプルです。
必要なソフト（作るものによって変わる）
→ 何が必要かはClaude Codeが教えてくれるし、インストールも手伝ってくれます
「こんなの欲しい」というぼんやりしたアイデア
→ 完璧な設計書は不要。曖昧でOK

Claude Code（セットアップ済み）
→ #1〜#3あたりを参考にどうぞ
正直、一番大事なのは2つ目です。
「何を作りたいか」がぼんやりでもあれば、Claude Codeが具体化してくれる。 「家計簿っぽいやつ」
でも「毎日の作業を楽にしたい」でもいい。
逆に言うと、「何か作りたいけど何を作るかは決まってない」だと厳しい。 そこだけは自分で考える必
要があります。
...まぁ、「何作ろう？」ってClaude Codeに相談するのもアリですけどね。
まとめ
Claude Codeは「コードが書ける相棒」です。
プログラミング経験がなくても、作りたいものがあれば一緒に作れる。 設計もコーディングもClaude
Codeがやってくれる。 あなたは「何を作りたいか」と「これでOK」を伝えるだけ。
次回 #23 からは、実際に手を動かしてWebアプリを作っていきます。 いよいよ実践編。ぜひ、ご自身
でも一緒に手を動かしてみてください！
