Samsungの情報漏洩（2023年）

Samsung（韓国の大手電子機器メーカー）の半導体部門で、従業員が3回にわたってChatGPTに機密 情報を入力してしまった事件。

不具合のあるソースコード（プログラムの設計図）

設備欠陥を検出するプログラム

社内会議の録音を文字起こししたもの

一度AIに送ったデータは、OpenAI（ChatGPTを作っている会社）のサーバーに保存されて回収不可 能。Samsungは全社でChatGPT使用禁止にした。（Forbes Japan）

AIを使った詐欺で37億円の被害（2024年）

Arup（イギリスの建築設計事務所）の香港支社で、財務担当者がビデオ会議でCFO（最高財務責任者） から送金を指示された。

実はそのビデオ会議の参加者全員がAIで作られた偽物（ディープフェイク）だった。

被害額は約37億円。相手の顔も声も本物にしか見えなかった。（日本経済新聞）

「シャドウAI」による被害は平均1億円増（2025年、IBM調査）

「シャドウAI」とは、会社が許可していないAIツールを従業員が勝手に使うこと。

IBMの調査によると、従業員の63%が機密データ（ソースコード、顧客情報など）をAIに貼り付けてい る。 シャドウAIが原因の情報漏洩は、通常の漏洩より平均で約1億円コストが高い。（IBM Cost of a Data Breach 2025）

AIエージェントSNSから150万件のAPIキーが流出（2026年2月）

AIエージェント専用SNS「Moltbook」から、150万件を超えるAPIキーや認証情報が流出した。セキュ リティ企業Wizが発見して公表した事件。

APIキーとは「AIサービスを使うための合鍵」みたいなもの。これが漏れると、他人があなたのアカウ ントでAIを使い放題になる。つまり、知らない間に数十万〜数千万円の利用料が請求される可能性が ある。（Forbes Japan）

AIエージェントが騙されて7.5億円の被害（2026年報告）

製造業の調達支援AIエージェントが、3週間にわたって攻撃者に巧みに「操作」された事件。

AIが「50万ドル以下なら人間の承認なしで発注していい」と思い込まされて、最終的に500万ドル（約 7.5億円）分の偽の注文を実行してしまった。

AIに権限を渡しすぎると、こういうことが起きる。（ZDNET Japan）

88%の組織がAIエージェント関連のセキュリティ被害を経験（2026年調査）

セキュリティ企業Graviteeが919社を対象に調査したところ、88%がAIエージェント関連のインシデン トを経験していた。さらに45.6%がAPIキーを複数サービスで使い回していた。

これは企業の話だけど、個人でも同じ。1つのAPIキーが漏れたら、そこから芋づる式にやられる。（タ オリス人機和総研）

IPA「情報セキュリティ10大脅威 2026」で3位にランクイン

IPA（日本の情報処理推進機構）が発表した2026年版のセキュリティ脅威ランキングで、「AIの利用を めぐるサイバーリスク」が初選出で3位に入った。去年まで存在しなかった脅威が、いきなりトップ 3。それくらいAIセキュリティの問題は急速に深刻化している。（ScanNetSecurity）

最近のAIセキュリティ事故の年表イメージ

Anthropicが言ってること

Claudeを作っているAnthropic（アンソロピック）社は、セキュリティについて公式にこう言ってい る。

「これらの保護はリスクを大幅に軽減しますが、すべての攻撃に完全に免疫のあるシステ

ムは存在しません。AIツール使用時は常に良いセキュリティ習慣を維持してください。」

つまり「AIツール側でも対策はしてるけど、完璧じゃない。ユーザー側でも気をつけてね」というこ と。

Claude Code（AIでプログラムを書くツール）の公式ドキュメントには、具体的な推奨事項がある。

AIが提案するコマンドを、実行する前に確認する

信頼できないコンテンツをAIに直接読ませない

重要なファイルへの変更は必ず検証する

AIにすべての権限を渡さない（必要な分だけ）

要は 「AIが言ったから正しい」と思わない こと。（Claude Code セキュリティドキュメント）

さらに2026年2月20日、Anthropicは「Claude Code Security」というセキュリティスキャン機能を発 表した。AIがコードを人間のセキュリティ研究者のように読んで、数十年見逃されていた脆弱性を500 件以上発見したという。（Anthropic公式ブログ）

この発表でサイバーセキュリティ関連株が軒並み暴落したくらい、インパクトがあった。

つまりAnthropic自身が「AIのセキュリティリスクは深刻。だから防御ツールも作った」と行動で示し ている。AIを作っている会社がここまで本気で対策しているということは、使う側もそれなりに気を つけるべきだということ。

じゃあ非エンジニアは何をすればいいのか

ここからが本題。

全部やる必要はない。自分に当てはまるレベルだけやればOK。

レベル1：今すぐやろう（5分で終わる）

AIにパスワードや個人情報を貼り付けない

Samsung事件の直接原因がこれ。 ChatGPTでもClaudeでも、AIとの会話に機密情報を入れたら、そ の時点でコントロールを失う。

「ちょっとこのコード見て」で社内の設計図が外部サーバーに保存される。

ルール：AIに渡す前に「これ、人に見せて大丈夫な内容か？」と1秒考える。

2段階認証をオンにする

2段階認証は「ログイン時にスマホの確認が必要になる仕組み」。

AIとは直接関係なさそうに見えるけど、AIツールのアカウントが乗っ取られると、そこから芋づる式 に情報が漏れる。

Google、Apple、AIツールのアカウント、全部オンにする。5分で終わる。

ブラウザにパスワードを保存しない

私が「842件見えた」のはまさにこれ。 ブラウザに保存されたパスワードは、PCで動くAIから見える 可能性がある。

パスワード管理アプリ（1PasswordやBitwardenなど）に移行する。 移行は面倒だけど、842件見えた 恐怖に比べたら安いもの。

レベル2：AIを日常的に使うならやっておこう（30分で終わる）

AIに渡すファイルに個人情報がないかチェックする

「このファイルを分析して」とAIに渡す前に、中身を確認。 顧客名、電話番号、住所が入ってたら、 その部分を消してから渡す。

AIの権限設定を確認する

Claude Codeは初期設定で「読み取り専用」になっている。 つまり、許可しない限りファイルを書き 換えたりコマンドを実行したりできない。

この初期設定を変えないこと。 必要な時だけ、必要な分だけ許可する。

作業用のフォルダを分ける

AIに触らせるファイルと、触らせたくないファイルを物理的に分ける。 「この中だけ自由にしていい よ」とフォルダを区切る。

私はAIが作業する専用フォルダを作って、そこ以外は触れないようにしている。

レベル1/2/3の階段図：5分 → 30分 → 1時間

レベル3：AIを本気で自動化に使うなら（1時間）

AIが外部にデータを送れる状態になっていないか確認する

AIエージェント（自動で動くAI）を使っている場合、AIがインターネット経由でデータを外に送れる 状態になっていないか確認する。

Anthropicの公式ドキュメントでも、ネットからコンテンツを取得するコマンド（ウェブページを取っ てくる命令など）は初期状態でブロックされている。この設定を解除する時は慎重に。

AIが何をやったかを定期的に確認する

自動で動かしている場合、AIが裏で何をしたかのログ（記録）を見る習慣をつける。

「知らない間にAIが変なファイルを作ってた」は実際に起こる。

バックアップを取る

AIが間違えてファイルを消しても大丈夫なように。 これはAIに限った話じゃないけど、AIを使い始め ると「AIが勝手にやった」系のミスが増えるので、より重要になる。

よくある「大丈夫でしょ」が一番危ない

「ChatGPTしか使ってないから大丈夫」 → ChatGPTでも入力した情報はサーバーに送られる。 Samsung事件がまさにChatGPT。

「ファイルは渡してないから大丈夫」 → Claude CodeのようにPC上で動くAIは、ファイルを渡さな くても読める場合がある。

「パスワード変えたから大丈夫」 → 管理方法（ブラウザ保存）が同じなら、また見られる。

「AIが提案してるんだから安全でしょ」 → Anthropic公式が「完璧なシステムは存在しない」と言って る。AIの提案を鵜呑みにしない。

私がやった対策リスト

参考までに、私が実際にやったこと。

ブラウザの保存パスワードを全削除 → 1Passwordに移行

2段階認証を全アカウントでオン

AIが触れるフォルダを限定

Claude Codeの権限は初期設定（読み取り専用）のまま運用

自動化タスクのログを毎日チェック

定期的にバックアップ

レベル1（5分でできること）だけで、8割の問題は防げる。

AIに機密情報を貼り付けない

2段階認証をオンにする

ブラウザにパスワードを保存しない

この3つだけ。5分で終わる。

「エンジニアじゃないからセキュリティはわからない」は、もう言い訳にならない時代。 AIは便利。 でも「鍵をかけずに家を空ける」のはダメ。

まずはレベル1から。今すぐ。
