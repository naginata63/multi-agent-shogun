ゲーム機の拡張スロットの比喩でMCPを説明する図
MCP = Model Context Protocol。
難しそうに聞こえますが、イメージはゲーム機の拡張スロットです。
本体だけでも遊べる。でも周辺機器をつなぐと、もっと楽しくなる。
Claude Codeもそれと同じです。
「Claude Codeの能力を拡張するプラグイン」と思えばOKです。
MCPサーバーという外部プログラムをClaude Codeに接続すると、Claudeが新しいツールを使えるよ
うになる。それだけの仕組みです。
MCPサーバーの追加方法 — AIに頼むだけ
本体のまま → ファイル操作、テキスト処理、コード生成
MCP追加後  → Web閲覧、DB操作、外部サービス連携

AIにMCPサーバーの追加を頼む図
MCPの追加も、Claude Codeに日本語で頼めばOKです。
これだけ。Claudeが必要な設定を全部やってくれます。
「今どんなMCPが入ってる？」と聞けば一覧を見せてくれるし、 「○○のMCPを削除して」と言えば
消してくれる。
自分でコマンドを打つ必要はありません。
最初の1個 — Playwright MCP（ブラウザ操作）
Playwright MCPでブラウザ操作を実現するフロー図
MCPサーバーはたくさんありますが、最初の1個にはPlaywright MCPをおすすめします。
なぜか。Webブラウザを自動操作できるようになるからです。
「このURLを開いて内容を要約して」 「このページのタイトルとリンクを一覧にして」
こういう指示が、そのまま通るようになります。
追加はClaude Codeにこう頼むだけ。
