/model画面の操作：上下でモデル選択、左右でがんばり度を切り替え
まず一番大事なやつから。
Claude Codeの画面で
/model
と入力してEnter。 するとモデル選択画面が出てきます。
ここでの操作はこれだけ。
上下の矢印キー → モデルの切り替え
左右の矢印キー → がんばり度の切り替え（設定③で説明します）
モデルは 「Default」 を選んでください。 Proプランなら「Default（Opus 4.6）」と表示されるはず。
これが一番賢いモデルです。
正直、体感でわかるくらい回答の質が変わります。 同じ質問をしても、深く・正確に答えてくれるよ
うになる。
設定②
Shift + Tab
で動作モードを切り替える

<!-- page 3 -->

通常モードとThinkingモードの比較：パッと回答→じっくり思考
Claude Codeは「どこまで勝手にやっていいか」を切り替えられます。
Shift + Tab
を押すたびにモードが変わります。
画面の下に今のモードが表示されるので、すぐわかります。
初期状態だと、ファイルを1つ書き換えるたびに「これ実行していい？」と聞かれます。 安全だけど、
正直めちゃくちゃ面倒。
おすすめは bypass permission（全おまかせ）。
Shift + Tab
を何回か押して切り替えてください。
「このフォルダ整理して」「この記事書いて」みたいに丸投げしたいとき、 いちいち確認なしで最後ま
でやりきってくれます。
慣れてきたら場面で使い分ければいいですが、 最初は bypass でガンガン動かすのが一番楽しいで
す。
(表示なし)        : 毎回「実行していい？」と聞いてくる（初期状態）
accept edits on   : ファイルの編集は確認なしでOK
plan mode on      : まず計画を立てて見せてくれる
bypass permission : 全部おまかせ。確認なしでガンガン進める

<!-- page 4 -->

ただし1つだけ注意。 bypass は 本当に何でもやっちゃいます。 ファイルの削除も、フォルダの移動
も、確認なしで実行します。
大事なファイルがあるフォルダで使うときは、 念のためバックアップを取ってからにしてください。
心配なら accept edits on から始めるのもアリです。
設定③ Effort Level（がんばり度）
Effort Level 3段階：Low（サクッと）→Medium（バランス）→High（じっくり丁寧）
設定①の
/model
画面、覚えてますか？ 上下でモデルを選ぶ画面で、左右の矢印キーも使えましたよね。
あれが「がんばり度（Effort Level）」の設定です。
3段階あって、使い分けるとかなり快適になります。
/model
high  : じっくり丁寧に考える（重い作業向け）
medium: バランス型（普段使いに最適）
low   : サクッと即答（簡単な質問向け）

<!-- page 5 -->

を開いて、左右キーで切り替えるだけ。
迷ったら medium でOK。
普段はmediumにしておいて、 「ガッツリ作業してほしい」ときだけhighに上げる。
逆に「今日の天気は？」みたいな軽い質問ならlowで十分。 このへんは使いながら感覚を掴んでいけば
大丈夫です。
