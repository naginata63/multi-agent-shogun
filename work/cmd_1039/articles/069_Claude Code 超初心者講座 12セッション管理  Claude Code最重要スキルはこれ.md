コンテキストウィンドウをバケツに例えた概念図
Claudeが一度に覚えていられる情報量には、限りがあります。
会話を続けるほど、その「記憶スペース」が埋まっていく。 イメージとしては、バケツに水を注いで
る感じ。
あふれたら、古い情報から消えていきます。
つまり、長い会話を続けるほど回答の質が落ちる。 最初の指示を忘れたり、同じ提案を繰り返した
り。 「おかしくなった」の正体は、だいたいこれです。
今どれくらい使ってるかは
/context
コマンドで確認できます。

3つの軌道修正テクニック
「あ、方向が違うな」と思ったら、早めに修正するのが鉄則です。 放っておくとコンテキストがどん
どん汚れていくので。
Escキーで止める
Claudeが長々と作業を始めて、「いや、そうじゃない」と思ったら。 Escキーを押せば即中断できま
す。
途中まで出力された内容はコンテキストに残りますが、 暴走させるより遥かにマシ。「あれ？」と思
った瞬間にEsc。これ大事です。
自然言語で方向転換
中断したあとは、普通に日本語で伝えればOK。
「ちょっと待って、そうじゃなくて〇〇して」
これだけで軌道修正できます。 機械的なコマンドは不要。普段の会話と同じ感覚ですね。
1タスク1セッションの原則
一番やりがちなNGパターンがこれ。
1つのセッションで何個もタスクを詰め込む。
「ファイル整理して、ついでにCSV集計して、あとメール文も作って」
気持ちはわかります。でもこれをやると、 タスクAの情報がタスクBに混ざって精度が落ちる。

1つのタスクが終わったら
/clear
してから次へ。 これだけで体感の精度がかなり変わります。
「調子悪いな」と思ったら
包丁を研ぎ直すように/clearでリセットする図
Claudeの反応がイマイチなとき、まずやることは1つ。
/context
で確認する。
80%を超えてたら、もう限界だと思ってください。 そこから粘っても良くなることはほぼないです。
素直に
/clear
して最初からやり直す方が早い。
「でも、ここまでの会話がもったいない...」
わかります。でも考えてみてください。 コンテキストが汚れた状態で頑張るのは、泥水で洗濯してる
ようなもの。 やればやるほど汚れが広がるだけなんですよね。
目安としては、2回修正してダメなら
/clear
