Claude Maxプラン月$100。マルチエージェントを全員opus4.6で回したら、1時間もたずにレートリミ ットに到達。

$200にアップグレードしてもリミット到達して作業が止まる。この絶望、伝わるだろうか。

ここから試行錯誤して、最終的に将軍システムの構成としては「将軍・家老はopus4.6、足軽はGLM- 5」という構成にたどり着いた。今はレートリミットに怯えずにAIを回せている。

ちなみに将軍システムとは

sasuu｜非エンジニアのAI活用術 2026年2月19日 12:00

Claude Codeの複数エージェントを戦国時代の軍制で動かす仕組み。将軍（指揮官）→ 家老（中間管 理職）→ 足軽（作業員）の三階層で、合計11体のAIがチームとして動く。詳しくは以下の記事で書い たので割愛。

将軍システムの三階層構造：将軍→家老→足軽の指揮系統

全員opus4.6で爆死した話

2チーム編成で運用していた。

・チームA：アプリ開発（フロント・バック・テストを並列で） ・チームB：リサーチ＆ドキュメント 作成（海外情報の調査・翻訳・マーケティング施策のアイデア出し）

最初は全員opus4.6で動かしていた。最高性能のモデルを全員に使わせれば、最高の成果が出るだろ う。そう思ってた。

甘かった。

Claude Maxプラン$100/月で始めて、1時間もしないうちにレートリミットに到達。画面に出てきたの は「アップグレードしてね」というエラーメッセージ。選択肢は2つ。

・プランをアップグレードする ・リミットが解除されるまで待つ

11体のAIが同時にopus4.6でトークンを消費したら、そりゃそうなる。冷静に考えれば当たり前なんだ けど、やってみるまで気づかなかった。

11体全員opus4.6でレートリミットに到達して爆死

$200にアップグレードしてもダメだった

「$100が足りないなら$200にすればいい」

脳死で課金した。

結果、多少はマシになった。でもClaude Maxには5時間ごとのレートリミットがある。$200プランで も、3時間くらいで上限に達して、2時間待たされる。

夜中も寝てる間にエージェントを回していたりするから、なおさらキツい。「朝起きたらリミットで止 まってた」なんてことがザラにあった。

月$200払って、1日に何回も作業が止まる。これはストレスというより、もはや怒り。いや膝から崩れ 落ちるくらい悲しんだ。

$100→$200にアップグレードしてもレートリミットの壁は越えられない

sonnetに切り替えてもダメだった

「全員opus4.6が贅沢すぎるなら、部下たちをsonnetに下げればいいのでは？」

やってみた。

結果、5時間のレートリミットにはギリギリ到達しなくなった。「お、いけるか？」と思った。

甘かった。今度は1週間のレートリミットが早い段階で厳しくなった。

Claude Maxには5時間ごとのリミットとは別に、週単位のリミットがある。5時間のリミットは待てば 数時間でリセットされる。でも週リミットに達すると、リセットまで2〜3日待たされる。

2〜3日。何もできない。エージェント全員が止まって、ただ待つだけ。

sonnetに下げてトークン単価は下がったけど、11体が回し続ける総量は変わらない。週リミットの枠 を数日で食い尽くしてしまう。

つまり「Claudeの中でモデルを下げる」は焼け石に水。根本的に、Claude側のトークン消費自体を減 らさないとダメだった。

sonnetでも週リミットに到達：2〜3日の強制停止

たどり着いた答え：足軽をGLM-5にする

Claudeのリミットを食うのは、Claudeを使うエージェントだけ。

だったら、Claudeを使わないエージェントを増やせばいい。

将軍と家老はopus4.6のまま。判断・指揮・レビューをする側は妥協しない。でも足軽9体は、判断力 より「指示通りに正確に作業する」能力があればいい。

そこで足軽たちをGLM-5に切り替えた。Z.AI（智谱AI）が提供するモデルで、Claude Codeでも動く形 式になっているから設定を切り替えるだけで使える。

現在の構成はこう。

・将軍（1体）：opus4.6 — 全体指揮・意思決定 ・家老（1体）：opus4.6 — タスク分解・足軽への指 示・品質チェック ・足軽（9体）：GLM-5 — コーディング・リサーチ・翻訳・ドキュメント作成

11体中9体がClaudeのリミット枠を消費しなくなった。

最終構成：将軍・家老はopus4.6、足軽9体はGLM-5

なぜGLM-5なのか

正直、足軽を動かせるモデルは他にもある。でもGLM-5を選んだ理由がある。

・Claude Codeで動く。Claude Codeでも動く形式になっているので、設定を切り替えるだけでエー ジェントとしてそのまま使える ・安い。最安プランは年間$36。月$3。9体の足軽を月$3で回せると 思ったけど、リミットにいってしまった ・GLM-4.7からの進化を体感している。以前GLM-4.7を足軽 に使っていて「まぁ悪くないな」という印象だったところ、GLM-5にしたら結構良くなった

最初は年間$36の最安プランで使っていた。「月$3で9体動かせるとか最高じゃん」と思ってた。

でもGLM側にもリミットがあった。足軽9体で回し続けるとさすがに年間$36プランではトークン容量 が足りなくてリミットに到達。またか。

もうリミットに怯える生活に疲れ果てて、勢いで年間$360のプランに切り替えた。claude proプラン と比べるとトークン容量は60倍。 claude maxプラン$200と比べても3倍ある。流石にやりすぎた。。。

結果、5時間のレートリミットに対して20%程度しか使わなくなった。余裕ありすぎ。正直ちょっとや っちまったなと思っている。$36プランの次のグレードでよかったかもしれない。でもまぁ、もうリミ ットのことを考えなくていい精神的な安定を$360で買ったと思えば...うん、まぁいいか。。。もう忘れ よう。。。くぅっ・・・。

GLM-5のプラン変遷：$36で始めて$360に課金した話

GLM-5の品質は正直どうか

opus4.6と比べると弱い。これは素直に認める。

ただ、足軽がやるタスクは「指示されたコードを書く」「指定された情報を調べてまとめる」「翻訳す る」みたいな、ゴールが明確な作業。自分で判断して方針を決める必要がない。

この種のタスクなら、GLM-5で実用レベルに回る。

それに、足軽の出力は家老（opus4.6）がレビューする構造になっている。品質のセーフティネットは 上位レイヤーが担保する。足軽単体の品質に完璧を求める必要がない。

とはいえ、出力の精度や家老のチェックの甘さに注意することもしばしばあるのだが、それはまた別 の機会に。

品質のセーフティネット：足軽の出力を家老がレビューする構造

Before → After

Before（全員opus4.6）： ・Claude Maxプラン：$200/月 ・レートリミット：4時間で到達 → 1時間強 制停止を1日に何度も繰り返す ・夜間稼働：朝起きたら止まってる

After（将軍・家老opus4.6 + 足軽GLM-5）： ・Claude Maxプラン：$200/月（変わらず） ・GLM-5プ ラン：$360/年（月$30相当） ・レートリミット：5時間枠に対して余裕あり。止まらない ・夜間稼 働：朝起きても動いてる

追加コストは月$30。でもレートリミットで止まるストレスがゼロになった。

「$200で足りない」を「$200 + $30で快適」にできた。

まとめ：判断にはコストをかけて、作業にはかけな

い

この構成がうまくいく理由はシンプル。

将軍・家老は「何をやるか」を決める。ここの判断を間違えると全部が無駄になる。だからopus4.6を 使う。妥協しない。

足軽は「言われたことをやる」。指示が正確なら、モデルの性能差は出にくい。だからGLM-5で十分。

人間の組織と同じ。経営判断をケチると全部崩れる。でも現場の作業員全員に役員級の給料を払う必 要はない。

全員opus4.6で回して爆死した経験があるからこそ、この使い分けの意味が身に染みてわかる。

判断にはコストをかけて、作業にはかけない
