4つの記憶システムの役割——授業の録画（全部残す）・ノート表紙裏メモ（開くたび目に入る）・教科書への書き込み（内容と紐づけ）・暗記

カード（要点だけ検索）

ログ生データ（過去ログhook） = 授業の録画。 講義を丸ごとビデオに撮っておくイメージ。 全部残 ってるけど、「あの話どこだっけ？」って探すのは大変。

memory.md = ノートの表紙裏に書いたメモ。 「先生はこの形式が好き」「テストは毎回ここから出 る」みたいな、ノートを開くたびに最初に目に入る覚え書き。

Entire.io = 教科書への書き込み。 「この公式は、あの演習問題を解くときに使った」みたいに、内容 と紐づいた注釈が残る。

claude-mem = 暗記カード。 大事なところだけ抜き出して、あとから「あれなんだっけ？」で引け る。

元ネタは全部同じ——AIとの会話です。 残し方と、残す量が違うだけ。

memory.mdって何？

memory.mdの仕組み——「覚えて」と言うだけでClaudeが自動メモ、次回も自動で読み込まれる

前回の記事で触れなかったので、ちゃんと紹介しますね。

memory.mdは、Claude Codeに最初から入ってる公式機能です。 追加インストールも、設定ファイル の編集も不要。 Claude Codeを使ってる時点で、もう使える状態。

仕組みはシンプルで。 Claudeが作業中に「これは次回も使いそうだな」と判断した情報を、勝手にメ モとして書き残してくれるんです。

たとえばこんな感じ。

「このユーザーは丁寧語よりカジュアルな口調が好み」

「ファイルの保存先はいつもdraftsフォルダ」

「画像を作るときは毎回Geminiを使ってる」

で、次にClaude Codeを開いたとき、このメモが自動で読み込まれる。 だから「前回も言ったけど、 私はカジュアルな口調が好きで...」っていう説明をやり直さなくて済む。

ただし制限もあります。 200行を超えると切り捨てられるんですよね。

だから「何でもメモする」じゃなくて、「毎回説明してる前提情報だけ残す」のがコツ。

具体的にどうやるかっていうと、めちゃくちゃ簡単で。 普通に話しかけるだけでいいんです。

「私はカジュアルな文体が好きだから覚えておいて」

「画像はいつもGeminiで作ってるから、次からそうして」

「出力はdraftsフォルダに保存して。毎回ね」

こう言うだけで、Claudeがmemory.mdに書き込んでくれる。 逆に「もうカジュアルじゃなくていい よ」と言えば消してくれます。

特別なコマンドとか、ファイルを自分で編集する必要は一切なし。 「覚えて」と言うだけ。「忘れて」 と言うだけ。

「覚えすぎてパンクしない？」

200行の上限を超えると古いメモは消える——消えた知識はclaude-mem（暗記カード）が引き出しに保管

しますよね、普通に考えたら。

実はmemory.mdには200行っていう上限があって。 超えた分は、Claudeが自分で古い情報を整理した り消したりしてくれるんです。

ノートの余白がなくなったら、古いメモを消して新しいことを書く感じ。

...ってことは、大事なことも消えちゃうのでは？

はい、消えます。

で、ここが「次のステップ」につながる話なんですけど。 memory.mdから消えた知識を、あとから探 せるようにしてくれるのが claude-mem（暗記カード）なんですよね。

memory.mdは「今、目の前に貼ってあるメモ」。 claude-memは「過去のメモを全部しまってある引き 出し」。

だから最初はmemory.mdだけで十分なんだけど、「あの時のやり方、消えちゃってた...」が増えてきた ら、claude-memを足す。

このステップアップが、後半の「組み合わせ表」にそのままつながります。

余談: OpenClawだと全部入り

OpenClawの公式メモリ構造——日次ログ・MEMORY.md・memory_searchが最初から全部入り

ちなみに、最近話題のOpenClaw（オープンソースのAIエージェント）。 こっちの公式メモリ構造を調 べてみたんですけど。

日次ログ（memory/日付.md）→ ログ生データに相当

MEMORY.md → memory.mdと同じ仕組み

memory_search → claude-memに相当するセマンティック検索

...これ、全部最初から組み込みなんですよね。 追加インストールなし。設定なし。

しかも古い記憶を自動で薄くする機能（半減期30日）まである。

で、ちょっとだけ自慢させてもらうと。 私がClaude Codeで「ログ + memory.md + claude-mem」を 自分で組み合わせてやってきた構成、OpenClawの公式設計とほぼ同じだったんですよね。

...鼻についたらごめんなさい。でもちょっと嬉しかったので。
