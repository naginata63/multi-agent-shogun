一番手軽な方法から。 localStorage（ローカルストレージ）という、ブラウザに備わっている保存機
能を使います。
名前がちょっと難しそうですが、分解すると簡単です。 local（ローカル） は「自分のところ」とい
う意味。ローカルPCと言えば「自分のPC」のことです。 storage（ストレージ） は「倉庫・保管場
所」。データを貯めておく場所のことです。
つまりlocalStorageは「自分のPC × 保管場所」という意味。
ここでいうブラウザとは、Google ChromeやSafariのこと。 普段はWebサイトを見るために使います
よね。でも実は、ブラウザにはちょっとしたデータを覚えておく機能もあります。
身近な例でいうと「ブックマーク」。お気に入りのサイトを登録すると、ブラウザが覚えていてくれま
すよね。あれはインターネット上ではなく、自分のPCの中に保存されています。
localStorageもこれと同じ。アプリのデータを自分のPCの中に保存してくれる。 だからページを閉じ
ても、次に開いたときにデータが復活するわけです。
#23で作ったToDoアプリ、ページを更新（リロード）すると消えちゃいますよね。 Claude Codeにこ
う伝えてみてください。
これだけで、保存機能が追加されます。
Claude Codeがやってくれることをざっくり言うと。
修正後にブラウザでページを更新してみてください。 タスクが残ってるはず。ちゃんと「記憶」して
る。
ページを更新してもデータが消えないようにして。
localStorageを使って保存と読み込みを追加して。
1. タスクを追加・削除するたびにlocalStorageへ保存
2. ページを開いたときにlocalStorageからデータを読み込み
3. 画面に反映

<!-- page 3 -->

ただし、制限が1つあります。 localStorageはそのブラウザ限定です。
PCのChromeで保存したデータは、スマホからは見れません。 別のブラウザ（Safariなど）からも見
れない。
自分だけで使うアプリなら、これで十分です。
Supabaseでクラウド保存
Supabase — クラウドデータベースでどこからでもアクセス！PC・スマホ・タブレットからデータ共有
「スマホからも見たい」「他の人とデータを共有したい」
そうなるとlocalStorageでは足りません。 データをインターネット上に保存する必要があります。
ここで登場するのがSupabaseです。
Supabaseは無料で使えるクラウドのデータ保管サービスです。 インターネット上にデータの置き場所
を作ってくれるもの、とイメージしてください。
流れはこんな感じ。
1. Supabaseでアカウントを作る（無料）
2. Claude CodeにSupabase MCPを接続する
3. 「ToDoを保存する場所を作って」と伝える
4. アプリをSupabase対応に変更してもらう

<!-- page 4 -->

正直、localStorageより手順は増えます。 でもClaude Codeが各ステップをサポートしてくれるので、
思ったほど難しくない。
MCPの接続方法を忘れた方は、#19を見返してみてください。
Supabase MCPを接続した状態で、こう伝えます。
Claude Codeがデータの保存場所の作成からアプリの書き換えまで、一気にやってくれます。
完了したら、スマホのブラウザからもアクセスしてみてください。 同じデータが表示されるはずで
す。
どっちを選ぶ？
どっちを選ぶ？ — 自分だけで使う→localStorage、みんなで使う→Supabase。天秤で比較
迷ったら、この基準で判断してください。
結論から言うと、最初はlocalStorageで十分です。
まずはlocalStorageで動くものを作る。 「スマホからも見たいな」と思ったタイミングでSupabaseに
切り替える。
この順番がおすすめです。
ToDoアプリのデータをSupabaseに保存するように変更して。
タスクの追加・完了・削除がちゃんと保存されるようにして。
localStorage → 自分1人で、1つのブラウザで使う
Supabase     → 複数端末で使う、他の人と共有する

<!-- page 5 -->

いきなりデータベースに手を出す必要はありません。 localStorageだけでも「ブラウザを閉じてもデー
タが残る」という体験は得られます。
それだけで、アプリの実用度がグッと上がる。
まとめ
ということで、データの保存方法を2つ紹介しました。

ゃんと残ってる」って嬉しくなります。
