まず完成形をイメージしておきます。今回作る家計簿アプリの機能は3つ。
1. 収入・支出の記録 — 日付、金額、カテゴリを入力して保存
2. カテゴリ分け — 食費、交通費、娯楽費などに分類
3. 月別集計 — 今月いくら使ったか、カテゴリごとの内訳を表示
ToDoアプリとの違いは何か。
ToDoは「やること1つ + 済み/未済のチェック」だけでした。家計簿は金額・日付・カテゴリという複
数のデータを扱います。しかも合計の計算や、カテゴリごとの絞り込みも必要になる。
ぐっと「本物のアプリ」に近づきますね。
伝え方のコツ — 複雑なアプリをどう指示するか
指示の出し方 — 1行指示（NG）vs 機能リスト（OK）。複雑なアプリは箇条書きで伝える
ToDoアプリのときは「ToDoアプリを作って」の一言でいけました。
家計簿アプリだとそうはいきません。機能が複数あるので、1行の指示だと Claude Code が迷いま
す。
ここで使えるテクニックが「機能リストを箇条書きで渡す」です。
実際のプロンプト例を見てみましょう。
家計簿アプリを作ってください。
- 収入・支出を記録できる（日付、金額、カテゴリ、メモ）
- カテゴリは「食費」「交通費」「娯楽費」「日用品」「その他」
- 月別の合計金額を表示

<!-- page 3 -->

ポイントは「欲しい機能を箇条書きで並べる」こと。こうするとClaude Codeが設計しやすくなりま
す。技術的な部分はClaude Codeが最適なものを選んでくれるので、指定しなくて大丈夫です。
機能が多いときは、一気に作らせるよりも「まず設計案を見せて」と伝えるのがおすすめです。
Claude Codeが「こういう画面構成で、こういう流れで作ります」と提案してくれます。内容を確認し
てから「これで作って」と伝える。
この「設計を確認 → 作ってもらう」の流れが、複雑なアプリでは効いてきます。
つまずきポイントと乗り越え方
3つのつまずきと解決法 — 計算ミス・レイアウト崩れ・データ消失。一発で完成しなくてOK！
家計簿アプリを作ると、だいたい3つの壁にぶつかります。
1. 計算が合わない
合計金額がおかしい、マイナスになる...。こういうときは具体的に伝えましょう。
- カテゴリ別の内訳を円グラフで表示
- データはlocalStorageに保存
この機能で家計簿アプリを作りたい。まず設計案を見せて。

<!-- page 4 -->

「計算がおかしい」だけだと伝わりません。何を入力して、何が表示されたか。この2つを書くのがコ
ツです。
2. 画面の配置（レイアウト）が崩れる
ブラウザで見るとボタンが重なる、表示がずれる...。
ここで活躍するのが「スクリーンショットを見せる」テクニック。画面のスクショを貼り付けて「こ
の部分を直して」と指示するだけ。Claude Codeは画像を理解できるので、言葉で説明するより早いで
す。
3. データが消える
ページを更新したら入力した記録が全部消えた...。
これは #26 で学んだ localStorage の出番ですね。もし保存されていなかったら、こう伝えましょう。
ここで覚えておきたいのは、一発で完成しなくていいということ。
作る → 試す → 直してもらう → また試す。
このループで仕上げるのが、Claude Codeでのアプリ開発の基本スタイルです。焦らなくて大丈夫。
まとめ
ToDoアプリから家計簿アプリへ。扱うデータが増えて、計算や絞り込みも加わりました。
食費3,000円と交通費1,500円を入力したのに、
合計が3,000円しか表示されません。直してください。
ページを更新するとデータが消えます。
localStorageに保存するように直してください。
