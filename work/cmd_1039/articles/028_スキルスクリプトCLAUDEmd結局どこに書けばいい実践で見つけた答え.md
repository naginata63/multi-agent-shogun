魔法の一問

Claude Codeには大きく4つの「知識の置き場」があります。

CLAUDE.md — プロジェクトのルール設定ファイル

スキル — /xxx で呼び出すカスタムコマンド

スクリプト — python xxx.py で実行する外部ファイル

サブエージェントへのプロンプト — 都度渡す一回きりの指示

で、どこに何を書くかで迷ったら、こう自問してみてください。

「この知識は、次のセッションの自分に、自動で伝わるべきか？」

この1問で4択が決まります。

YES + 毎セッション必要 → CLAUDE.md

コーディング規約、ディレクトリ構造、禁止事項。セッションが変わっても毎回知っておくべきルー ル。CLAUDE.mdに書けば、起動するだけで自動的に読み込まれます。

YES + 特定タスク時だけ → スキル

note記事の生成、画像生成、トピックリサーチ。毎回じゃないけど「やって」と言えば自動で手順を 思い出してほしいもの。これはスキルにする。

NO + 手動実行でOK → スクリプト

python note_update.py --note-id 123 みたいに、コマンドを打てば完結するもの。AIに手順を覚えさせ る必要がない。スクリプトで十分。

NO + 都度調べてOK → サブエージェントに渡すプロンプト

一回きりのリサーチや、その場限りの調査タスク。どこにも保存しなくていい。

迷ったらこの1問を唱えるだけ。まぁこれだけでほぼ答えが出るかなと。
