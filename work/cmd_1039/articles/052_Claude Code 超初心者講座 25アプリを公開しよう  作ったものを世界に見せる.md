<!-- page 3 -->

GitHub（ギットハブ） とは、ファイルをインターネット上に保管・共有できるサービスです。
GoogleドライブやiCloudと似ていますが、プログラムのファイルを置くのに特化しています。 アカウ
ントは誰でも無料で作れます。
GitHub Pages は、GitHubが提供している「公開機能」です。 GitHubに置いたファイルを、そのまま
Webサイトとして公開してくれる。 つまり、GitHubにファイルを置く → 公開設定をONにする →
URLが発行される、という流れ。 追加料金なし。GitHubアカウントがあればすぐ使えます。
アカウントがまだの方は、GitHub公式サイト（github.com）でメールアドレスだけで作れます。
Claude Codeに「GitHubアカウントの作り方を教えて」と聞けば、手順も案内してくれますよ。
#23で作ったToDoアプリみたいなシンプルなアプリなら、GitHub Pagesが最適。
アカウントができたら、Claude Codeにこう伝えるだけ。
これだけ。本当にこれだけです。
Claude Codeが裏側でやってくれることを、ざっくり説明すると。
GitHubに保存場所を作成
ファイルをアップロード
公開設定をONにする
数分待つと、URLが発行されます。
このURLを友達に送れば、スマホでもPCでも開ける。 「自分が作ったもの」にURLが付く瞬間は、素
直に嬉しいです。

もう少し本格的に — Vercel / Netlify
もっと本格的に — Vercel（React向け）とNetlify（静的サイト向け）。覚えておくだけでOK！
GitHub Pagesは万能じゃありません。 シンプルなアプリ向けです。

のがVercelとNetlifyです。
どちらも無料プランがあって、個人開発なら十分すぎるほど。
使い方の流れはGitHub Pagesと似てます。
