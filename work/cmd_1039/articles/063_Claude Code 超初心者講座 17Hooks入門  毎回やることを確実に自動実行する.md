CLAUDE.md（張り紙）とHooks（自動装置）の比較図
CLAUDE.mdは、Claudeへの「アドバイス」です。 読んでくれるし、だいたい守ってくれる。でも
100%ではない。
Hooksは違います。 あらかじめ決めておいた処理（パソコンへの命令文）が自動で実行される仕組み
です。
例えるなら、こう。
CLAUDE.md = 「毎朝歯磨きしてね」と書いた張り紙
Hooks = 目覚ましが鳴ったら自動で歯ブラシが出てくる装置
張り紙は無視できるけど、装置は勝手に動く。 「毎回必ずやってほしいこと」は、Hooksに任せた方
が確実です。
4つのタイミング — いつ動くか
4つのイベントタイミング — PostToolCallとStopが重要
Hooksが動くタイミングは4種類あります。

<!-- page 3 -->

全部覚えなくて大丈夫です。
よく使うのは2つだけ。
PostToolCall — ファイル保存後に何かチェックする、とか。 Stop — 作業完了したらMacに喋らせて
通知する、とか。
まずこの2つを押さえておけば十分です。
設定の仕方 — AIに頼むだけ
日本語で頼むだけでOK — AIが自動で設定してくれる
Hooksの設定ファイルはJSONという形式なんですが、自分で書く必要はありません。
Claude Codeに日本語で頼めば、勝手に作ってくれます。
これだけ。Claudeが設定ファイルを自動で作ってくれます。
PreToolCall   … ツール実行の「前」
PostToolCall  … ツール実行の「後」
Notification  … 通知が出るとき
Stop          … タスク完了時
ファイルを保存したら「保存しました」と表示するHooksを設定して

<!-- page 4 -->

裏側ではこんな形式のファイルが作られています。
中身を理解する必要はないですが、ざっくり言うと：
matcher: いつ動くか（Write = ファイル保存時）
command: 何を実行するか
「ファイルが保存されたら表示する」だけ。仕組みはシンプルです。
最初の一歩 — 完了通知を作ってみる
ほったらかしでOK、終わったらMacが教えてくれる
初めてのHooksには、完了通知がおすすめです。
Claude Codeってタスクが終わっても音が鳴らないので、 別のことしてると「あれ、終わってた？」
ってなりがち。
Claude Codeにこう伝えてください。
{
  "hooks": {
    "PostToolCall": [
      {
        "matcher": "Write",
        "command": "echo '保存しました'"
      }
    ]
  }
}
タスクが完了したらMacに「タスク完了」と喋らせるHooksを設定して

<!-- page 5 -->

これだけで、次からタスクが終わるたびにMacが音声で教えてくれます。
地味だけど、一度設定すると手放せなくなりますよ。
