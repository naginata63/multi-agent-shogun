アイキャッチ

毎日のAIニュースチェック、どうしてますか？

X（旧Twitter）を開いて、Redditを開いて、TechCrunchを開いて...。 気づいたら1時間経ってて、しか も大事なニュースを見逃してたりする。

僕もそうでした。

「これ、AIにやらせたらよくない？」

sasuu｜非エンジニアのAI活用術 2026年3月22日 06:30

ということで、AIが勝手にニュースを集めて、大事なやつだけDiscordに届けてくれる仕組みを作りま した。

今日はその話をします。

何ができるようになったか

朝起きてDiscordを開くと、こんな感じで通知が来ています。

「Claude Codeの新バージョンがリリースされました」

「MetaのAIエージェントがデータ漏洩事故を起こしました」

「新しいオープンソースモデルが公開されました」

しかも、ただのニュース一覧じゃない。

AIがスコアリングしてくれているんです。

「これは重要度95点。あなたのX投稿ネタになりそうです」 「これは重要度35点。ターゲット層と関 係薄いのでスキップ推奨」

つまり、僕がやることはDiscordを開いて、スコアが高いやつだけ読む。それだけ。

以前は毎日1時間かけていた情報収集が、5分で終わるようになりました。

どうやって作ったか（ざっくり）

仕組み自体はシンプルです。

スクリプトが定期的にネットを巡回する

新しい記事やポストを見つけたら保存する

AIが「これ重要？」をスコアリングする

スコアが高いやつだけDiscordに通知する

仕組みの全体図

巡回先はこんな感じ。

Anthropic（Claudeを作っている会社の公式ブログ）

OpenAI（ChatGPTを作っている会社の公式ブログ）

Apple（AIの研究ジャーナル）

TechCrunch（テック系のニュースサイト）

Reddit（海外の掲示板。AI関連のコミュニティがたくさんある）

X（AIの有名人のポスト、キーワード検索）

GitHub（AIツールの新バージョンリリース）

Hacker News（エンジニアが集まるニュースサイト）

全部で10ソース以上。人間が毎日全部チェックするのは無理です。

「集める」だけじゃない。 「選ぶ」のがキモ

集めるだけなら、RSSリーダー（ニュースの自動取得ツール）でもできます。

でもRSSリーダーの問題は全部同じ重さで表示されること。

100件のニュースが並んでいて、「どれが自分にとって大事か」は自分で判断しないといけない。結 局、全部目を通すことになる。

この仕組みでは、AIが「あなたにとっての重要度」を100点満点で採点してくれます。

採点基準は自分でカスタマイズできるのがポイント。

僕の場合は、

自分のX投稿ネタになるか？

自分のnote記事のテーマになるか？

自分のフォロワー層（AI初中級者）が興味を持ちそうか？

この3つの軸でスコアをつけています。

だから「研究者向けの論文」は低スコアになるし、「非エンジニアでも使える新機能」は高スコアにな る。

自分専用のAIニュースキュレーターが手に入った感覚です。

Discord通知にしたワケ

「なぜDiscordなの？LINEじゃダメなの？」

って思う方もいらっしゃると思います。

理由は3つあります。

1. 無料で使える

LINEの通知サービス（LINE Notify）は2025年にサービス終了してしまいました。Discordの通知機能 （Webhook）は今も無料で使えます。

2. リアクションでフィードバックできる

Discordの通知には👍と👎のボタンが自動でつきます。

「このニュース良かった」→ 👍 「これは要らなかった」→ 👎

このフィードバックを後で集計すると、AIのスコアリング精度がどれくらい合ってるかわかるんです。

「AIが95点つけたのに👎されてる」→ 採点基準を調整

こうやってAIの目利き精度を育てていくことができます。

3. ✍️ボタンでX投稿に直結

最近追加した機能で、✍️（ペンマーク）のリアクションも付くようにしました。

「このネタ、Xに投稿したい」と思ったら✍️を押すだけ。 あとでまとめて✍️がついたネタを拾って、 X投稿用の下書きに変換できます。

つまりDiscordがニュースの受信箱と投稿ネタの選別装置を兼ねている。

運用してみてわかったこと

良かったこと

情報の「漏れ」がなくなった。

以前は「あのニュース、昨日見た気がするけどどこだったっけ...」みたいなことがしょっちゅうあり ました。

今は全部ファイルに保存されているので、いつでも検索できます。 「Claude」って検索すれば、過去 のClaude関連ニュースが全部出てくる。

「今日何を投稿しよう」の迷いが消えた。

朝起きて、Discordにスコア90以上のネタが3つ来ていたら、もうその日の投稿ネタは決まりです。

ネタ探しに30分かけていた時間がゼロになりました。

想定外だったこと

Redditが一番おもしろい。

最初は公式ブログ（AnthropicやOpenAI）がメインになると思ってました。 でも実際にスコアが高い ニュースは、Redditの個人投稿が多いんです。

「No-codeのAI開発ツールを2日で捨てた話」とか、 「Claude 3アカウント契約する羽目になった話」 とか。

公式発表は「何ができるか」しか書いてないけど、Redditは「実際に使ってどうだったか」が書いて ある。

これは人間がニュースサイトだけ見てても気づかない視点でした。

最初からうまくいったわけじゃない

最初のバージョンは1時間くらいで動きました。

Claude Codeに「ニュースサイトを巡回して、AIでスコアリングして、Discordに通知するスクリプト を作って」と頼んだら、だいたい動くものが出てきた。

...が、ここからが長かった。

失敗1: スコアリングが間に合ってない

最初のバージョン、154件ニュースを集めて97件をDiscordに通知したんですが、全部スコアが50点だ ったんです。

なぜか。

スコアリング処理が通知の後に動いていたから。

つまり、AIが「これは重要？」と判断する前に、もう全部Discordに流れてしまっていた。

「いや、順番逆じゃん...」

これは処理の順番を入れ替えるだけで直りましたが、最初は気づかなかった。

失敗2: 「重要なニュースを選んで」が曖昧すぎた

次の問題はスコアリングの中身です。

AIに「重要なニュースを選んで」とだけ指示したら、研究論文とかGPUのベンチマーク結果とか、エ ンジニア向けのガチ技術記事ばかり高スコアになった。

まぁ、AIの世界で「重要」って言ったらそうなるよな...。

でも僕のフォロワーは非エンジニアです。「CUDA対応の最適化手法」なんて投稿しても誰も反応しな い。

結局、こういう具体的なスコア基準を書き出しました。

スコアリング基準表

90〜100点: Claude Codeの公式発表、全ユーザーに影響するアップデート

75〜89点: X投稿ネタになりそうな実践的な使い方の事例

60〜74点: 業界的には面白いけど、直接は関係ない

40〜59点: エンジニア寄りで、非エンジニア向けに翻訳しにくい

0〜39点: GPU、研究論文、ニッチ技術（投稿ネタにならない）

さらに「GPU/ハードウェアのベンチマーク」「学術論文の数学的解説」「暗号資産関連」は問答無用で 30点以下にするルールも追加。

ここまで具体的にしてやっと、期待通りのスコアが出るようになりました。

失敗3: Discordが通知スパムになった

スコアリングは改善されたんですが、今度は60点台のボーダーライン記事が大量に流れる問題が起き ました。

朝起きたらDiscordに30件とか通知が来てる。

...読む気が失せる。

これは「2段階通知」で解決しました。

2段階通知の仕組み

70点以上 → すぐにDiscord通知（本当に重要なやつだけ）

50〜69点 → 一旦バッファに溜めて、翌日まとめて1つの通知にする

こうしたら、朝起きたときに見る通知が5〜8件くらいになって、ちょうどいい量になりました。

失敗4: Discord APIの罠

地味にハマったのがDiscordの技術的な部分です。

スクリプトからDiscordに通知を送ろうとしたら403エラー（アクセス拒否）。

なんでだ...と30分くらい悩んで、結局Claude Codeに「別の方法で送って」と投げたら力技で解決して くれました。美しくはない。でも動く。

もう1つ。👍👎のリアクションボタンをつけるには、通知を送った後に「そのメッセージのID」が必 要なんですが、Discordの通知機能はデフォルトで「送りっぱなし」モード。IDが返ってこない。

これもClaude Codeに「リアクションつけたいんだけど」と相談したら、URLの末尾にパラメータを1 つ追加するだけで解決してくれました。自分では絶対気づかなかった。

こういう細かい罠が積み重なって、「1時間で動いた」ものを「ちゃんと使えるレベル」にするまで、 2〜3日くらいちょこちょこ見ては直して、を繰り返しました。

...でもこういう罠こそ、Claude Codeに相談すると一瞬で解決してくれたりする。「指示の具体性」が 大事なのはスコアリング設計の方で、技術的なハマりはAIに丸投げしちゃっていい。

自分でも作りたい人へ — 実践ガイド

「面白そう、自分もやってみたい」と思った方向けに、始め方をまとめます。

プログラミングの知識は要りません。Claude Codeに相談しながら進めれば大丈夫です。

必要なもの

Claude Code（月額のProプランかMaxプラン）

Discordアカウント（無料）

パソコン（MacでもWindowsでもOK）

これだけです。

ステップ1: Discordの通知先を作る

まずDiscordに「自分専用のサーバー」を作ります。

✅ この先の有料パートでは：

Discordの通知先セットアップ（Webhook URLの取得手順）

Claude Codeへの具体的な相談プロンプト（そのままコピペで使える）

スコアリング基準のカスタマイズ方法（自分の読者に合わせる書き方）

定期実行の設定方法

Claude Codeへの相談で失敗しないコツ3つ

Discordを開いて、左の「＋」ボタンからサーバーを作成

作ったサーバーの中に「ai-news」みたいなチャンネルを作る

チャンネルの設定から「連携サービス」→「ウェブフック」→「新しいウェブフック」

表示されるURLをコピーして、メモ帳に保存しておく

このURLが「通知の届け先」になります。

ステップ2: Claude Codeに相談する

ここからがポイントです。

Claude Codeを開いて、こんな感じで相談してみてください。

最初の相談（ざっくり）:

```
AIニュースを自動収集するスクリプトを作りたいです。
以下のサイトから新着記事を取得して、
重要度をスコアリングして、
スコアが高いものだけDiscordに通知する仕組みを作ってください。
巡回したいサイト:
- TechCrunch（AI関連）
- Reddit（r/ClaudeAI）
- Anthropicの公式ブログ
Discord Webhook URL: （さっきコピーしたURLを貼る）
```

これだけで、だいたい動くスクリプトが出てきます。

ステップ3: スコアリング基準を「自分用」にする

ここが一番大事なステップです。

最初のスクリプトはたぶん動くけど、スコアリングが自分に合ってないはずです。

Claude Codeにこう追加で伝えてください。

```
スコアリングの基準を変更してください。
僕の読者は（ここに自分のターゲットを書く）です。
高スコアにしてほしいもの:
-（自分が欲しい情報の種類）
-（自分が投稿したいネタの方向性）
低スコアにしてほしいもの:
-（自分には関係ない情報の種類）
-（難しすぎて翻訳できないもの）
```

たとえば僕の場合は、

```
読者はAI初中級者で、技術的な深い話は求めていません。
「実際に使ってみた系」「便利な新機能」が高スコア。
「GPU」「研究論文」「学術的な話」は低スコアにしてください。
```

こういう感じです。

ここを曖昧にすると、AIが勝手に「重要」を判断して、自分には要らないニュースばかり届きます。 僕もここで一番苦労しました。

ステップ4: 定期実行を設定する

スクリプトが動いたら、あとは自動で動くようにするだけです。

Claude Codeに「このスクリプトを毎日朝6時に自動実行するように設定して」と頼めばOK。

Macなら「launchd」、Windowsなら「タスクスケジューラ」という仕組みで、パソコンが勝手にスク リプトを動かしてくれます。

ステップ5: 使いながら育てる

ここからは「使いながら調整」のフェーズです。

Discordに届いた通知を見て、

「このニュース要らなかったな」→ スコア基準を調整

「こういうニュースも拾ってほしい」→ 巡回先を追加

「通知が多すぎる」→ 通知する点数の基準を上げる

こういう微調整をClaude Codeに相談しながらやっていきます。

僕もまだ調整中です。「完成」はなくて、育てていく感覚ですね。

コツ: Claude Codeへの相談の仕方

この仕組みを作るとき、Claude Codeへの相談で意識するといいことが3つあります。

1. 最初から完璧を目指さない

「全部入りのすごいやつを作って」より、「まずTechCrunchだけ巡回してDiscordに通知するやつを作 って」の方がうまくいきます。

小さく作って、動いたら機能を足していく。

2. 「なぜそうしたいか」を伝える

「RSSを取得して」じゃなくて、「毎朝のニュースチェックに1時間かかってるから自動化したい」と 伝えると、AIが全体の設計を考えてくれます。

3. 不満を具体的に伝える

「なんか違う」じゃなくて、「研究論文が高スコアになってるけど、僕の読者には関係ないから低スコ アにして」と言う。

具体的に言えば言うほど、AIの出力が良くなります。

こういう人におすすめ

毎日のAIニュースチェックに時間を取られている人

X投稿のネタ探しに困っている人

情報収集を「仕組み」にしたい人

Claude Codeを使っていれば、この仕組みは作れます。

というか、「AIに毎日のルーティンを自動化させる」って、Claude Codeの一番おいしい使い方だと僕 は思っています。

「AIすごい」で終わらせるんじゃなくて、自分の毎日が実際に楽になる仕組みを作る。

ぜひ、ご自身の「毎日やってるけど面倒な作業」をAIに任せてみてください。

意外と、1時間で動くものができますよ。
