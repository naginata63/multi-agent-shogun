Claude Codeに部下が2種類いるって知って た？ — サブエージェントとAgent Teamsの

違いを徹底解説

AIに「これやっといて」って頼む方法、実は2種類あるって知ってましたか？

1つは"お使い"タイプ。 もう1つは"プロジェクトチーム"タイプ。

この違いを知ってるかどうかで、AIの使い方がガラッと変わります。

今回は「Claude Code」というツールに搭載されている、2種類の"部下"について解説していきます。

Claude Codeって何？という方もいると思うので、ざっくり説明しますね。

Anthropic社が作った「パソコンの黒い画面（ターミナル）でAIを動かすツール」のことです。

ChatGPTみたいにブラウザで会話するんじゃなくて、パソコンの中で直接AIが作業してくれるイメー ジ。 ファイルを作ったり、コードを書いたり、調べものしたり。けっこう何でもやってくれます。

で、このClaude Codeには「部下」を2種類使い分ける機能がある。

それが サブエージェント と Agent Teams です。

「お使い」と「チーム」— まずはざっくり

お使いとチームの比較（左: 社長→お使いキャラへ「牛乳買ってきて」→「はい、牛乳です」。お使い同士はバツ印。右: リーダー→メンバー3

人がお互いに矢印で会話「ここどうする？」「こうしよう」。一番の違い: メンバー同士が話せるかどうか）

サブエージェント はコンビニにお使いに行かせる感覚。

「牛乳買ってきて」と頼む。 買ってくる。

「はい、牛乳です」と渡してくる。 終わり。

頼んだ自分（親）に結果だけ返してくれます。 お使い同士はお互いの存在すら知りません。

一方、Agent Teams はプロジェクトチームを組んで丸ごと任せる感覚。

「来週の発表資料、作っといて」と頼む。 チームが自動で組まれる。

リーダーが仕事を割り振って、メンバー同士で「ここどうする？」と相談しながら進めてくれる。 最 後に完成品が出てくる。

一番の違いはここ。

お使いは「1人で行って、1人で帰ってくる」。 チームは「メンバー同士で話し合える」。

ざっくり言うと、これだけの違い。 でもこれ、けっこう大きいんですよね。

もうちょい具体的に — 5つの違い

5つの違いインフォグラフィック（報告先・管理方法・記憶・得意な仕事・お金の5項目でお使いとチームを比較）

表にまとめるとこんな感じです。

1つずつ見ていきましょう。

報告先の違い

お使いは、頼んだ自分にだけ結果を返します。 「牛乳買ってきたよ」と報告して、おしまい。

チームだと、メンバー同士で直接やりとりできる。 「デザインこうしたけど、どう思う？」みたいな 会話が生まれます。

管理方法の違い

お使いは、自分が全部コントロールしなきゃいけない。 「Aくんはこれ、Bくんはあれ」と指示を出す のは自分。

チームは、共有のTo-Doリストを持っていて、自分たちで役割を分担してくれる。 自分は「よろしく」 と言うだけでOK。

記憶の違い

お使いは、作業が終わったら頭の中がリセットされます。 結果だけ親に返して、経緯は全部忘れちゃ う。

比較ポイント | お使い（サブエージェント）| チーム（Agent Teams） ───────────────────────────────────────────────────── 報告先 | 自分にだけ報告 | メンバー同士で直接やりとり 管理方法 | 自分が全部指示を出す | 共有のTo-Doリストで自分たちで分担 頭の中の記憶 | 終わったら忘れる | それぞれが自分の頭で覚えてる 向いてる仕事 | 「調べて報告して」系 | 「相談しながら作って」系 AI利用料 | 安め | 高め（人数分かかる）

チームだと、それぞれのメンバーが自分の作業内容を覚えている。 だから「さっきの続きだけど...」 みたいな連携ができるわけですね。

向いてる仕事

お使いは「調べてきて」「実行して結果だけ教えて」系が得意。 答えが1つに決まるような作業向き。

チームは「意見を出し合って決めてほしい」系に強い。 正解がない仕事を、議論しながら進めてくれ ます。

AI利用料

お使いは軽いです。1人分のAI使用料で済む。

チームはメンバーの人数分かかる。 3人チームなら、ざっくり3倍。

まぁ、安く済ませたいならお使い一択かなと。

会社に例えるとわかりやすい

会社の例え（左: お使い型=社長が全社員に個別指示、社員同士はバツ印。右: チーム型=社長→リーダーに丸投げ、メンバーはLINEグループで

自走）

もうちょっとイメージしやすくしましょう。

サブエージェント（お使い）の場合

あなたが社長だとします。

「〇〇くん、これ調べてきて」と頼む。 〇〇くんは調べて「できました」と報告してくれる。

別の△△くんにも別件を頼める。 でも〇〇くんと△△くんはお互い面識なし。

全部あなた（社長）が管理するスタイル。

Agent Teams（チーム）の場合

あなたが社長だとします。

「新商品の企画、よろしく」とプロジェクトリーダーに丸投げ。

リーダーがメンバーを集める。 メンバー同士でLINEグループみたいなものを作る。

自分たちで役割を分担して、自分たちで議論して、完成品を持ってくる。

要するに、 「社長がいちいち指示を出す」 vs 「チームが自走する」 この違い。

「全部自分で管理するのダルい...」って思ったら、チーム型の出番ですね。 楽したい社長にはチーム 型がぴったり。

じゃあどっちを使えばいいの？

使い分けフローチャート（「頼む人たち同士が会話する必要ある？」→ NO=お使い（調べて報告して/テスト回して/ファイル探して）、YES=チ

ーム（3方向からレビュー/仮説を検証し合って/並行開発して）。迷ったらまずお使いでOK！）

迷ったときの判断基準はシンプル。

「頼む相手同士が会話する必要ある？」

Noならお使い。Yesならチーム。これだけ。

もう少し具体的に見てみましょう。

お使い（サブエージェント）が向いてる場面：

「このフォルダの中からエラーを探して」

「テストを実行して結果だけ教えて」

「この機能の使い方を調べてきて」

→ 要するに「答えだけ欲しい」とき。

チーム（Agent Teams）が向いてる場面：

「この機能を安全性・速度・テストの3方向からチェックして」

「バグの原因を3つの仮説で同時に調べて、検証し合って」

「画面側・裏側・テストの3人で並行開発して」

→ 要するに「話し合って進めてほしい」とき。

正直、8割くらいの場面はお使いで十分。

「AIに議論させたい」って場面だけ、チームを使う。 それくらいのイメージでOKです。

Agent Teamsはまだ実験中（ここ注意）

1つ大事なことを言っておくと、Agent Teamsは2025年に追加されたばかりの新機能。

まだ「実験的」な段階です。

設定ファイルに1行追加すれば使えるんですが、制約もあります。

たとえば、

作業の途中でClaude Codeを閉じると、続きから再開できない

同時に動かせるチームは1つだけ

みたいな制限がある。

なので、まずはサブエージェント（お使い）を使いこなすところから始めるのがおすすめ。

お使いだけでも「え、こんなことも自動でやってくれるの？」ってなるはず。

焦ってチームを使う必要はないかなと。 お使いで物足りなくなったら、そのとき試してみればOKで す。
