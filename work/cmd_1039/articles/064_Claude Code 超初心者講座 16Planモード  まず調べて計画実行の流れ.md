通常モードとPlanモードの比較フロー図
通常モードだと、指示したらすぐ実行に入ります。 速くていいんですが、タスクが大きいと暴走リス
クがある。
Planモードはこの流れを変えます。
間に「計画を見せる」ステップが入るだけ。 でも、この1ステップがかなり効きます。
切り替え方は簡単で、Shift+Tab を何回か押すだけ。 画面下に「plan」と表示されたらONの状態で
す。

使うべき場面と使わなくていい場面の比較図
通常モード:  指示 → 即実行
Planモード:  指示 → 計画を提示 → 確認 → 実行

使うべき場面
フォルダ構成を大きく変えるとき
複数ファイルをまとめて編集するとき

判断基準はシンプル。
「失敗したらやり直すのが面倒か？」
YESならPlanモード。NOなら通常モードで十分。 迷ったらPlanモードにしておけば間違いないです。
計画が出たらどうする？
計画が出たあとの3つの選択肢

<!-- page 4 -->

Planモードで指示を出すと、Claudeがまず計画を見せてくれます。 「こういう手順でやりますよ」と
いうリストが出てくる感じ。
ここからの選択肢は3つ。
1. そのまま進める 計画を読んで問題なければ「OK」「いいよ」と言えば実行開始。
2. 修正を入れる 「ステップ3はこう変えて」「先にバックアップ取って」など、計画を調整できます。
修正した計画で再提示してくれるので、納得してから進められる。
3. やめる 「やっぱやめて」で中止。何も実行されません。
これが最大のメリットなんですよね。
通常モードだと、実行してから「違う！」と気づく。 Planモードなら、実行前に「あ、ここ違うな」
と気づける。
事後のリカバリーより、事前のチェックの方がずっと楽です。
まとめ
Planモードは、Claude Codeの安全装置です。
使い分けはこれだけ。
覚えるのは Shift+Tab だけ。 タスクの規模に合わせて切り替えてみてください。
さて、これで効率化編は完了です。
大きめのタスク → Planモード
小さいタスク　 → 通常モード
迷ったら　　　 → Planモード
