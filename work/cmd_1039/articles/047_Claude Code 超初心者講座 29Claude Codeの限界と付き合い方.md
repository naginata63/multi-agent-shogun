まず、苦手な領域を整理しておきましょう。
1. 大規模アプリの全体設計
大規模アプリ — AパターンとBパターンのファイルが混在してバラバラに
ファイルが数十個になってくると、一貫性を保つのが難しくなります。 「ここではAパターンで書いた
のに、別のファイルではBパターンになってる」みたいなことが起きがち。
小さいアプリなら問題ないんですが、規模が大きくなると人間側の管理が必要になってきます。
2. リアルタイム通信やゲーム開発
リアルタイム通信 — 送信中...で止まるチャットとカクカク動くゲームキャラ
チャットアプリのように「メッセージを送った瞬間に相手に届く」仕組みや、ゲームのキャラクター
を滑らかに動かす処理。 こういった高度な技術は、うまくいかない場合があります。
動くものは作れても、動きの滑らかさや安定性の面で調整が必要になることが多いです。
3. デザインのセンス

<!-- page 3 -->

デザインセンス — 無難な見た目→「こんな雰囲気にして」と参考を渡すと改善
Claude Codeが提案してくる見た目は、無難なものが多いです。 「動く」けど「おしゃれ」ではな
い、という感じ。
見た目にこだわりたいなら、参考サイトのURLやスクリーンショットを渡して「こんな雰囲気にして」
と伝えるのがコツですね。
4. 最新情報への対応
最新情報 — 知識に期限ありの古い本。でもURLを渡せばOK
Claude Codeの知識には期限があります。 最新の技術やツールの情報には、追いつかない場合があ
る。
ただし、公式サイトの説明ページのURLを渡せば読んでくれるので、完全にお手上げというわけでは
ありません。
うまく付き合うコツ

<!-- page 4 -->

うまく付き合う5つのコツ — 小さく作る、こまめに確認、CLAUDE.md活用、具体的に指示、80点でOK
苦手なことがあるなら、こちらの使い方で補えばいい。 実際に効果があった5つのコツを紹介しま
す。
小さく作る。
小さく作る — ×一気に全部（崩れる）vs ○少しずつ積み上げ（成功）
「ToDoアプリを全部作って」より「まずタスクの追加機能だけ作って」のほうが精度が上がります。
一度に大きなものを頼まない。これ、めちゃくちゃ大事。
こまめに確認する。
こまめに確認 — 作る→確認→作る→確認のリズム
たくさん書いてもらってから確認するより、少しずつ確認するほうが手戻りが少ないです。 小さく作
って、動かして、次に進む。このリズムが最強。

<!-- page 5 -->

CLAUDE.mdで好みを伝えておく。
CLAUDE.md活用 — 好みリストをメモ帳に書いて渡すとClaude Codeが「了解！」
#10で学んだCLAUDE.mdに、コードの書き方やデザインの好みを書いておくと、毎回のズレが減りま
す。 「日本語でコメントを入れて」「シンプルなデザインにして」みたいなことを事前に伝えておく。
「作り直して」じゃなく「ここを直して」。
具体的に指示 — NG「全部作り直して」vs OK「ここを直して」ピンポイント
全体のやり直しを頼むと、前に良かった部分まで変わってしまうことがあります。 「3行目の表示がず
れてる」「ボタンの色を青にして」など、具体的に指示しましょう。
完璧を求めない。
完璧を求めない — 80%でOK!（笑顔）vs 100%は疲れる（ぐったり）
80%の出来でOKとして、残り20%は自分で微調整する。 Claude Codeが出してくれた土台に、自分の
好みを加える。この感覚が大事です。
人間がやるべきこと
人間とClaude Codeの役割分担 — アイデア・ユーザー視点は人間、コード・修正・設計はClaude Code。最強の相棒
ここまで読んで気づいたかもしれませんが、Claude Codeは「道具」です。
「何を作るか」を決めるのは、自分。

<!-- page 6 -->

何を作るかは自分で決める — 人間「家計簿アプリがほしい！」とひらめく
どんなアプリが必要か、誰のために作るのか。この判断はAIにはできません。 「家計簿アプリがほし
い」と思いついたのは、あなた自身ですよね。
「使う人の気持ち」を考えるのも、自分。
使う人目線 — AIがコードを書き、人間がアプリのボタンや文字をチェックする
ボタンの位置はここでいいか。文字は読みやすいか。 ユーザーの立場で考えるのは、人間にしかでき
ない仕事です。
だから、胸を張っていい。
やった！ — Claude Codeが拍手、王冠をかぶった人間が完成アプリの前でガッツポーズ
「Claudeが作った」じゃなく「自分が作った」でいいんです。 アイデアを出して、指示を出して、確
認して、調整した。それは間違いなく「あなたの作品」。
料理人が包丁を使って料理を作るのと同じ。 包丁がすごいんじゃなくて、使いこなしている人がすご
いんです。
まとめ
限界を知ることで、逆にうまく使いこなせるようになります。 苦手な部分は自分で補い、得意な部分
は全力で任せる。それが最強の付き合い方です。
万能じゃないけど、最強の相棒ではある。ここまでの28回で、それは実感できたんじゃないでしょう
か。
