<!-- page 2 -->

ファイル検索：「見積書どこ？」でフォルダの中から見積書.pdfを発見
PCの中からファイルを探すの、地味にダルいんですよね。
フォルダを開いて、奥の方まで潜って、「あれ、どこ保存したっけ...」。 Spotlight検索を使っても、候補
が多すぎて目的のファイルにたどり着けない。
Claude Codeなら、日本語で頼むだけです。
「先週作った見積書どこ？」
「PDFだけ一覧にして」
「〇〇プロジェクトの資料、全部探して」
こんな感じで聞くと、フォルダの中を全部調べて見つけてくれます。
しかもSpotlight検索より賢いポイントがあって、 ファイルの中身まで見てくれるんですよ。
たとえば「鈴木さんとの契約書」って聞いたとします。 ファイル名に「鈴木」が入ってなくても大丈
夫。 本文に「鈴木」と書いてあるファイルを、ちゃんと拾ってきてくれます。
Finder検索だとファイル名しか見てくれないこと、ありますよね。 Claude Codeは中身まで読むか
ら、探し物のヒット率が段違いです。
これだけでも「おお...」ってなりませんか。
中身を読む — 「このファイルの内容教えて」
ファイル読み出し：PDF・Word・CSVなんでも読める！

<!-- page 3 -->

ファイルが見つかったら、次は中身。
「この報告書の中身を教えて」
これで全文読んで、内容を説明してくれます。
対応してるファイル形式もけっこう幅広くて、 テキスト、PDF、CSV、コードファイル...だいたいい
けます。
使い方の例をいくつか挙げると、
「このCSVのデータ構造を教えて」
「この契約書の重要な条件だけ教えて」
「このファイル、前回のバージョンと何が変わった？」
自分で全部読まなくても、必要な部分だけ聞き出せるのがポイント。
正直、長い資料を頭から読むのってしんどいじゃないですか。 Claude Codeに「気になるところだけ
教えて」と頼めば、読む時間がぐっと減ります。
しかも一度読ませたファイルの内容は会話の中で覚えてくれるので、 「さっきのファイルの3ページ目
の金額、合ってる？」みたいな追加質問もできます。
要約する — 「3行でまとめて」
要約：30ページの報告書を「3行でまとめて」で一瞬で要約
ここが個人的に一番使ってる機能かもしれません。

<!-- page 4 -->

30ページの資料を渡して、「要点だけ教えて」。
ほんとに3行で返ってきます。 会議前にサッと把握したいとき、めちゃくちゃ助かるんですよね。
しかも、まとめ方の指定もできます。
「箇条書きで5つにまとめて」
「この議事録の決定事項だけ抜き出して」
「専門用語をなくして、誰でもわかるように要約して」
「30ページ読んでおいて」って言われたとき、 全部読むのに30分かかるところが、1分で済む。
この差、けっこうデカいです。
しかも自分で読むと見落としがちなポイントも、 「重要な数字だけ抜き出して」とか「リスクになり
そうな箇所は？」みたいに聞けば拾ってくれる。 読むだけじゃなくて「読み方」まで指定できるのが
強いですね。
まとめ
今回紹介したのは、この3ステップ。
ファイル操作のスタート地点ですね。 地味に見えるかもしれませんが、これだけでも十分便利です。
特に「探す → 要約」のコンボは毎日使えます。 朝イチで「昨日のメモ探して、要点まとめて」と頼む
だけで、 昨日の続きからスムーズに仕事に入れる。
「探す → 読む → まとめる」が日本語だけで完結する。 まぁ、月3,000円の元はこれだけでも取れる
かなと。
ステップ1: 探す  → 「〇〇ってファイル探して」
ステップ2: 読む  → 「中身を教えて」
ステップ3: 要約  → 「3行でまとめて」
