AIを使って思考を深く潜らせる2つの方法

「AIに文章を書かせる」「AIに調べものをさせる」——

AIの使い方、ここで止まっていませんか？

でも私が強く推奨したいのは、AIの言葉や考え方と、自分の感覚をぶつけていくという使い方なんで すよね。

1人で考えてると、言語化できない瞬間ってありませんか。 方向性はなんとなくあるのに、それを形に する方法が見つからない。 モヤッとしたまま、結局「まぁこれでいいか」で終わっちゃう。

そういうときに、AIに考えさせた内容と自分の感覚や思考をぶつけ合うと。 すると「あ、それは違う な」「そっちじゃないんだよな」って、感覚的にズレを感じる瞬間がある。

この「違う」と気づくプロセスが、実はめちゃくちゃ大事で。 ズレを感じるたびに、自分が本当に考 えていたことが少しずつ言葉になっていくんです。

今回は、このAIとの思考のぶつけ合いを実現する2つの方法を紹介します。

方法1: 1つのAIに5人の専門家を演じさせて討論さ せる

やり方

ChatGPTやClaude（月3,000円くらいの有料AI）に、こんな感じで指示を出します。

「このテーマについて、5人の専門家のペルソナを作り、多角的に議論を交わして、最終的にどのよう な結論になるか（あるいは効果的な方法は何か）を提示してください」

ポイントは、5人の設定だけでなく最後の目的まで伝えること。 「結論を出して」「効果的な方法を提 示して」と目的を添えるだけで、より具体的で実践的な意見が返ってきます。

例えばこんな5人が自動で立ち上がる：

批判者：「それ本当にうまくいく？」とツッコむ人

実務家：「現場ではこうだよ」と経験から語る人

研究者：「データではこう出てます」と根拠を示す人

消費者目線：「普通の人はそう思わないよ」と冷静な人

哲学者：「そもそもの前提を疑おう」と深掘りする人

1回のプロンプト（AIへの指示文）で、5つの視点からの議論と結論がまとめて出てきます。

【画像: 1つのAIが5人の専門家を演じて討論するイメージ図 (02-five-personas.png)】

何が起きるか

自分1人だと「いいアイデアだ！」で止まるところに、「いや、それコスト的に無理じゃない？」って ツッコミが入る。

5人の専門家を、自分の頭の中に召喚するイメージです。

正直、耳が痛い意見もたくさん出てきます。 「それ、ユーザーは求めてないよ」とか「前提がそもそ も間違ってない？」とか。

でも、こういう意見こそ本当は大事で。 耳が痛い指摘に向き合うことで、それを乗り越える方法や、 打破する新しい考えに至ることができる。

「こんな角度から考えたことなかった」が、ほぼ確実に出てくる。 しかもたった1回の指示で。

この方法の強み

手軽。1つのAI、1つの指示で完結する

「多角的に見る」ことに特化してる

5分もあれば、自分の視野の外にある視点を引っ張り出せる

この方法の限界

ただし、弱点もあります。

全員が「同じ脳みそ」から出てきてるんですよね。

例えるなら、1人の役者が5役を演じ分けてる状態。 演技は上手いけど、根っこは同じ人。

あくまで個人的な感覚ですが、「それ本当に大丈夫？」「それでうまくいくの？」みたいな批判的な意 見を出させるとき、ちょっと弱いなと感じています。 同じ脳みそから出てくる批判って、どこか遠慮 が入るというか、本気で刺してこない。

だから「本気で違う意見」にはなりにくい。 ここが、2つ目の方法が必要になる理由です。

なぜ2つ目の方法が必要なのか

方法1は「視点の広がり」を作ってくれます。 でも「思考の深さ」には限界がある。

なぜかというと、1つのAIは1つの学習データ、1つの思考パターンしか持ってないから。

人間でも同じですよね。

同じ会社の人5人に聞く → 「うちではこうだよね」で終わる。 異業種の人5人に聞く → 「え、そんな 考え方があるの？」が出る。

AIも全く同じ構造なんです。

だからこそ、そもそも「脳みそ」が違うAI同士にぶつけさせる。 これで議論の質が一気に変わりま す。

方法2: 3つのAI（Claude × Gemini × ChatGPT） に議論させる

やり方

同じテーマを3つのAIに投げて、それぞれの回答を他のAIに見せて反論させます。

Claude → Gemini → ChatGPT → Claude...みたいにリレーしていく感じ。

私はClaude Code（Claudeをターミナルから操作できるツール）から、GeminiやChatGPT（現在は Codexという名前になっています）を呼び出して自動化しています。 ただし、この自動化にはClaude Codeの設定が必要になるので、そこはご了承ください。

大事なのは、ただ同じ質問を投げるだけじゃなく、各AIに「あなたはこういう立場で意見してね」と 役割を与えること。

私が実際に設定している役割はこうです：

Claude → 批判者：リスク・弱点・見落とし・反例を中心に分析する。楽観的な見方に対して鋭く 疑問を投げかける役

ChatGPT（Codex） → 実行者：具体的な手順・コスト・スケジュール・現実的な解決策を考え る。実装可能性を重視する役

Gemini → 調査者：データ・事例・根拠・統計で裏付ける。関連する研究や先行事例を引用する役

なぜこの割り当てにしたかというと、使い込んでいくうちに感じた各AIの「クセ」に合わせているか らです。 公式に「Claudeは慎重です」と書いてあるわけではなく、あくまで私が何百回と使ってきた 中での肌感覚です。

Claudeは回答にリスクや注意点を添えてくることが多いので批判者に。ChatGPTは「じゃあこうしま しょう」とすぐ動き出す傾向があるので実行者に。Geminiは関連情報を幅広く引っ張ってくるのが得 意なので調査者に。

この「なんとなくこのAIはこういうタイプだな」という感覚に沿って役割を振ると、無理なく本気の 議論が生まれます。

流れはこんな感じ：

まずChatGPT（実行者）に「〇〇についてどう思う？」と聞く

その回答をコピーして、Gemini（調査者）に「これに対してどう思う？」と聞く

Geminiの回答をコピーして、Claude（批判者）に「これに対してどう思う？」と聞く

これを2周繰り返し、最後にClaudeが結論をまとめる

【画像: 3つのAIがリレー形式で議論する流れ (04-three-ai-relay.png)】

私が感じる3つのAIの「性格」の違い

ここが面白いところなんですが、使い込んでいくと3つのAIにそれぞれ「性格」の違いを感じるように なるんですよね。

何が起きるか

Claudeが「このプランにはリスクがある」と指摘する。 ChatGPTが「でもこう対策すれば行ける」と 返す。 Geminiが「データ的にはこうなってます」と根拠を添えてくる。

特にGeminiは調査者として設定しているので、圧倒的な情報量をもとに「こういうデータがあります が、それでも進めますか？」と詰めてくることがあります。 根拠付きの批判って、けっこう耳が痛 い。たまに「お前は検察官か？」と言いたくなるくらいの圧で来ます。

でも、1つのAIだけだと見えなかった「思考の死角」が、こうやって炙り出されるんです。

3つの異なる「脳みそ」がぶつかるから、議論の質が段違いに上がる。

例えるなら、方法1は「1人の器用な人に5つの役を演じてもらう一人芝居」。 方法2は「育ちも専門も 違う3人を同じテーブルに座らせる会議」。

一人芝居はどれだけ上手くても、結局は1人の引き出しの中で完結する。 でも、バックグラウンドが違 う3人が集まると、肯定的な意見も批判的な意見も出てくる。そこから「じゃあどうしようか」とい う、さらに一歩先の思考にたどり着くことがある。

Claude（月3,000円くらいの有料AI） → 慎重で丁寧。「でもこういうリスクもありますよね」と 必ずリスク面を指摘してくる。 石橋を叩いて渡るタイプ。

Gemini（Googleが作ったAI。無料で使える） → 情報量が多い。最新のデータを引っ張ってくるのが得意。 「調べてみたらこういう情報があります」と、 根拠ベースで話を広げてくれる。図書館の司書タイプ。

ChatGPT（月3,000円くらいの有料AI。無料版もあり） → 実践的でアクション志向。「じゃあこうしましょう」と すぐ具体的な行動プランを出してくる。 話をまとめて前に進めるのが上手い。コンサルタントタイプ。

この方法の強み

肯定と批判の両方が出た上で、そのいいとこ取りをした結論に至れる

批判的な意見がちゃんと入るので、勢いだけで「GO！」とはならない。割と信頼できる意見が出 やすい

1人だと思考が偏りがち。「できる」と思い込めばリスクが見えなくなるし、逆にリスクを感じす ぎるとリターンを軽く見てしまう。3つのAIにぶつけることで、そのバランスをもう少し冷静に整 理できる

【画像: 方法1と方法2の思考の深さの違い (05-depth-comparison.png)】

2つの方法の使い分け

おすすめの流れは、まず方法1で視点を広げて → 方法2で深掘りする。

例えば新しいサービスを作りたいとき：

方法1 = 「どんなサービスが求められてる？」とアイデアを片っ端から出す段階

方法2 = 出てきたアイデアの中から1つずつ、「このサービスはこの市場で本当にうまくいくの か？」を批判的な視点も含めてちゃんと吟味する段階

広げてから深める。この順番が最強です。

【画像: 方法1（広げる）→ 方法2（深める）の使い分けフロー (06-when-to-use.png)】

まとめ

方法1（1AI × 5ペルソナ） → アイデア出し、ブレスト、多角的に眺めたいとき → 手軽さ重視。5分でできる → 「広げる」フェーズ

方法2（3AI討論） → 意思決定、戦略を練る、大事な判断をするとき → じっくりやる。30分〜1時間 → 「深める」フェーズ

AIを「作業の道具」としか使ってないのは、正直もったいない。

「考える相棒」として使うと、思考の質が一気に変わります。

特に1人で仕事してる人、フリーランス、経営者には効果がデカい。 壁打ち相手がいない問題を、AI が解決してくれる。

まずは方法1から試してみてください。 ChatGPTでもClaudeでもGeminiでも、「5人の専門家になりき って討論して」と入れるだけです。

方法2の3AI討論を自動化するやり方も作ったので、そのうち記事にします。

私が思うこと（個人的な意見として）

ここまで2つの方法を紹介しましたが、1つだけ伝えておきたいことがあります。

AIのアウトプットだけで「思考が深まった」と満足しやすいので、そこはご注意ください。

AIがやってくれるのは、あくまで「初段階で考えられること」を全部やってくれた、という話なんで すよね。

現実世界で生きてる私たちには、AIが持っていない膨大な「体験」がある。

実際に失敗した経験。 肌で感じた直感。 人と話して生まれた気づき。

これはAIにはない。

もちろん、人間の自分が間違っていることもたくさんある。 でもAIが出した答えが本当に正しいかと いうと、それも定かじゃない。

だからこそ、最終的には自分とAIとで意見をぶつけていくことが大事なんじゃないかなと思っていま す。 AIが整理してくれた土台の上に、自分のリアルな経験や感覚を重ねていく。そのやりとりの中 で、本当の意味で思考が深まっていく。

つまりAIは「思考のゴール」じゃなくて、「思考のスタートライン」を一気に引き上げてくれるもの。

そこから先は、行動して改善を続けていく。 でもそのプロセスもAIと一緒に回していけるので、1人で 走るよりずっと速く進めるはずです。
