CLI・デスクトップアプリ・IDE拡張の3つを説明する図
CLI（シーエルアイ）
このシリーズでずっと使ってきた、黒い画面に文字を打って操作する方法です。 「ターミナル」とも
呼ばれます。
見た目は地味だけど、Claude Codeの全機能が使える。
デスクトップアプリ
普通のアプリと同じように、ダウンロードしてアイコンをクリックすれば開ける。 見た目はチャット
画面に近くて、直感的に使えます。
「デスクトップアプリの方が簡単じゃない？」と思うかもしれません。 実際、そう勧める人も多いで
す。
IDE拡張（アイディーイーかくちょう）
プログラマーが使うコード編集ソフト（VS Codeなど）の中で動くやつです。 プログラマー向けなの
で、今回は深く触れません。
実はどれでも同じことができる
3つとも同じ機能が使える図
意外かもしれませんが、中身は全部同じClaude Codeです。
このシリーズで紹介してきた機能はどれでも使えます。

「じゃあ使いやすい方でよくない？」
その通りです。で、僕はCLIを推します。
それでもCLIを推す理由
CLIを推す理由 — 情報が多い・応用が広がる・シリーズもCLI前提
デスクトップアプリを勧める人が多いのは知っています。 見た目が分かりやすいし、とっつきやす
い。それは間違いない。
でも、CLIを使えるようになっておく方が後々有利です。
理由1: ネット上の情報がCLI前提
Claude Codeの使い方を検索すると、ほとんどがCLIの画面で解説されています。 困ったときに調べや
すいのはCLIの方。
理由2: できることがどんどん広がる
CLIに慣れると、自動化やスクリプト連携など応用の幅が一気に広がります。 デスクトップアプリだ
と、こうした拡張がしにくい。
CLAUDE.md   → 全部で使える
Hooks       → 全部で使える
MCP         → 全部で使える
Planモード  → 全部で使える

理由3: このシリーズもCLI前提
ここから先の上級編は、CLI前提で進めます。 #1からずっとCLIベースで解説してきたので、一貫性を
保ちます。
最初はちょっと慣れが必要だけど、「黒い画面に文字を打つ」に慣れてしまえば大丈夫。 このシリーズ
の初級編を乗り越えた方なら、もう使えているはずです。
