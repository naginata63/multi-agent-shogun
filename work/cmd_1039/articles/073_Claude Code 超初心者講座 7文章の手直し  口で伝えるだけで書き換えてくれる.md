トーン変更：硬い文章をカジュアルに、やわらかい文章をフォーマルに自在に変換
まずは一番使う場面が多いやつから。 文章のトーン調整です。
たとえば、こんなメール文を書いたとします。
「もうちょっとカジュアルにして」と頼むと、
一瞬で空気が変わりますよね。
逆もいけます。 ラフに書いた文章を「フォーマルにして」と頼めば、ビジネス文書っぽく整えてくれ
る。
しかも微調整がきくのがポイントで、
「相手が社長なんだけど、もうちょっと丁寧に」
「親しい取引先だから、硬すぎず柔らかめに」
「社内チャット向けに、もっとラフでいい」
こういう「ちょうどいい塩梅」って自分で書くと迷うんですよね。 Claude Codeに任せると、感覚的
な指示でもちゃんと汲み取ってくれます。
ファイルを直接編集したい場合は、パスを指定するだけ。
【Before】
お世話になっております。
先日ご依頼いただきました件につきまして、
添付の通りご報告申し上げます。
ご査収のほど、何卒よろしくお願い申し上げます。
【After】
お疲れさまです。
先日の件、資料をまとめたので添付しますね。
確認よろしくお願いします！

「デスクトップの挨拶文.txtをカジュアルにして」
元ファイルの中身を読んで、書き換えまでやってくれます。
誤字・表記ゆれの修正 — 「全部チェックして」
誤字修正：赤バツだらけの書類が一括チェックで「誤字ゼロ！」に
次は誤字チェック。 正直、これが一番地味に助かってます。
「このファイルの誤字を全部直して」
これだけで、一括修正が走ります。
見つけてくれるのは、こんなもの。
漢字の間違い（「以外」と「意外」の取り違え）
送り仮名の揺れ（「行なう」と「行う」が混在）
半角・全角の混在（「１０個」と「10個」がバラバラ）
句読点の不統一（「、」と「，」が混ざってる）
長文でもまとめて処理してくれるので、人間が目視でチェックするより正確で速い。 10ページの報告
書でも、数秒です。
しかも「修正箇所を一覧で見せて」と頼めば、 どこを直したか確認してから反映できます。
修正一覧:
1. 3行目: 「以外に多い」→「意外に多い」

<!-- page 4 -->

こんな感じで出してくれるので、安心。 「全部おまかせで直して」でも、「確認してから直して」で
も、好きな方でいけます。
フォーマット整形 — 「見やすく整えて」
フォーマット整形：ベタ打ちメモが見出し・箇条書き付きの議事録に変身
最後は体裁の話。 ベタ打ちのメモを、ちゃんとした文書に仕上げるやつです。
たとえば会議のメモをこんな感じで殴り書きしたとします。
「議事録のフォーマットに整えて」と頼むと、
2. 8行目: 「行なう」→「行う」に統一
3. 15行目: 「１０個」→「10個」に統一

見出しがつく。箇条書きになる。情報が整理される。 同じ内容なのに、読みやすさが全然違いますよ
ね。
テンプレートを使いたい場合も簡単です。
「このファイルを、テンプレート.mdの形式に合わせて」
既存のフォーマットに揃えたいときに便利。 他にもこんな指示ができます。
「Markdown形式にして」
「表にまとめて」
「箇条書きに分解して」
体裁を整える作業って、内容を考えるのとは別の頭を使うんですよね。 そこをClaude Codeに任せる
と、中身に集中できます。
