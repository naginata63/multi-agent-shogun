最初のアプリに何を選ぶか。ここ、実は結構大事です。
選んだ理由はシンプル。
機能が少ない（タスク追加・完了チェック・削除の3つだけ）
実用的（作ったあと実際に使える）
達成感がある（10分で「動くもの」ができる）
完成イメージはこんな感じ。
ブラウザで開くと入力欄とボタンがある。 テキストを入力して「追加」を押すとタスクが並ぶ。 チェ
ックを入れると完了。ゴミ箱ボタンで削除。
...以上。これだけです。
最初のアプリは小さく作る。欲張らない。 ここが一番のコツかもしれません。
Claude Codeに伝えてみよう
Claude Codeとの会話フロー — 「ToDoアプリ作って」→「この設計でOK？」→「OK！」
さて、いよいよClaude Codeに指示を出します。
「プログラミングの知識がないと伝えられないんじゃ...」 と思う方もいらっしゃると思います。
安心してください。普通の日本語で大丈夫です。
こんな感じで伝えてみましょう。

<!-- page 3 -->

ここでワンポイント。
Shift+Tabを何回か押してPlanモードに切り替えてから伝えると、まず設計案を見せてくれます。
（Planモードの詳細は#16で紹介しました）
Claudeが「こういう構成で作りますよ」と計画を出してくるので、内容を確認して「OK」と答える。
するとコードの生成が始まります。
ファイルの作成許可を求められたら、承認ボタンを押すだけ。 数十秒でHTMLファイルが出来上がり
ます。
動かす → 直す → 完成
動かす→直す→完成の循環フロー — 動かして、直してもらって、完成へ
ファイルができたら、ブラウザで開いてみましょう。
Claude Codeに「ブラウザで開いて」と伝えれば、開いてくれます。 Macなら
シンプルなToDoアプリを作って。
ブラウザで動くやつ。HTMLとCSSとJavaScript、
1ファイルにまとめて。
機能はこの3つ:
- タスクの追加
- 完了チェック（取り消し線が入る）
- 削除ボタン

<!-- page 4 -->

open todo.html
を実行してくれるはず。
開いたら、まずは触ってみる。 タスクを追加して、チェックして、削除して。
「お、動いてる！」
...ってなったら大成功です。
でも大抵、何かしら気になるところが出てくる。 「ボタンの色がダサい」「文字がちょっと小さい」
「余白が詰まってる」とか。
一発で完璧にならなくても大丈夫。直してもらえばいい。
修正の伝え方も、そのまんまでOK。
こんな感じ。日本語で気になったことを伝えるだけです。

全体的にもう少し余白を増やして
