職場の引き継ぎメモとCLAUDE.mdの対応関係
職場で仕事を引き継ぐとき、「引き継ぎメモ」って書きますよね。 「このプロジェクトのルールはこ
う」「ファイルはここに置く」「連絡はSlackで」みたいな。
CLAUDE.mdは、それのAI版です。
プロジェクトのルールや自分の好みを書いておくファイル。 Claude Codeは新しいセッションを始め
るたびに、このファイルを自動で読んでくれます。
つまり、一度書いておけば毎回説明しなくていい。
「あー、日本語で書いてって言い忘れた...」 「またファイルを変な場所に保存された...」
こういうの、CLAUDE.mdがあると全部なくなります。 正直、あるとないとで快適さが全然違うんで
すよね。
/initで自動生成 — 1コマンドで完成
/initで自動生成する3ステップのフロー

CLAUDE.mdに書く内容、最初は迷うと思います。 ざっくり分けると、こんな感じ。
書くべきもの
言語の指定（「日本語で回答して」）
ファイルの保存先（「出力はdraftsフォルダに」）
命名ルール（「ファイル名はYYYY-MM-DD形式で」）
よく使うコマンドやツール
書かなくていいもの
一回きりの指示（その場で伝えればOK）
AIの性格設定（「明るく話して」とか。正直あんまり効かない）
長すぎる文章（読み込みに時間がかかる）
最初は3〜5行で十分です。
たとえば、こんなシンプルなものでもOK。
これだけでも、毎回の「日本語で書いて」がゼロになる。 たった3行で、セッションのたびに説明し
直すストレスが消えます。
CLAUDE.mdがない場合と比べてみると、差は歴然。
# プロジェクトルール
- 日本語で回答する
- ファイルはdrafts/に保存する
- ファイル名はYYYY-MM-DD-タイトル.md形式
CLAUDE.mdなし:
  → 毎回「日本語でお願い」「ファイルはここに」と指示
  → 言い忘れると英語で返ってきたり、変な場所に保存される
