通常フローと逆転フローの比較
普通、Claude Codeの使い方ってこうですよね。
でも、こう頼むこともできます。
すると、Claude Codeが聞いてきます。
インタビューを受けてる感覚です。 自分は聞かれたことに答えるだけ。
面白いのが、答えてるうちに自分の考えが整理されること。 「あ、一覧にフィルター機能もほしいな」
とか、「管理者画面は後でいいか」とか。
自分が指示を出す → Claude Codeが実行する
「日報管理アプリを作りたいんだけど、何を作りたいか質問して」
CC: 「誰向けのものですか？」
自分: 「社内チーム向け」
CC: 「必須の機能は何ですか？」
自分: 「日報の提出と一覧表示」
CC: 「デザインの好みはありますか？」
自分: 「シンプルでいい」

<!-- page 3 -->

質問されて初めて気づくことって、けっこう多いんですよね。
インタビュー結果が仕様書になる
会話から仕様書への変換図
やりとりが終わったら、こう伝えます。
これだけで、さっきの質疑応答が整った仕様書に変わります。
ふわっとした箇条書きのメモが、項目ごとに整理された文書になる。 ターゲットユーザー、必須機
能、画面構成、優先度。ちゃんと分類してくれます。
自分で仕様書を書こうとすると、何をどの順番で書けばいいか迷いますよね。 Claude Codeはその構
造化を勝手にやってくれるので、楽です。
しかも、そのまま続けられるのがポイント。
仕様確定から実装まで、会話の流れのまま一気に進む。 「指示書を書いて、読み直して、修正して...」
というステップが丸ごとなくなります。
「今の会話内容を仕様書にまとめて」
「この仕様で作って」

<!-- page 4 -->

こんな場面で使える
3つのユースケース — 24時間対応の壁打ち相手
この「インタビュー方式」、アプリ開発だけじゃないです。
Webサイト作成 「どんなサイトがいいか質問して」
プレゼン資料 「プレゼン資料の構成を一緒に考えて。質問形式で」
業務改善 「今の業務の課題を整理したい。ヒアリングして」
「質問形式で」「ヒアリングして」「インタビューして」。 どの言い方でも通じます。好きな表現を使っ
てください。
特に効くのが、壁打ち相手がいない場面。 個人事業主やフリーランスだと、「ちょっと聞いてくれる
人」がいない。 自分の頭だけで考えてると、抜け漏れに気づけないんですよね。
Claude Codeなら24時間いつでも壁打ち相手になってくれます。 嫌な顔もしないし、「それ前にも聞い
たよね」とも言わない。 深夜2時でも遠慮なく相談できる。これは地味にありがたい。
まとめ
「自分で指示を書く」だけがClaude Codeの使い方じゃない
「質問させる」ことで、頭の中を整理してもらえる
インタビュー結果はそのまま仕様書にできる
月3,000円のコンサルタント。24時間対応。嫌な顔しない
