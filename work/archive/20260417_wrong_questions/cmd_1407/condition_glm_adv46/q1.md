### 回答

pretool_check.sh L149-206のblacklistチェックのバイパス可否を4パターンで分析。

#### チェックの仕組み

1. **L156-157**: `python3 <スクリプト>.py`パターンにマッチするか判定（`-c`/`-m`等は除外）
2. **L158**: `grep -oE '[^/[:space:]]+\.py'`でファイル名部分のみ抽出（`/`を含まない = basename相当）
3. **L163-169**: 抽出したファイル名が`config/script_blacklist.txt`と**完全行一致**（`grep -qxF`）するか照合

#### 各パターンの分析

**(a) `python3 /full/path/to/generate_odai_panels.py`（フルパス）**

→ **検出される。** L158の正規表現`[^/[:space:]]+\.py`は`/`を含まない文字列にマッチするため、パスから`generate_odai_panels.py`だけが抽出される。L164の`grep -qxF`でblacklistと完全行一致 → 検出可能。

**(b) `python3 scripts/generate_odai_panels.py`（相対パス）**

→ **検出される。** (a)と同様。L158が`generate_odai_panels.py`だけを抽出 → blacklistと照合。

**(c) `python3 generate_odai_panels.py`（ファイル名のみ）**

→ **検出される。** 最もシンプルなケース。L158でそのまま抽出 → blacklist一致。

**(d) `python3 safe_script.py generate_odai_panels.py`（引数に禁止名）**

→ **検出される（誤検出あり）。** L158はコマンド全体から`.py`にマッチする全文字列を抽出する。`safe_script.py`と`generate_odai_panels.py`の両方が`$PY_FILES`に入る。forループで各々blacklistと照合 → 引数側の禁止名も検出。意図しないファイル名を引数に含めた場合もブロックされるが、足軽運用では実害ほぼなし。

#### 真のバイパス経路

上記4パターンは全て検出可能だが、実際にバイパス可能な経路が3つある:

1. **`python3 -c "exec(open('banned.py').read())"`**: L157の`grep -qE 'python3[[:space:]]+-[cm]'`がTrue → チェック全体スキップ。インラインPythonで禁止スクリプトを丸ごと実行可能
2. **`bash banned.py`**: `python3`パターンにマッチしないためチェック対象外。shebang付き.pyはbash経由でも実行可能
3. **環境変数・コマンド置換**: `python3 $(cat banned_path.txt)` → grep時点で未展開 → `.py`マッチせずスルー
