# AIに漫画を描かせてYouTubeショートを量産した話

## はじめに

YouTubeショートで漫画動画を毎日投稿したい——でも漫画を描くのは時間がかかる。4コマで2〜4時間。毎日投稿なんて無理だ。

そこで「AIに漫画を描かせて、ショート動画にしてYouTubeに上げる」という実験を始めた。結果、1本あたり30分程度で漫画ショートが作れるようになった。

この記事では、実際のワークフローと試行錯誤の記録を共有する。

---

## なぜ「AI漫画ショート」なのか

YouTubeショートは60秒以内の縦型動画。漫画形式は視覚的にわかりやすく、エンタメとして需要がある。

手描きだと1コマ30分〜1時間。AI画像生成に置き換えることで制作時間を大幅短縮できる。

## 全体ワークフロー

実際の制作フローは5ステップ:

1. **候補選定**: YouTube動画から面白いシーンを選ぶ
2. **セリフ抽出**: 音声認識（WhisperX）でセリフを文字起こし
3. **構成作成**: 漫画パネル構成JSONを作る
4. **画像生成**: Gemini APIで各パネルの画像を生成
5. **動画合成**: ffmpegで縦型動画に合成→YouTube非公開アップ

この全工程をCLIツールとシェルスクリプトで半自動化している。

## AI画像生成の実際

### 使用ツール

- **Gemini 2.0 Flash**（画像生成API）: 各パネル画像の生成
- **Claude Code**（CLI）: プロンプト設計と品質管理
- **ffmpeg**（NVENC GPU）: 動画エンコード

### プロンプト設計のコツ

- **黒白指定**: 「monochrome manga style」で統一
- **コマ割り禁止**: 「single panel, no panel borders」を指定（AIが勝手にコマ割りするのを防止）
- **キャラ一貫性**: キャラクターの外見特徴を毎回同じプロンプトに含める
- **吹き出し禁止**: セリフは後から合成するので「no speech bubbles」を指定

### ガチャ問題と対策

AI画像生成は「ガチャ」——同じプロンプトでも毎回違う結果になる。

最初は全パネルを一度に生成していたが、1枚でも崩れると全部作り直し。今は:
1. **まず1パネルだけ試し生成**
2. 問題なければ残りを生成
3. NGパネルだけ個別に再生成（最大3回）

これでAPI呼び出し回数を半分以下に抑えられた。

## 動画化の工程

画像が揃ったらffmpegで動画に合成:

1. パネル画像→PNG連番
2. 音声（元動画から抽出）を結合
3. ffmpegで縦型（1080x1920）動画に変換（NVENC必須）
4. YouTube APIで非公開アップロード

CPUエンコード（libx264）だと32分動画に3時間半かかった。NVENCなら数分。GPUエンコードは必須。

## ハマりどころと対策

**手が6本になる問題**: プロンプトで「normal 5 fingers」と指定しても100%防げない。試し生成→NGなら再生成→3回でダメなら構図変更。

**キャラクターの一貫性**: パネルごとに別人になる。対策はreference image（参考画像）を渡すこと。

**Vertex API 504エラー**: 画像生成APIが頻繁にタイムアウト。チェックポイント機能で完了済みパネルをスキップし、再実行時に最初からやり直さなくて済むようにした。

**吹き出し内の文字化け**: AIが吹き出し内に謎の文字を生成。そもそもAIに吹き出しを描かせないのが正解。セリフは別レイヤー合成。

## コストの話

### Gemini API無料枠の罠

ある時、プロンプト調整しながら3,409枚生成したら無料枠を超過して**22,000円の請求**が来た。

現在の対策:
- 同一パターンは最大3回まで
- 1パネル試し打ち→確認→残り生成
- GCPで月額上限設定（予算アラート付き）

### コスト試算（1本あたり）

| 項目 | コスト |
|------|--------|
| 画像生成（4-6パネル） | 0円（無料枠内）〜数十円 |
| YouTube API | 0円 |
| ffmpeg（GPU） | 電気代のみ |
| 音声認識（WhisperX） | 0円（ローカル実行） |
| **合計** | **0円〜数十円** |

## 成果と今後

- 1本あたり制作時間: **30分〜1時間**（手描きの1/10以下）
- 品質: 「AI感」はあるが、ショート動画としては十分
- 運用: CLIから一撃で非公開アップまで完了

今後の課題:
1. キャラクター一貫性の向上
2. セリフ合成の自動化
3. 投稿の自動化（タイトル・説明欄・サムネイル）
4. ガチャ成功率の向上（現在7-8割）

## まとめ

AIに漫画を描かせるのは「手描きの代替」ではない。しかし「ショート動画用エンタメ」としては実用レベル。

重要なのはAIの弱点（手、一貫性、文字）をワークフロー側でカバーすること。そしてコスト管理を怠らないこと。「AIに任せきり」ではなく「AIの得意なことを活かしつつ、人は確認する」という体制が現時点の最適解。
