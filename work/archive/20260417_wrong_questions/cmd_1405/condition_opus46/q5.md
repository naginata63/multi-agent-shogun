# AIに漫画を描かせてYouTubeショートを量産した話

## はじめに

「切り抜き動画、もっと目を引くサムネとか作れないかな」

そんな軽い思いつきが、まさか**AIに漫画を描かせてYouTubeショートを自動生成するパイプライン**を作ることになるとは思わなかった。

私はドズル社（マイクラ実況グループ）の切り抜きチャンネルを運営している。登録者は約1,000人。普段はハイライト切り抜きやショート動画を投稿しているが、あるとき「名場面を漫画にしたら面白いんじゃないか」と思い立った。

結論から言うと、**Google Gemini（画像生成API）+ Claude（テキスト処理）+ ffmpeg + YouTube API**を組み合わせて、1本のショート動画を約30分で制作できる仕組みが完成した。この記事ではその過程と、実際にぶつかった壁、そして「AIに絵を描かせる」ことのリアルな体験を書く。

## なぜ「漫画ショート」なのか

YouTubeショートの世界では**最初の1秒で目を引けるかどうか**が全て。テロップだけの切り抜きよりも、漫画のコマ割りで「おっ、なんだこれ」と思わせた方がスクロールを止められる。

実際に投稿してみた結果:
- 通常の切り抜きショート: 平均視聴回数 200〜500回
- 漫画ショート: 平均視聴回数 500〜2,000回

数倍の差が出た。「手描き風」の画が入ると、視聴者の指がスクロールから離れる瞬間がある。

## 制作パイプラインの全体像

```
元動画 → 音声抽出 → 文字起こし → 名場面選定 → セリフ整理
                                                    ↓
コマ割りテンプレート ← panels JSON ← 漫画構成生成（Claude）
                                                    ↓
キャラ立ち絵 + 背景ref → Gemini画像生成 → パネルPNG
                                                    ↓
                        パネルPNG + 音声 → ffmpegスライドショー → 縦型動画
                                                    ↓
                                        YouTube非公開アップ → 殿（自分）確認 → 公開
```

### 各工程の詳細

**1. 音声抽出・文字起こし**
元動画からDemucsでボーカルを分離し、WhisperXで文字起こし。話者IDを振って誰が何を言ったか特定する。ここは人力ゼロ、完全自動。

**2. 名場面選定**
「どの場面を漫画にするか」は最も重要な判断。5つのAI（Claude×3 + GPT×1 + Gemini×1）に独立して候補を出させ、投票で決める「集合知方式」を採用している。1社に偏らない多様な視点が得られる。

**3. 漫画構成生成（panels JSON）**
選ばれた場面のセリフ・状況をClaudeに渡して、パネル構成を自動生成する。各パネルには:
- コマ割りテンプレート（36種類から選択）
- キャラクター配置
- 表情指定（怒り/驚き/笑顔/泣き等）
- 背景指示
- セリフ配置
- 演出指示（director_notes）

が含まれる。この**panels JSON**が漫画制作の設計図になる。

**4. Gemini画像生成**
panels JSONに基づいて、Google Gemini（Vertex AI経由）で各パネルの画像を生成する。キャラクターの立ち絵や背景のリファレンス画像をGCS（Google Cloud Storage）にアップロードし、GCS URI方式で渡す。

ここが最もドラマのある工程。

**5. 動画合成**
生成されたパネルPNGと元音声をffmpegで合成し、縦型（1080×1920）のショート動画にする。各パネルの表示時間はセリフの長さに合わせて自動調整。

**6. YouTube非公開アップ・確認・公開**
YouTube Data APIで非公開アップロード。自分で確認して、OKなら公開。説明欄にはAI画像生成の使用を明記している。

## 「ガチャ」との戦い

AI画像生成で最も苦労したのは**品質のばらつき**だ。

同じプロンプトを投げても、出てくる画像は毎回違う。キャラクターの顔が崩れたり、指の本数がおかしかったり、背景に意味不明なオブジェクトが混入したり。

最初は「何度でも回せばいいじゃん」と思っていた。しかし:

- **Gemini APIは無料ではない**。Vertex AI経由で1回の画像生成に数円〜十数円かかる
- 「もう1回」「もう1回」と際限なく回した結果、1ヶ月で**22,000円**のAPI費用が発生
- しかも、回数を増やしても品質が上がるわけではない

この痛い経験から「**ガチャ上限3回ルール**」を設けた。同じパネルの画像生成は最大3セットまで。それでダメなら、プロンプト（演出指示）を見直す。

## 「演出指示」がAI画像の品質を決める

画像生成AIに「いい感じの漫画を描いて」と言っても、いい絵は出てこない。

重要なのは**director_notes**（演出指示）の精度だ。具体的には:

```
❌ ダメな例: 「ドズルが驚いている場面」
✅ 良い例: 「ドズル（赤髪・赤甲冑の大男）が画面中央で目を見開き、
　　　　　口を大きく開けて「えっ!?」と叫んでいる。背景はマイクラの
　　　　　エンドラの巣（黒い石の柱と紫のパーティクル）。
　　　　　カメラはドズルの胸から上をアップで捉え、驚きの表情を強調する」
```

場面の文脈、キャラの外見的特徴、カメラワーク、感情表現——ここまで言語化して初めて、AIは「漫画っぽい」絵を描いてくれる。

さらに重要なのが**リファレンス画像**。キャラクターの立ち絵（表情差分付き）を事前にGeminiで生成しておき、毎回のパネル生成時に「このキャラクターはこういう見た目です」と添付する。これでキャラの一貫性が大幅に向上した。

## 吹き出しセリフは後から載せる

AI画像生成で日本語テキストを画像内に正しく描画するのは**現時点では不可能**に近い。文字が化けたり、変な位置に入ったり、そもそも読めなかったりする。

そこで「画像にはセリフを描画させない」方針に切り替えた。

1. Geminiには「吹き出しの中は白い空白にして」と指示
2. 生成された画像の吹き出し領域をflood fill（塗りつぶし）で完全に白く処理
3. Pythonスクリプト（serif_replace.py）で正確なフォント・サイズ・位置でセリフを合成

この3ステップで、AI画像+正確な日本語セリフの漫画パネルが完成する。手間は増えるが、セリフの品質は確実にコントロールできる。

## マルチエージェントで「量産」する仕組み

1人でこの全工程をやると、1本のショート動画に3〜4時間かかる。

そこで**Claude Codeのマルチエージェントシステム**を構築した。tmuxで8つのペインを同時稼働させ:

- **将軍（Shogun）**: 全体指揮。殿（自分）の指示を受けてコマンド発令
- **家老（Karo）**: タスク管理。足軽への作業割当、品質チェック
- **足軽（Ashigaru）×7**: 実作業担当。並列でコード実装、API実行、動画合成
- **軍師（Gunshi）**: 品質検査、戦略分析

このシステムのおかげで、複数の動画を並列で制作できるようになった。ある足軽が動画Aの画像生成をしている間に、別の足軽が動画Bの文字起こしをしている、というイメージだ。

## やってみて分かったこと

### AI画像生成は「魔法」ではない
プロンプトを投げればいい絵が出てくる——そんな甘い世界ではなかった。リファレンス画像の準備、演出指示の精緻化、品質チェック、セリフ後処理。人間の「目」と「判断」は依然として不可欠。

### コスト管理は生命線
「無限にガチャを回せる」感覚は危険。月額予算を決め、1パネルあたりのコスト上限を設け、ガチャ回数に上限を設ける。この規律がなければ、趣味の切り抜きチャンネルが赤字事業になっていた。

### 「60点の量」が「100点の1本」に勝つ
完璧な漫画ショートを1本作るより、そこそこの品質で10本出した方がチャンネルは伸びる。AI画像生成の品質は手描きに及ばないが、「量を出せる」ことに価値がある。視聴者は意外と気にしない。

### パイプライン化が全て
個別のAIツールの性能より、**工程をつなげる仕組み**の方が100倍重要。音声→テキスト→構成→画像→動画→公開がシームレスにつながって初めて「量産」が成立する。

## 今後やりたいこと

- **表情差分の自動選択**: 現在は手動で表情を選んでいるが、セリフの感情分析から自動で選びたい
- **視聴者反応のフィードバックループ**: どのパネル構成が再生回数に効くのか、データで検証
- **リアルタイムプレビュー**: panels JSONを編集するとブラウザ上で即座にパネル画像がプレビューできるツール（一部実装済み）

## おわりに

「AIに漫画を描かせる」と書くと簡単そうに聞こえるが、実態は**AIと人間の共同作業**だ。AIは下絵を描き、人間は構成を考え、品質を判断し、セリフを載せ、最終的に「これを世に出していいか」を決める。

完全自動化ではない。でも、1人で月に数十本のショート動画を出せるようになった。それは、AIなしでは絶対に不可能だったこと。

もし同じように切り抜きチャンネルを運営していて、「もうちょっと目を引く動画を作りたい」と思っている人がいたら、まずは1枚だけAIに漫画を描かせてみてほしい。きっと、「おっ、これいけるかも」と思う瞬間がある。

その1枚が、パイプラインの第一歩になる。
