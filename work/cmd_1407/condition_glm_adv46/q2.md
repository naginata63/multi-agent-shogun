# Q2: inbox_write.sh + inbox_watcher.sh 統合リファクタリング設計

advisor model: claude-opus-4-6[1m]
実測開始時刻: 2026-04-17T13:28:00

## 現状分析

### inbox_write.sh の責務
1. flock付きファイル書き込み（排他制御）
2. YAML形式でのメッセージ追加
3. inotifywaitイベントのtrigger（touch）

### inbox_watcher.sh の責務
1. inotifywait によるファイル変更監視
2. tmux send-keys によるエージェント起床
3. エスカレーション（2min: escape+nudge, 4min: /clear）

### 結合点
- writeが書き込むファイルをwatcherが監視する
- 共通のファイルパス規約: queue/inbox/{agent}.yaml
- 共通のエージェント-paneマッピング

## 統合案設計

### 方針: 段階的統合（Big Bang禁止）

統合は3フェーズで実施。各フェーズで既存機能を維持しつつ段階的に移行する。

### Phase 1: 共通設定の外部化

新しいファイル: config/inbox_config.sh

```bash
#!/usr/bin/env bash
INBOX_DIR="${REPO_DIR}/queue/inbox"
INBOX_LOCK_DIR="${REPO_DIR}/logs/inbox_locks"
declare -A AGENT_PANE_MAP=(
  [karo]="multiagent:0.0"
  [gunshi]="multiagent:0.1"
  [ashigaru1]="multiagent:0.2"
)
ESCALATION_ESCAPE_SEC=120
ESCALATION_CLEAR_SEC=240
```

メリット: paneマッピングの二重管理を排除。将来のエージェント追加時も1箇所修正。

### Phase 2: inbox_core.sh — 共通関数ライブラリ

inbox_append(): flock付きYAML追記（現行inbox_write.shのロジックを関数化）
inbox_unread_count(): 未読数取得
inbox_nudge(): エージェント起床（pane lookup + send-keys）

### Phase 3: 統合スクリプト inbox_manager.sh

send/watch/countのサブコマンド形式でwriteとwatcherの機能を統合。

## 移行戦略
