## Q4: エッジケース考慮 — generate_manga_short.py Vertex API 504根本対策
- advisor model: claude-opus-4-6[1m]
- 開始時刻: 2026-04-17T14:07:34
- 終了時刻: 2026-04-17T14:08:17
- advisor呼び出し: 1回
- advisor応答要約: status:assignedに戻してtarget_pathを認識させる

### 回答

#### 現状のコード分析

`generate_panel_with_retry()`（L2134-2191）がリトライを担当:
- `MAX_RETRIES=3`, `BASE_WAIT=120s`, `MAX_WAIT=600s`
- `RETRYABLE_ERRORS = ("429", "RESOURCE_EXHAUSTED", "503")`
- **504 DEADLINE_EXCEEDEDがRETRYABLE_ERRORSに含まれていない**（根本問題）
- L2536: `HttpOptions(timeout=120_000)` — HTTP通信のタイムアウトは120秒

#### 504 DEADLINE_EXCEEDEDが頻発する3原因

1. **リクエスト過大**: ref_image（参考画像）複数 + 長プロンプトでサーバー側処理が60秒超過
2. **Vertex API同時実行制限**: フリーティアは2-4並列。キュー溢れでタイムアウト
3. **リトライ対象外**: RETRYABLE_ERRORSに"504"が含まれないため、504は即座に失敗扱い

#### 根本対策設計（4段階）

**対策1（即効）: RETRYABLE_ERRORSに504を追加**

```python
# L2131
RETRYABLE_ERRORS = ("429", "RESOURCE_EXHAUSTED", "503", "504", "DEADLINE_EXCEEDED")
```

変更1行で504/DEADLINE_EXCEEDEDが指数バックオフレトライの対象になる。

**対策2（高）: チャンク分割アプローチ**

現行の`generate_panel()`は1パネル1リクエスト。504はref_imageの画像サイズに起因することが多い:

```python
# generate_panel()内 L1760付近 — ref_image送信前
MAX_REF_SIZE = 2 * 1024 * 1024  # 2MB上限
for ref_path in ref_image_paths:
    if os.path.getsize(ref_path) > MAX_REF_SIZE:
        # Pillowでリサイズ（品質保持）
        from PIL import Image
        img = Image.open(ref_path)
        img.thumbnail((1024, 1024))
        resized = ref_path.with_suffix('.resized.png')
        img.save(resized, 'PNG')
        ref_path = resized
```

**対策3（中）: パネル間ウェイト + ジッター**

Vertex APIのレート制限を回避するため、パネル間にランダムウェイトを挿入:

```python
# main()内のパネル生成ループ（L2600付近）
import random

for i, panel in enumerate(panels_to_gen):
    if i > 0:
        wait = random.uniform(3, 8)  # 3-8秒のランダムウェイト
        time.sleep(wait)
    result = generate_panel_with_retry(client, panel, ...)
```

**対策4（中）: チェックポイント+レジューム**

既存コードのmain()ループ（L2600付近）に、完了済みパネルのスキップ機能を追加。CLAUDE.mdの中間成果物ルールに合致:

```python
# main()ループ開始前
checkpoint_path = output_dir / "generation_checkpoint.json"
done_panels = set()
if checkpoint_path.exists():
    done_panels = set(json.loads(checkpoint_path.read_text()).get("done", []))
    print(f"  Resuming: skipping {len(done_panels)} completed panels")

for panel in panels_to_gen:
    if panel["id"] in done_panels:
        continue
    result = generate_panel_with_retry(...)
    if result["status"] == "success":
        done_panels.add(panel["id"])
        checkpoint_path.write_text(json.dumps({"done": list(done_panels)}))
```

#### 並列化の可否

Vertex APIはフリーティアで同時2-4リクエスト。パネル生成は**順次実行が安全**。並列化する場合は:
- `asyncio.Semaphore(2)` で同時実行数を制限
- サーキットブレーカーと組み合わせて連鎖防止
- ただし実装コストが高いため、まずは順次+ウェイトで対応

| パネル間ウェイト | 中 | ~3行 | レート制限回避 |
