## 問題の概要

`generate_manga_short.py` から Vertex AI Gemini API（画像生成）を呼び出す際、`504 DEADLINE_EXCEEDED` が頻発する。リトライは実装済みだが、根本的な対策が不十分。

## 原因分析

### 504発生の3パターン

1. **リクエストタイムアウト**: 単一リクエストの処理時間がサーバー側タイムアウト（通常60秒）を超過
   - 原因: プロンプト過長、ref_image（参考画像）の大容量、サーバー側負荷

2. **キューイング遅延**: Vertex AI側のリクエストキューが溢れ、待ち時間がタイムアウトに到達
   - 原因: フリーティア/従量課金の同時リクエスト制限（通常2-4並列）

3. **一時的サービス障害**: Vertex AI側の間欠的な500系エラー
   - 原因: Google Cloud側のメンテナンス・過負荷

### 現状の対策と限界

現行実装（推測）:
```python
for attempt in range(MAX_RETRIES):
    try:
        response = client.generate_content(...)
        return response
    except Exception as e:
        if "504" in str(e):
            time.sleep(2 ** attempt)
        else:
            raise
```

**限界**:
- 単純リトライは同一パラメータで再送するため、過長プロンプト由来の504は永久に再発
- Exponential backoffの上限が設定されていない
- リトライ間のポーリングが全体の処理時間を膨張させる

### 対策1: リクエスト最適化（予防）

```python
def optimize_request(prompt, ref_images):
    """リクエストを最適化して504リスクを下げる"""

    # 1. プロンプト長の制限
    MAX_PROMPT_CHARS = 2000
    if len(prompt) > MAX_PROMPT_CHARS:
        prompt = prompt[:MAX_PROMPT_CHARS] + "\n（以下省略）"

    # 2. 参考画像のサイズ制限
    MAX_IMAGE_SIZE = 4 * 1024 * 1024  # 4MB
    optimized_images = []
    for img in ref_images:
        if os.path.getsize(img) > MAX_IMAGE_SIZE:
            # リサイズして縮小
            optimized = resize_image(img, max_size=MAX_IMAGE_SIZE)
            optimized_images.append(optimized)
        else:
            optimized_images.append(img)

    return prompt, optimized_images
```

効果: タイムアウトの主要因（過長プロンプト・大容量画像）を排除。

### 対策2: サーキットブレーカー（連鎖防止）

```python
class APICircuitBreaker:
    def __init__(self, failure_threshold=3, recovery_timeout=300):
        self.failure_count = 0
        self.failure_threshold = failure_threshold
        self.recovery_timeout = recovery_timeout
        self.last_failure_time = None
        self.state = "closed"  # closed, open, half_open

    def record_failure(self):
        self.failure_count += 1
        self.last_failure_time = time.time()
        if self.failure_count >= self.failure_threshold:
            self.state = "open"
            logger.warning("Circuit breaker OPEN: %d consecutive failures",
                          self.failure_count)

    def record_success(self):
        self.failure_count = 0
        self.state = "closed"

    def can_proceed(self):
        if self.state == "closed":
            return True
        if self.state == "open":
            if time.time() - self.last_failure_time > self.recovery_timeout:
                self.state = "half_open"
                return True
            return False
        return True  # half_open

    @property
    def wait_seconds(self):
        """次回リトライまでの待機時間"""
        if self.state == "open":
            remaining = self.recovery_timeout - (time.time() - self.last_failure_time)
            return max(0, remaining)
        return 0
```

効果: 連続504時にAPI呼び出しを停止し、無駄なリクエストを防止。5分後にハーフオープンで再試行。

### 対策3: パネル間ウェイト（レート制限回避）

```python
import random

PANEL_DELAYS = {
    "normal": (2, 5),      # 通常パネル間: 2-5秒
    "after_504": (10, 30), # 504後: 10-30秒
    "after_429": (30, 60), # 429後: 30-60秒
}

async def generate_panel_with_delay(panel_config, circuit_breaker):
    if not circuit_breaker.can_proceed():
        logger.info("Circuit breaker open. Waiting %.0f seconds",
                    circuit_breaker.wait_seconds)
        await asyncio.sleep(circuit_breaker.wait_seconds)

    delay_range = PANEL_DELAYS["normal"]
    await asyncio.sleep(random.uniform(*delay_range))

    try:
        result = await generate_panel(panel_config)
        circuit_breaker.record_success()
        return result
    except Exception as e:
        circuit_breaker.record_failure()
        if "504" in str(e):
            delay_range = PANEL_DELAYS["after_504"]
            await asyncio.sleep(random.uniform(*delay_range))
        elif "429" in str(e):
            delay_range = PANEL_DELAYS["after_429"]
            await asyncio.sleep(random.uniform(*delay_range))
        raise
```

### 対策4: チェックポイント+レジューム（長時間実行の耐障害性）

```python
import json

CHECKPOINT_FILE = "work/manga_generation_checkpoint.json"

def save_checkpoint(panel_id, status, output_path=None):
    checkpoint = load_checkpoints()
    checkpoint[panel_id] = {
        "status": status,
        "output_path": output_path,
        "timestamp": time.time(),
    }
    with open(CHECKPOINT_FILE, "w") as f:
        json.dump(checkpoint, f, indent=2)

def load_checkpoints():
    if os.path.exists(CHECKPOINT_FILE):
        with open(CHECKPOINT_FILE) as f:
            return json.load(f)
    return {}

def generate_manga_with_resume(panels):
    checkpoint = load_checkpoints()
    for panel in panels:
        pid = panel["id"]
        if pid in checkpoint and checkpoint[pid]["status"] == "done":
            logger.info("Skipping %s (already done)", pid)
            continue

        save_checkpoint(pid, "in_progress")
        try:
            result = generate_panel_with_delay(panel, breaker)
            save_checkpoint(pid, "done", result.output_path)
        except Exception as e:
            save_checkpoint(pid, "failed")
            if isinstance(e, CircuitOpenError):
                logger.error("Circuit breaker open. Stopping generation.")
                break
            raise
```

効果: 途中で504連発しても、完了済みパネルはスキップして再開可能。再実行コストを最小化。

### 対策5: フォールバック戦略

```python
FALLBACK_STRATEGIES = [
    {"action": "retry", "max_attempts": 3, "backoff": "exponential"},
    {"action": "simplify_prompt", "max_prompt_chars": 1000},
    {"action": "remove_ref_image"},
    {"action": "skip_panel", "placeholder": "⚠️ 生成失敗"},
]

async def generate_with_fallback(panel, breaker):
    for strategy in FALLBACK_STRATEGIES:
        try:
            if strategy["action"] == "retry":
                for attempt in range(strategy["max_attempts"]):
                    result = await generate_panel(panel)
                    return result
            elif strategy["action"] == "simplify_prompt":
                panel["prompt"] = panel["prompt"][:strategy["max_prompt_chars"]]
                result = await generate_panel(panel)
                return result
            elif strategy["action"] == "remove_ref_image":
                panel.pop("ref_image", None)
                result = await generate_panel(panel)
                return result
            elif strategy["action"] == "skip_panel":
                return PlaceholderResult(panel["id"], strategy["placeholder"])
        except Exception:
            continue
    raise GenerationError(f"All fallback strategies exhausted for {panel['id']}")
```

## 結論

最優先は**対策1（リクエスト最適化）**と**対策4（チェックポイント+レジューム）**。前者は504の発生頻度を下げ、後者は504が起きても再実行コストを最小化する。この2つだけで実害の80%を削減可能。サーキットブレーカーは連鎖防止に有効だが、まずは予防（対策1）と復旧（対策4）を優先すべき。
