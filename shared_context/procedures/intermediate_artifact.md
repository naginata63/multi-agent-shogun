# 中間成果物保存ルール

重い外部処理 (WhisperX, Demucs, Gemini API, Claude CLI 等) の中間成果物は **必ずファイルに保存**せよ。

## 鉄則

1. **保存必須**: 実行に 1 分以上かかる処理の出力は変数だけでなく**ファイル**にも書き出す
2. **保存先**: `/tmp` 禁止 (再起動で消える)・`work/` 配下またはプロジェクト出力ディレクトリに保存
3. **キャッシュ再利用**: 2回目以降は保存済みファイルから読込む option (`--cache`, `--wx-cache`等) を実装
4. **命名規則**: `{処理名}_{動画ID}_{タイムスタンプ}.{拡張子}` (例: `wx_words_rDYmTp_20260312.json`)

## 教訓
- cmd_597: WhisperX (5-10分/回) を 3 回再実行 → 合計 15-30 分浪費。中間データ保存していれば 2 回目以降は瞬時。
